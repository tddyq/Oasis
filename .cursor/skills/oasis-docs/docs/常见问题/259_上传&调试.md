# 上传&调试

## Q1：工程完成后，如何在手机上进行测试

1. 目前真机测试只支持安卓手机，需先安装手机客户端，将pak包放入安卓手机安装即可，pak包路径为：``…\WeGameApps\rail_apps\和平精英绿洲启元编辑器(2001776)\安卓``
2. 安装完成后，开发者需将工程上传，分为测试上传与发布上传，其中测试上传的版本仅用于内部的玩法流程测试；发布上传的版本可用于部署到正式服，可以模拟玩法在和平精英绿洲启元大厅的上架状态，适用于更接近实际发布环境的测试。详细内容可查看 [上传工程与真机测试](https://developer.gp.qq.com/wikieditor/#/catalog/372)

<br>

## Q2：真机测试时DS日志显示在加载资源对象时报错，但客户端日志正常，这种情况是什么原因导致的

工程cook时不会将贴图、材质等表现类资源打进DS包，因此在DS端执行这类资源加载会报错，需要确保表现类逻辑只在客户端执行。

## Q3：工程上传后为什么在移动端上找不到该玩法

工程上传后不显示玩法可能有以下原因，需要逐一排查：
1. 上传包含工程包烘焙和服务器推送等多个环节，上传完成只代表烘焙结束，可能存在延迟显示的情况，通常时间间隔在10分钟内
2. 编辑器版本与和平精英外研线客户端版本号的前三位要保持一致（如：1.34.X），当编辑器新版本发布且wegame自动更新后上传的工程，如果客户端未同步更新则会因版本不一致而无法显示玩法
3. 上传后的玩法工程包存在定时清理策略，超过一周的玩法版本将被自动清理

<br>

## Q4：真机测试发现贴图/材质变白模，甚至有些场景中的物件资源不加载，但是PIE调试正常，这种情况如何解决

如果出现异常的资源为资源目录下的公共资源，则可能被和平精英标记为旧资源而剔除主包导致，可以尝试利用 [Lua资源引用](https://developer.gp.qq.com/wikieditor/#/catalog/314?autoJump=%E9%80%9A%E8%BF%87Lua%E8%B5%84%E6%BA%90%E5%BC%95%E7%94%A8%E6%8C%87%E5%AE%9A%E8%B5%84%E6%BA%90) 的方式解决；如果为工程内导入的资源，则可能因cook问题导致，可以尝试重新上传解决。

<br>

## Q5：通过画刷工具绘制的地表贴图，PIE调试时正常显示，但真机测试时被还原成默认材质是什么原因

刷地形需要变更地表材质引用的贴图资源，因此需要替换landscape引用的地表材质实例，详细内容可查看 [地表材质使用介绍](https://developer.gp.qq.com/wikieditor/#/catalog/300?autoJump=%E5%9C%B0%E8%A1%A8%E6%9D%90%E8%B4%A8%E4%BD%BF%E7%94%A8%E4%BB%8B%E7%BB%8D)。

<br>

## Q6：为什么同样的场景，编辑器视口包括PIE调试时显示的光照效果和真机测试效果不同？

由于PIE调试的渲染管线与真机的渲染管线不同，所以存在差异，目前需要调整光照来达到预期效果。

<br>

## Q7：场景中的点光源在PIE调试时正常，但是打包到真机上就丢失了，是不支持点光源功能吗

目前确实不支持点光源/聚光源功能，需要等待后续新版本支持，可使用【天气控件】处理游戏光照，详细内容可查看 [天气系统](https://developer.gp.qq.com/wikieditor/#/catalog/225)。

<br>

## Q8：编辑器是否有配备测试用的IOS安装包，如何自测玩法在IOS上的表现效果

目前仅提供了安卓测试包，玩法提审后内部测试会覆盖IOS并提供测试结果反馈，开发者自测阶段无需关注IOS端。
