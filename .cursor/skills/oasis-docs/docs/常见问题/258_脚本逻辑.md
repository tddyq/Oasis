# 脚本逻辑

## Q1：在已成功加载的子关卡中，为什么无法获取到放置在关卡中的出生点对象

子关卡的出生点通过 ``FindPlayerStartByBornPointID`` 无法获取，建议把出生点全部放置在主关卡中，或者为出生点添加Tag，使用 [GetAllActorsWithTag](https://developer.gp.qq.com/api/#/searchContent/UGCActorComponentUtility?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCActorComponentUtility.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCActorComponentUtility&autoJump=GetAllActorsWithTag) 方法查找。

## Q2：如何控制Actor实体或者Component组件在游戏中的显示与隐藏

- 控制Actor的显隐可使用 [SetActorHiddenInGame](https://developer.gp.qq.com/api/#/searchContent/AActor?classDetailShow=true&path=class%2Fdetail%2FOthers%2FAActor.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=AActor&autoJump=SetActorHiddenInGame) 方法，例如： ``actor:SetActorHiddenInGame(true)``
- 控制Component的显隐可使用 [SetVisiblity](https://developer.gp.qq.com/api/#/searchContent/USceneComponent?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUSceneComponent.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=USceneComponent&autoJump=SetVisibility) 方法，例如： ``Component:SetVisiblity（true, false, false）``

<br>

## Q3：如何获取玩家的Pawn，PlayerController以及PlayerState

获取玩家的Pawn：
1. 玩家的 Pawn 通常是通过 PlayerController 访问的，因为每个 PlayerController 都控制一个 Pawn，可以使用 PlayerController 获取玩家的Pawn，方法：``PlayerController:GetPlayerCharacterSafety()``，客户端与服务端皆可
2. 通过 PlayerKey 获取玩家的Pawn：``UGCGameSystem.GetPlayerPawnByPlayerKey(PlayerKey)``，在客户端服务端皆可
3. 在客户端获取玩家的Pawn：``UGCGameSystem.GetLocalPlayerPawn()``、``GameplayStatics.GetPlayerCharacter(gameState, 0)``，GetPlayerCharacter方法第一个参数为WorldContextObject，它提供了在【哪个世界】 中查找或执行操作的信息，所以此处需传入一个UObject对象，推荐传入gamestate


获取玩家的PlayerController：
1. 通过玩家的Pawn获取PlayerController：``Pawn:GetPlayerControllerSafety()``、``UGCGameSystem.GetPlayerControllerByPlayerPawn(PlayerPawn)``，客户端与服务端皆可
2. 通过 PlayerKey 获取玩家的PlayerController：``UGCGameSystem.GetPlayerControllerByPlayerKey(PlayerKey)``，支持服务端与客户端
3. 通过 UID 获取玩家的 PlayerController：``UGCGameSystem.GetPlayerControllerByUID(UID)``，仅支持服务端
4. 客户端获取玩家的PlayerController：``UGCGameSystem.GetLocalPlayerController()``、``GameplayStatics.GetPlayerController(gameState, 0)``


获取玩家的PlayerState：
1. 通过PlayerController获取玩家的PlayerState，直接访问属性PlayerState，方法：``PlayerController.PlayerState``，客户端服务端皆可
2. 通过 PlayerKey 获取 PlayerState：``UGCGameSystem.GetPlayerStateByPlayerKey(PlayerKey)``， 客户端服务端皆可
3. 通过 UID 获取玩家的 PlayerState：``UGCGameSystem.GetPlayerStateByUID(UID)``， 仅支持服务端
4. 客户端获取玩家的 PlayerState：``UGCGameSystem.GetLocalPlayerState()``

## Q6：如何获取真实的服务器时间,在外研线环境下能否修改服务器时间

1. 由于和平精英现网的限制，不建议使用os库函数查询时间，统一调用 [UGCGameSystem.GetServerTimeSec](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=GetServerTimeSec) 获取
2. 自定义修改服务器时间的功能尚在研发中，计划于后续版本支持

## Q7：API手册中的RankingListManager与排行榜模板中的RankingListManager有什么区别，如何调用排行榜中的UpdateScore接口

模板中的RankingListManager是静态类，用于将模板里声明的函数统一封装起来；而API手册中的 [RankingListManager](https://developer.gp.qq.com/api/#/searchContent/RankingListManager?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%95%86%E4%B8%9A%E5%8C%96%E4%B8%8E%E5%8A%9F%E8%83%BD%E6%A8%A1%E6%9D%BF%2FRankingListManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=RankingListManager) 实际上是排行榜的GamePart模块化派生类，概念上为排行榜功能模块的全局管理器，由于内部实现的原因，两者命名相同因此容易被混淆。

以UpdateScore函数为例，调用示例如下：

``` lua
local GamePartActorInst = UGCGamePartSystem.GetGamePartGlobalActor("RankingListManager");
GamePartActorInst:UpdateScore(PlayerController, 10001, 1, 100, true)
```

除了排行榜模板外，其他模板也存在类似设计，开发者可以简单理解：
- 模板中封装的函数可以通过静态函数的方式直接调用
- GamePart类函数需要获取到模板对应的管理器实例，以对象函数的形式调用

## Q8：SpawnActor函数能否用于创建怪物对象

[UGCActorComponentUtility.SpawnActor](https://developer.gp.qq.com/api/#/searchContent/UGCActorComponentUtility?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCActorComponentUtility.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCActorComponentUtility&autoJump=SpawnActor) 是通用的用于生成指定类型Actor的函数，虽然可以用于生成怪物，但是更推荐使用 [UGCGenericCharacterSystem.SpawnGenericCharacter](https://developer.gp.qq.com/api/#/searchContent/UGCGenericCharacterSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E6%80%AA%E7%89%A9%E7%B3%BB%E7%BB%9F%2FUGCGenericCharacterSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGenericCharacterSystem&autoJump=SpawnGenericCharacter) 函数，[UGCGenericCharacterSystem](https://developer.gp.qq.com/api/#/searchContent/UGCGenericCharacterSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E6%80%AA%E7%89%A9%E7%B3%BB%E7%BB%9F%2FUGCGenericCharacterSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGenericCharacterSystem) 是专用于新怪物系统的函数库。

需要注意的是，旧怪物的基类与新怪物不同，所以旧怪只能使用 [UGCActorComponentUtility.SpawnActor](https://developer.gp.qq.com/api/#/searchContent/UGCActorComponentUtility?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCActorComponentUtility.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCActorComponentUtility&autoJump=SpawnActor) 创建。

<br>

## Q9：EObjectTypeQuery枚举类对应的各值分别代表什么含义

对应的是带有物理碰撞对象的碰撞预设值中【物体响应】的各值，按顺序一一对应，例如“ObjectTypeQuery1”对应“WorldStatic”。

<br>

## Q10：ETraceTypeQuery枚举类对应的各值分别代表什么含义

对应的是带有物理碰撞对象的碰撞预设值中【追溯响应】的各值，按顺序一一对应，例如“TraceTypeQuery1”对应“Visibility”。

<br>

## Q11：通过SpawnActor生成的物件对象PIE调试时能正常创建及显示，但在移动端未显示出来是什么原因

SpawnActor第二个参数通过LoadClass加载类引用，加载的路径需要以Asset作为根目录，否则因为路径错误导致无法查找到对应的资源，正确示例如下：

``` lua
  UGCObjectUtility.LoadClass(UGCGameSystem.GetUGCResourcesFullPath('Asset/Blueprint/BP_Horse.BP_Horse_C'))
```

<br>

## Q12：调用引擎类的函数时，例如UKismetSystemLibrary.IsValidClass(ObjectClass)，为什么日志显示报错

UE引擎函数与和平封装的UGC函数不同，调用时需要去掉U前缀，正确的写法：``KismetSystemLibrary.IsValidClass(ObjectClass)``
，建议尽量查找使用UGC函数替代，例如该函数可由 [UGCObjectUtility.IsObjectValid](https://developer.gp.qq.com/api/#/searchContent/UGCObjectUtility?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCObjectUtility.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCObjectUtility&autoJump=IsObjectValid) 替代。
