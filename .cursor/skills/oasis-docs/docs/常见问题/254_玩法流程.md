# 玩法流程

## Q1：游戏运行时出现较大的网络延迟、波动甚至断开服务器，该如何排查

该类问题主要由于网络流量负载过大造成，可从两方面排查：
- RPC：检查可靠RPC调用的数量和频率，大量的可靠RPC调用会导致网络带宽和延迟的压力，特别是在需要保证每个调用都成功的情况下。可尽量减少不必要的可靠RPC，或者替换使用不可靠RPC
- 属性同步：属性同步目前只支持Lua的数据类型，即bool、number（int，float）、string、table以及nil几种类型，其他的如actor、component等UE对象类型，以及FVector、FHitResult等UE结构体，都不可以直接使用属性同步，在同步之前需先将数据转为table形式

## Q2：管理平台观察到玩法在灰度/现网中出现很多DS超时，应该如何处理

首选需要确保对局流程在DS有效时长内能正确结束，且结束后需要主动关闭DS，防止DS进程空跑，具体步骤可参考 [游戏结束时的逻辑处理](https://developer.gp.qq.com/wikieditor/#/catalog/272)。
