# 表格管理器

在游戏制作中，表格是存储和管理各类游戏数据的必备工具，表格管理器就是编辑器中专门用于筛选和处理表格的工具，它为开发者提供了一套统一的表格创建、浏览、编辑与数据流转解决方案。

<br>

## 表格类型

根据功能用途，表格管理器将表格划分为以下类型，各类型的具体功能与操作权限如下：

||自定义表格|功能表格|模板功能表格|内置表格|
|:-:|:-:|:-:|:-:|:-:|
|表格创建|√|√|||
|表格浏览|√|√|√|√|
|表结构修改|√||||
|行数据编辑|√|√|√||
|删除|√|√|√||
|导入导出|√|√|√||

<br>

## 管理器功能

### 表格创建
> 当前仅支持创建自定义表格与功能表格

首先在【表格类型区】选择表格类型，然后在【表格浏览区】的【Template】单击选择表格模板，点击“选择模板创建”，在弹出的【输入名称】窗口中输入表格名称，最后点击“确定”，成功创建一个全新的表格。

![草稿绘图-创建表格.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/hzA5U%E8%8D%89%E7%A8%BF%E7%BB%98%E5%9B%BE-%E5%88%9B%E5%BB%BA%E8%A1%A8%E6%A0%BC.drawio.png)

---

### 表格浏览

首先在【表格类型区】选择表格类型，然后前往【表格浏览区】的【Asset Tree】搜索定位目标表格，双击表格后，在【表格结构编辑区】的【数据表格】区域查看具体数据。

![草稿绘图-表格浏览.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/GV3AG%E8%8D%89%E7%A8%BF%E7%BB%98%E5%9B%BE-%E8%A1%A8%E6%A0%BC%E6%B5%8F%E8%A7%88.drawio.png)

---

### 表结构修改

> 当前仅自定义表格支持表结构修改

表结构修改位于【表格结构编辑区】中，表结构修改提供了列的新增、删除及属性修改功能。

![草稿绘图-列编辑 的副本.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/IfboA%E8%8D%89%E7%A8%BF%E7%BB%98%E5%9B%BE-%E5%88%97%E7%BC%96%E8%BE%91%20%E7%9A%84%E5%89%AF%E6%9C%AC.drawio.png)

|序号|功能名称|功能说明|
|-|-|-|
|A|添加列|在表格末尾新增空白列|
|B|编辑列名称|修改列头显示名称|
|C|编辑列注释|编辑列描述信息以提升可读性，在编辑行值时悬停列名即可显示注释。|
|D|修改列数据类型|设置列的数据类型（如布尔型/字节型/整型等）|
|E|修改数据容器类型|配置列的数据结构形式（单值/数组/集合/映射等）|
|F|删除列|永久移除该列及其数据|

---

### 行数据编辑

> 当前支持行数据编辑的表格类型包括：自定义表格、功能表格和模板功能表格

首先双击打开目标表格，然后在【表格结构编辑区】中，通过【行编辑器】选择行（或直接在【数据表格】单击选择目标行）并编辑行数据，完成行数据编辑后点击“保存”。

![草稿绘图-表格编辑.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/8bOCB%E8%8D%89%E7%A8%BF%E7%BB%98%E5%9B%BE-%E8%A1%A8%E6%A0%BC%E7%BC%96%E8%BE%91.drawio.png)

行数据编辑提供了行的新增、删除、移动及编辑功能。

![草稿绘图-行编辑.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/xgU0p%E8%8D%89%E7%A8%BF%E7%BB%98%E5%9B%BE-%E8%A1%8C%E7%BC%96%E8%BE%91.drawio.png)

|序号|功能名称|功能说明|
|-|-|-|
|A|添加行|在表格末尾添加新行（所有数据自动填充默认值）|
|B|删除行|永久删除选定行及其所有数据|
|C|选择行|选定待操作行（支持在【数据表格】区域直接单击选择）|
|D|编辑行值|修改选定行的数据，左边显示字段名称，右侧提供编辑输入框|
|E|修改行名称|编辑行首标识名称（推荐以数字按顺序命名）|
|F|移动行|	提供行位置调整功能：上移/下移/置顶/置底|

### 表格删除

>删除操作不可逆，进行表格删除时请仔细确认核对信息。

首先在【表格浏览区】的【Asset Tree】单击选择目标表格（支持shift+单击进行多选），然后点击工具栏的“删除”图标，在弹出的核对窗口中再次点击“删除”后完成删除。

### 设置根目录

>设置根目录是为表格的导入导出确定路径。

点击工具栏的“设置根目录”图标，然后在弹出的文件夹窗口中，根据自身需求选择根目录，点击“选择文件夹”后完成根目录设置。

![草稿绘图-设置根目录.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/kvgIo%E8%8D%89%E7%A8%BF%E7%BB%98%E5%9B%BE-%E8%AE%BE%E7%BD%AE%E6%A0%B9%E7%9B%AE%E5%BD%95.drawio.png)

### 表格导入与导出

>进行导入/导出操作前需先通过【设置根目录】指定路径。

进行表格导出时，首先在【表格浏览区】的【Asset Tree】单击选择目标表格或文件夹（支持Ctrl+单击进行多选），然后点击工具栏的“导出”图标，导出成功后会弹出提示窗口，对应表格会以CSV的格式导出在根目录下。

![导出.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/6FP63%E5%AF%BC%E5%87%BA.gif)

导入的步骤与导出类似，在【表格浏览区】的【Asset Tree】单击选择目标表格或文件夹（支持Ctrl+单击进行多选），然后点击工具栏的“导入”图标，根目录中对应表格的数据将会覆盖编辑器目标表格的数据，导入成功后会在编辑器中间和右下角弹出提示。

![导入.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/irpZK%E5%AF%BC%E5%85%A5.gif)

## 读取表格数据

在玩法进程中，可以调用 [GetTableData](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=GetTableData) 动态读取表格，调用 [GetTableCount](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=GetTableCount) 读取表格行数，调用 [GetTableDataByRowName](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=GetTableDataByRowName) 读取指定表格的指定行内容。

为按钮添加对应功能逻辑，部分代码如下:

``` lua
--UI控件的初始化函数
function MyUI:Construct()
	--给按键绑定点击事件回调
	self.ButtonViewAllData.OnClicked:Add(self.ButtonViewAllDataOnClicked, self);
	self.ButtonViewNewRowData.OnClicked:Add(self.ButtonViewNewRowDataOnClicked, self);
	--使用GetTableData获取表格内容
  self.MyTable = UGCGameSystem.GetTableData("Data/Table/Customized/myNewTable");
	--输出表格获取状况
  if self.MyTable ~= nil then
  	ugcprint("成功读取表格");
		--使用GetTableCount获取表格总行数
    local TotalCount = UGCGameSystem.GetTableCount("Data/Table/Customized/myNewTable");
    ugcprint(string.format("表格一共 %d 行数据", TotalCount));
  else
    ugcprint("未读取表格");
  end
end

function MyUI:ButtonViewAllDataOnClicked()
	ugcprint("查看所有数据");
	--遍历并输出表格的所有内容
  for k, v in pairs(self.MyTable) do
		ugcprint(string.format("RowName:%s|Data1:%s|Data2:%s|Data3:%s", tostring(k), tostring(v.Data1), tostring(v.Data2), tostring(v.Data3)));
	end
end

function MyUI:ButtonViewNewRowDataOnClicked()
	ugcprint("查看NewRow一行的数据");
	--方法1：使用GetTableDataByRowName获取对应表格的对应一行数据
	local NewRowData1 = UGCGameSystem.GetTableDataByRowName("Data/Table/Customized/myNewTable","NewRow");
	ugcprint(string.format("Data1:%s|Data2:%s|Data3:%s", tostring(NewRowData1.Data1), tostring(NewRowData1.Data2), tostring(NewRowData1.Data3)));
	--方法2：获取表格后直接利用索引获取对应一行数据
	local NewRowData2 = self.MyTable["NewRow"];
	ugcprint(string.format("Data1:%s|Data2:%s|Data3:%s", tostring(NewRowData2.Data1), tostring(NewRowData2.Data2), tostring(NewRowData2.Data3)));
end
```

## 注意事项

表格导入时必须满足以下条件，否则无法正常导入：
- 进行表格导入前，必须确保编辑器内与项目根目录中存在同名的表格文件，且双方均不可删除
- 列结构相同（列的数量/对应位置的名称/对应位置的数据类型）
- 根目录文件类型需要为.csv
- 根目录文件编码格式需要为utf-8，避免出现中文乱码
- 导出操作将自动包含其父文件夹结构（如：Asset\Data\Table）以维持此路径。导入文件时必须确保根目录的父文件夹结构与工程目录的完全一致。若文件被移动或文件夹结构改变导致路径不符，则导入失败
- 自定义结构体数据仅限表格管理器内修改，不支持导出在根目录修改
