# 场景破坏物

玩法中可能涉及可被破坏的场景物件对象，例如墙体、炮塔、箱子等，为此实体编辑器提供了基于静态网格体的实体配置方案。

<br>

## 破坏物模板

实体编辑器预设了三种破坏物品模板，并且对于模板对象逻辑都提供了可参考的Lua脚本，开发者可以在【实体编辑器】中的工具栏找到编辑对应实体的【Lua】按钮打开Lua代码进行修改。若开发者有自身的需求，可以参考这三个模板的lua代码自行实现。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/PHI8Eimage.png)

**礼盒**

摆放在场景中的一个对象，受到攻击之后若血量在50%之上，则会小幅震荡；若小于50%的血量了，则会大幅震荡；血量归零之后会销毁并掉落一些道具。震荡的逻辑在Lua脚本中实现。

## 创建破坏物

在编辑器上方工具栏中打开【实体编辑器】，在【场景破坏物】页签下可见 ```普通破坏物``` ，点击打开下拉栏可以看到预置的静态模型破坏物模板。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/avx5Himage.png)

选择其中任意一个模板创建破坏物蓝图并打开该蓝图。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/hQqqOimage.png)

此类对象使用静态网格体组件来作为实体的模型配置，没有移动逻辑与动画。此类对象底层仍然是怪物，但是在编辑器面板中屏蔽了一些无用的组件，只保留了掉落、属性、技能、伤害组件，所以这个破坏物蓝图比较适合一些无复杂动画表现的静态破坏物。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/o6fMyimage.png)

<br>

## 替换破坏物模型

开发者若有替换破坏物模型的需求，可以修改 ```StaticMesh``` 上的静态网格体资产。其他组件的配置和怪物配置一样。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/nqD6Iimage.png)

> 不建议修改 ```Collision``` 组件中的碰撞设置，若新创建一个网格体组件，也需要保证与默认的碰撞设置一致

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/sSY3zimage.png)

## 案例演示

开发者可以在破坏物蓝图中扩展自定义的功能逻辑，以实现一个陷阱箱为例，当玩家攻击宝箱的时候，宝箱对周围发出震荡并造成伤害。

1. 在【实体编辑器】中场景破坏物一栏下的普通破坏物中选择 ``礼盒`` 模板创建蓝图。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/mpbzXimage.png)

2. 场景破坏物模板预先添加了技能组件 ```PersistClientStateComponent```，找到该组件并为该组件添加初始化技能列表，开发者可以根据需求前置 [创建技能](https://developer.gp.qq.com/wikieditor/#/catalog/20091?autoJump=%E6%8A%80%E8%83%BD%E5%88%9B%E5%BB%BA%E6%B5%81%E7%A8%8B) 。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Rmw36image.png)

3. 在【实体编辑器】上方工具栏中找到【Lua】按钮，打开对应的蓝图脚本文件，对脚本逻辑进行修改。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/MaZp1image.png)

通过 [```AddSkillByClass```](https://developer.gp.qq.com/api/#/searchContent/UGCPersistEffectSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E6%8A%80%E8%83%BD%E7%B3%BB%E7%BB%9F%2FUGCPersistEffectSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCPersistEffectSystem&autoJump=AddSkillByClass) 的返回值获取技能实例对象，并创建全局变量进行存储。

``` Lua
function box:ReceiveBeginPlay()
    local SkillPath = UGCMapInfoLib.GetRootLongPackagePath() .. 'Asset/Blueprint/Prefabs/Skills/jianta.jianta_C'
    local SkillClass = UGCObjectUtility.LoadClass(SkillPath)
    -- 全局变量存储技能实例对象
    self.SkillInstance = UGCPersistEffectSystem.AddSkillByClass(self, SkillClass)
end
```

通过上面获取到的技能实例对象，可以在 [受击相关处理函数](https://developer.gp.qq.com/wikieditor/#/catalog/20252?autoJump=%E4%BC%A4%E5%AE%B3%E8%A7%A6%E5%8F%91%E6%97%B6%E5%BA%8F) 中调用 [```ActivateSkill```](https://developer.gp.qq.com/api/#/searchContent/UPersistEffectSkill?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUPersistEffectSkill.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UPersistEffectSkill&autoJump=ActivateSkill) 方法释放技能。

``` Lua
---受击后置事件
---生效范围：服务器
---@param Damage float 伤害值
---@param EventInstigator AController 伤害来源的Controller
---@param DamageCauser AActor 伤害来源
---@param DamageContext FGameMagnitudeContext  伤害上下文
function box:PostTakeDamageEvent(Damage, EventInstigator, DamageCauser, DamageContext)
		self.SkillInstance:ActivateSkill()
end
```

运行游戏，对场景中放置的盒子对象进行攻击，盒子将自动触发震荡效果并对玩家目标造成伤害。
