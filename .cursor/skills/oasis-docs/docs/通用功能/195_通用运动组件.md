# 通用运动组件

通用运动组件是一个通过给Actor添加UGCMotion组件，为其ACtor赋予移动数据，然后让Actor动起来的功能。

<br>

## 运动器概述

![Snipaste_2025-06-23_18-01-27.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/DjrsXSnipaste_2025-06-23_18-01-27.png)

## 添加运动组件

先勾选Actor的属性【复制】

![复制.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/SyU8o%E5%A4%8D%E5%88%B6.png)

其次将移动性设置为【可移动】

![移动.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/9dQGl%E7%A7%BB%E5%8A%A8.png)

最后点击【组件】模块的【+】搜索UGCMotion，选择【运动器组件】添加即可

![添加.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/4UllS%E6%B7%BB%E5%8A%A0.png)

## 配置运动数据

### 线性运动

![1.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/ohynJ1.png)

|属性名|属性说明|
|-|-|
|运动模式：线性运动|线性运动：物体沿着指定的路线进行直线移动，初始速度单位都是cm/s|
|是否自动开启|勾选后进入游戏自动开始运动|
|是否往复运动|勾选后进行循环运动|
|循环周期时间|开启往复运动后出现，每循环一次所需要的事件<br>循环周期时间是一个来回需要的时间|
|运动总时间|指该组件运动数据，会在第x秒后停止运动|
|坐标系类型|局部坐标：以Actor中心点为[0,0,0]<br>世界坐标：以世界中心点为[0,0,0]|
|初始速度|对应坐标系类型。填写终点坐标|
|支持运动预览|点击【运动预览】即可查看该Actor的运动轨迹<br>运动配置支持运动预览开关，关闭则在预览时不会进行运动模拟|

## 功能使用

### 放置场景使用

将上述步骤创建的Actor摆放到场景中，并检查Actor的移动性是否为【可移动】如果不是需要修改成【可移动】

![去水印.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/KdwOe%E5%8E%BB%E6%B0%B4%E5%8D%B0.png)

放置后，在【配置方式】里面，将【是否自动开启】勾选上即可

![自动开启.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/qf4vy%E8%87%AA%E5%8A%A8%E5%BC%80%E5%90%AF.png)

### 动态启用

#### 开始运动特定运行器

```lua

function UGCPlayerController:ServerRPC_TestUGCMotion(TestType, MotionIndex)
    --调用测试函数ServerRPC_TestUGCMotion，传入运动器当前状态TestType和运动器id MotionIndex
    
    local UGCMotionTestActorClassPath = UGCGameSystem.GetUGCResourcesFullPath('Asset/Blueprint/Cube_Blueprint.Cube_Blueprint_C')
		--获取添加了运动器组件的Cube_Blueprint组件的完整路径，赋值给UGCMotionTestActorClassPath
		
    local UGCMotionTestActorClass = UGCObjectUtility.LoadClass(UGCMotionTestActorClassPath)
    --创建一个运动对象的类 UGCMotionTestClass，获取赋值给UGCMotionTestActorClassPath的完整路径

		---@type Cube_Blueprint_C[]
    local UGCMotionTestActorList = UGCActorComponentUtility.GetAllActorsOfClass(self, UGCMotionTestActorClass)
    --获取场景中所有根据UGCMotionTestClass类创建出来的对象
    
    for _, MotionActor in pairs(UGCMotionTestActorList) do
        --遍历场景中所有的运动对象
        local UGCMotionComp = MotionActor.UGCMotion
        --获取每个运动对象的运动组件实例
				UGCMotionComp:StartMotion(MotionIndex)
				--调用运动组件实例的方法
    end
end
```

---

## 功能调试

配置好组件的运动轨迹之后，可以在【世界大纲视图】找到对应的Actor，然后在【细节】窗口选中UGCMotion组件，点击UGCMotion Component里面的【运动预览】按钮即可在视窗中直接查看actor的运动轨迹

![预览.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/3h0ac%E9%A2%84%E8%A7%88.png)

## 注意事项
运动器启动之后会tick修改位置，如果需要在运动器启动之后动态修改运动器坐标，需要先通过[SetComponentTickEnabled](https://developer.gp.qq.com/api/#/searchContent/UActorComponent?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUActorComponent.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UActorComponent&autoJump=SetComponentTickEnabled)接口将运动器禁用，然后再去修改，不然运动器位置会因为tick而一直被固定不可修改

<br>
