# 实体类型

实体类型是编辑器中对Actor进行高级逻辑分类的重要机制，为玩法逻辑提供了一个统一、灵活且语义化的筛选抽象层，开发者可利用实体类型标签，在玩法逻辑中对目标进行筛选，目前已支持自定义实体类型。

![0119.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/TZqHv0119.drawio.png)

<br>

## 预设实体类型

系统已经预设多种实体类型，目前已有的实体类型如下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/tsQsRimage.png)

|实体类型|说明|
|:-:|-|
|EntityType.Pawn|所有继承自APawn的类|
|EntityType.Monster|由实体编辑器创建的怪物|
|EntityType.Player|UGC角色实体|
|EntityType.Projectile|由技能编辑器创建的抛体|
|EntityType.SkillArea|由技能编辑器创建的法术场实体|
|EntityType.Pickup|拾取物|
|EntityType.Vehicle|载具|

<br>

## 添加自定义实体类型

首先在工程Asset任意目录下创建一个自定义实体类型表，资源类型选择数据资源。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Qpu6cimage.png)

资源类别选择 ``CommonEntityTypeConfigDataAsset`` 。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/L8JaUimage.png)

打开自定义实体类型表，配置自定义的实体类型，相关配置项以及说明如下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/PW0jkimage.png)

|属性名称|说明|
|:-:|:-:|
|Entity Type Name|新自定义的实体类型对应的 [Gameplay标签](https://developer.gp.qq.com/wikieditor/#/catalog/20102) ，需要定义在 ``EntityType`` 层级下|
|Actor Classes|该实体类型对应的Actor基类，所有继承自该Actor基类的子类都属于该实体类型|
|Collision Channels|该实体类型对应的碰撞通道，Actor必须属于该通道，才会属于该实体类型|

打开【玩法通用设置】，在属性【额外实体类型配置】中选择已经配置的自定义实体类型表。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/YgdnMimage.png)

<br>

## 应用实体类型

**目标过滤**

在项目中配置好的实体类型，可用于抛体 [实体过滤器](https://developer.gp.qq.com/wikieditor/#/catalog/20218?autoJump=%E5%AE%9E%E4%BD%93%E7%B1%BB%E5%9E%8B%E8%BF%87%E6%BB%A4%E5%99%A8)、技能 [单点选取器](https://developer.gp.qq.com/wikieditor/#/catalog/20186?autoJump=%E6%8A%80%E8%83%BDTask-%E9%80%89%E6%8B%A9%E7%82%B9) 中对重叠对象的检测等场景。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/yMqmvimage.png)
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/57nBAimage.png)

**动态检测实体**

脚本中调用 [UGCEntityTypeSystem](https://developer.gp.qq.com/api/#/searchContent/UGCEntityTypeSystem?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUGCEntityTypeSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCEntityTypeSystem) 的函数，可以对实体类型进行检测、判断，以下为点击按钮检测附近特定实体类型的示例，界面设计如下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/1yPJPimage.png)

为按钮添加对应功能逻辑，点击按键的执行代码如下:

``` lua
--Main.lua部分代码

function MainUI:ButtonGetBossOnClicked()
		--获取玩家当前位置
    local PlayerPawn = UGCGameSystem.GetPlayerPawn();
    local Location = PlayerPawn:K2_GetActorLocation();
		--检测玩家周围半径500cm范围内实体类型为"EntityType.Boss"的Actor数量
    local Actors = UGCEntityTypeSystem.OverlapSphereByEntityType(self, "EntityType.Boss", Location, 500);
    if Actors and #Actors > 0 then
        ugcprint("附近BOSS的数量:" .. tostring(#Actors));
    else
        ugcprint("附近没有BOSS");
    end
end
```

调试效果如下，点击按钮后，调试日志打印出附近BOSS的数量。

## 注意事项

自定义实体类型与预设实体类型为追加关系，为Actor配置自定义实体类型后，该Actor原有的预设实体类型依然生效。
