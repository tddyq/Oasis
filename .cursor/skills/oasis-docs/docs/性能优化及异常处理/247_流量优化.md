#  流量优化

## 使用Unreliable版RPC

---

- 客户端表现类RPC需要全部替换成Unreliable的版本以节省流量
- Unreliable不保证会收到，可能会丢包
- 音效，动作，特效等客户端表现类的推荐使用

```
UnrealNetwork.CallUnrealRPC_Multicast_Unreliable
UnrealNetwork.CallUnrealRPC_Unreliable
```

## 高频率RPC采用短名称

---

- RPC的名称也会占用一定的流量
- 如果是高频RPC和广播用的RPC，尽量缩短名称，可以省流量

### 对于广播动作，音效，特效这类的RPC，参数传id代替字符串路径

- 参数不要传整个资源路径，而是应该建立一个id索引表，只传id，客户端收到id后，再查询真实的资源路径
