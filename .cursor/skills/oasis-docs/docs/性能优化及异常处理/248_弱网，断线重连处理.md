#  弱网，断线重连处理

## 需要保存状态的使用属性同步，而非RPC

---

- rpc使用需要考虑网络断线重连，玩家大退客户端重连的情况
- 玩家如果错过此rpc是否还能正常
- 如果玩家错过本次rpc会表现异常，建议统一使用属性同步来实现

```lua
--在此函数中返回的变量为属性同步变量
function MyMonster:GetReplicatedProperties()
    return "Health"
end

--OnRep_变量名为客户端受到变量同步后，响应的函数，此函数被调用时，变量已是同步后的值
function MyMonster:OnRep_Health(PrevHealth)
    print("MyMonster:OnRep_Health")
end
```

## 断线和重连相关委托和参数

---

### 委托相关

委托|解释|触发环境
--|--|--
PlayerController.PlayerControllerReconnectedDelegate|断线重连委托（杀进程重连）|战斗服务器和客户端触发
PlayerController.PlayerControllerRecoveredDelegate|断线重连委托（静默重连）|战斗服务器触发
PlayerController.ClientPlayerControllerRecoveredDelegate|断线重连委托（静默重连）|客户端触发
PlayerController.PlayerControllerLostDelegate|断线委托| 战斗服务器触发

### 参数相关

- PlayerState.isReconnected 是否重连
- PlayerState.isLostConnection 是否断开连接

<span style="color: #ff0000"> - 如果想要获取正确的重连委托，需要同时绑定</span>

- `PlayerControllerReconnectedDelegate`
- `PlayerControllerRecoveredDelegate`
