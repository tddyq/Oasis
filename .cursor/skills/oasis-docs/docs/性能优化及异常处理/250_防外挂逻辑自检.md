# 防外挂逻辑自检

游戏外挂通过非授权手段篡改客户端/服务端交互逻辑或数据，破坏游戏平衡性。主要攻击内容包括**内存篡改、网络协议破解、自动化脚本、资源篡改**四大类，核心原理是绕过游戏的正常逻辑校验机制。

<br>

## 内存修改类外挂

**技术原理**

通过扫描/修改游戏进程内存数据，直接篡改角色属性或游戏状态。

**常见手段**

- **数值篡改**（比如金币、血量）
使用工具（Cheat Engine）定位内存地址并动态修改数据。
- **DLL注入**
注入动态链接库劫持游戏函数调用（比如修改伤害计算函数）
- **代码注入（Code Injection）**
通过Hook技术修改游戏代码执行流程（比如跳过技能冷却检测）

**检测难点**

内存加密、指针混淆、反调试机制可增加定位难度。

<br>

## 网络协议破解类

**技术原理**

截获/篡改客户端与服务端通信数据包

**常见手段**

- **封包重放（Replay Attack）**
截获有效操作封包（比如领取奖励）重复发送。
- **协议逆向（Reverse Engineering）**
解密通信协议伪造虚假请求（比如模拟装备强化成功）
- **中间人攻击（MitM）**
篡改传输中的关键数据（比如战斗结果上报）

**检测难点**

需要强化协议加密（TLS）、增加时序校验与唯一操作令牌。

<br>

## 自动化脚本类

**技术原理**

模拟玩家操作实现挂机或高精度操作。

**常见手段**

- **按键模拟（Macro）：**
固定流程的键盘鼠标操作（比如自动打怪脚本）
- **图像识别（Image Recognition）：**
通过屏幕像素分析自动定位目标（比如自动拾取物品）
- **内存行为分析：**
直接读取游戏内存数据驱动脚本（比如自动躲避技能）

**检测难点**

需要结合操作随机性检测、行为模式分析（聚类算法）及图像识别防御。

<br>

## 资源篡改类

**技术原理**

修改本地游戏资源文件获取非法优势

**常见手段**

- **模型/贴图修改：**
透明化障碍物或高亮显示敌人（比如"透视外挂"）
- **配置文件篡改：**
修改技能范围/冷却时间等配置参数
- **反编译/重打包：**
篡改游戏逻辑代码后重新签名（常见于单机游戏）

**检测难点**

需要实现资源文件完整性校验（CRC/MD5）与运行时资源监控。

目前来说，开发者不需要关注1、3、4类型的外挂对抗，主要需要关注第二类协议破解类的对抗，而在和平绿洲里面，常见的漏洞点集中在上行RPC这个地方，从常规外挂手段来看
1. 抓包重发，即客户端会不断收到RPC请求，因为依赖RPC次数的关键逻辑，是不可靠的，一定需要依赖DS端校验RPC的执行合理性。
2. 参数篡改
例如：Server_RPC_AddItem( ItemID,ItemNum)  其中ID和Num都可以被外挂拦包然后篡改，因此DS不能信任这个数据，如果有客户端的购买逻辑，建议把RPC改成
Server_RPC_BuyItemA(WantNum),以玩家行为而非结果去上报DS，然后DS根据持有的货币数量和条件等真正的执行购买和添加物品的逻辑
3. 结果篡改
例如：在客户端做了一个随机强化系统，其成功与失败全在客户端随机，然后直接报结果给DS，同2，一样可以无限报成功结果。
