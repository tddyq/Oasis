# 输入映射

和平精英作为一款具备移动端和 [PC模拟器](https://developer.gp.qq.com/wikieditor/#/catalog/20211) 跨端特性的游戏，在不同平台上处理输入流程的方式不同，为了提高可移植性引入了类似 [增强输入](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/enhanced-input-in-unreal-engine) 的输入映射方案，允许开发者扩展新的输入流程，也支持覆盖修改和平默认的输入交互方式。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Cy9TDimage.png)

<br>

## 输入概述

玩家输入是指将玩家对硬件操作的指令传递给指定对象，并经由输入映射和输入组件将指令转换为具体的游戏事件和逻辑的处理流程，例如按键触发技能的释放、点击按钮弹出界面等都属于输入行为。

依据输入行为和映射方式的不同可分为操作映射和轴映射两种：

- 操作映射：离散的开关型输入，对应按下（或释放）单个按键、鼠标点击触发某个游戏行为，例如跳跃、趴下等
- 轴映射：有连续范围且持续轮询输入值的输入方式，通常用于表现平稳过渡的游戏行为，例如玩家移动

输入映射即是将操作映射或者轴映射定义为具体的输入动作，经过修饰器处理原始输入值，并通过触发器激活输入动作的输入配置系统，多组输入映射可描述同一个抽象的输入事件，例如跳跃事件在移动端与PC模拟器上分别对应两组输入映射配置，编辑器通过输入映射表来管理所有的输入事件。

## 输入映射表

编辑器提供了一套和平角色的默认输入映射表 ``DA_UGCInputMapping``，该映射表以数据资产蓝图的形式配置于 ``UGCPlayerController`` 蓝图上。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/WbFX5image.png)

开发者如果希望添加自定义的输入映射配置，可以创建新的输入映射数据资产蓝图来覆盖此默认配置。

**创建输入映射资产蓝图**

于工程内容浏览器中，右键 ``各种各样 -> 数据资源``，搜索“Input Mapping Preset”创建数据资产蓝图，并将该数据资产蓝图替换 ``UGCPlayerController`` 蓝图上的 ``Input Mapping Preset`` 属性引用。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/b0dzZimage.png)
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/aZX7zimage.png)

## 配置输入映射规则

输入映射表由一组多层级结构的输入事件组成，每个输入事件由单个或一组输入动作、输入修饰器和输入触发器定义，各个输入动作也可以包含独立的输入触发器和输入修饰器，如此动作行为、输入值和触发条件共同决定了玩家的输入效果。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/UDTmIimage.png)

### 输入事件

输入事件是玩家执行动作的行为名称，是一个抽象的行为描述，输入系统中通过 [Gameplay标签](https://developer.gp.qq.com/wikieditor/#/catalog/20102) 的形式表达，是单一输入行为的唯一标识符，例如 ``Input.Action.Jump`` 代表跳跃行为。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/ce25simage.png)

输入事件需要被显式监听，当该事件的触发处于不同状态时，通过广播通知的回调函数处理相应的业务逻辑，支持已开始、进行中、已触发、已完成和已取消五种触发状态。

|触发状态|状态说明|枚举值|
|:-:|-|-|
|已开始|输入事件发生时触发一次，例如按键抬起，当按下按键那一刻将触发一次该状态|ETriggerEvent.Started|
|进行中|输入触发器持续处理过程中触发，例如按下按钮，在达到指定持续时间之前，"按住"动作处于进行中状态|ETriggerEvent.Ongoing|
|已触发|输入值满足了触发器的条件时触发，例如按下并松开，触发器会在松开按键时触发该状态|ETriggerEvent.Triggered|
|已完成|结束该输入事件时触发|ETriggerEvent.Completed|
|已取消|输入事件开始后，未满足触发器条件前取消时触发，例如按住1秒，在达到1秒前就已经松开了按键|ETriggerEvent.Canceled|

### 输入动作

输入动作是操作映射和轴映射的别名，用于定义具体行为的触发方式和输入值计算形式，在概念上属于输入事件的具象化表示，操作映射和轴映射分别对应InputAction和InputAxis两种类型的动作：

- InputAction：按下和抬起表示的按键型输入，输入值以1代表按下，0代表抬起
- InputAxis：通常为摇杆形式的XY二元轴输入，各轴的值范围为0~1，例如不移动时为0，摇杆推到移动最大速度时为1

### 输入修饰器

输入修饰器是一种输入值的预处理器，能够修改玩家输入行为的原始输入值，然后再将其发送给输入触发器，例如防止摇杆被误触产生输入时，就需要将较小的输入值重映射为0来处理。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/g6U7nimage.png)

目前仅支持 ``死区`` 修饰器，作用是将输入值限定在指定的数值范围内。

- Lower Threshold：输入值范围的最小值
- Upper Threshold：输入值范围的最大值
- Type：重映射的规则类型
	- Unscaled：小于 ``Lower Threshold`` 时为0，大于 ``Upper Threshold`` 时为 ``Upper Threshold`` 的值，否则为原始输入值
	- Scaled：小于 ``Lower Threshold`` 时为0，大于 ``Upper Threshold`` 时为 ``Upper Threshold`` 的值，否则执行归一化处理，例如原始值为0.6归一化后的结果为 ``(0.6 - Lower Threshold) / (Upper Threshold - Lower Threshold)``

如果配置了多组修饰器，则根据 ``Accumulation Behavior`` 指定的计算类型决定最终的输入值。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/aDIoDimage.png)

- Take Highest Absolute Value：取最大值作为最终结果
- Cumulative：将多组输入值相加的结果作为最终输入值

---

### 输入触发器

输入值在经过输入修饰器处理后，是否真正激活该输入动作由输入触发器决定，触发器检查输入值并验证是否满足激活条件，目前支持 ``Pressed`` 和 ``Released`` 两种类型。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/gbgllimage.png)

- Pressed：当输入值大于 ``Actuation Threshold`` 时的当帧触发
- Released：当输入值大于 ``Actuation Threshold`` 后，重新小于 ``Actuation Threshold`` 时的当帧触发，例如按键抬起时

当配置了多个触发器时，只需满足任何一个触发器条件即会激活该输入动作。

---

### 输入拦截

如果希望拦截和平预设输入的功能并修改成其他效果，需要启用 ``Consume Input`` 属性，该属性控制是否只消耗当前输入，避免被其他低优先级的Actor捕获触发，即排他型响应。

例如，将和平默认空格键触发的跳跃改为冲刺技能，通过一个输入事件关联 ``Jump`` 输入动作，监听该输入事件并在回调中释放冲刺技能，当输入事件触发时只执行技能的释放，而不会触发跳跃行为。

## 应用输入映射

### 操作映射输入

操作映射适用于按钮点击触发相应效果的输入场景，这种类型的输入需要先定义一个输入事件，监听事件并实现事件回调的逻辑，当触发该输入事件时即执行相应的逻辑表现。

以点击触发冲刺技能的InputAction输入为例，通过 [GameplayTag标签](https://developer.gp.qq.com/wikieditor/#/catalog/20102) 系统添加一个名为“Input.Action.Test”的Tag，该Tag作为输入事件的名称。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/JcO5Iimage.png)

脚本中通过 [``BindInputMapping``](https://developer.gp.qq.com/api/#/searchContent/UGCInputSystem?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUGCInputSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCInputSystem&autoJump=BindInputMapping) 函数监听该输入事件的 ``已触发`` 状态，并且在回调函数中通过RPC从服务端执行冲刺技能的释放。

```lua
-- 在PlayerPawn中绑定输入事件并实现回调逻辑
function UGCPlayerPawn:ReceiveBeginPlay()
    UGCPlayerPawn.SuperClass.ReceiveBeginPlay(self)

    if not UGCGameSystem.IsServer() then
        local ActionTag = UGCGameplayTagSystem.RequestGameplayTag("Input.Action.Test")
        UGCInputSystem.BindInputMapping(self, ActionTag, ETriggerEvent.Triggered, self.OnTriggerJumpAction)
    end
end

function UGCPlayerPawn:OnTriggerJumpAction(InputValue, ElapsedTime, TriggeredTime, InputTag)
    local selfPC = UGCGameSystem.GetPlayerControllerByPlayerPawn(self)
    if selfPC then
        UnrealNetwork.CallUnrealRPC(selfPC, selfPC, "ServerRPC_TriggerSprintSkill");
    end
end

-- 服务端触发冲刺技能的释放
function UGCPlayerController:ServerRPC_TriggerSprintSkill()
		local PlayerPawn = UGCGameSystem.GetPlayerPawnByPlayerController(self)
    local TriggerSkill = UGCPersistEffectSystem.AddSkillByClass(PlayerPawn, UGCGameSystem.GetUGCResourcesFullPath('Asset/Blueprint/Prefabs/Skills/Sprint_test.Sprint_test_C'))
    TriggerSkill:ActivateSkill()
end
```

通过按钮的点击事件经由 [``InjectInputMapping``](https://developer.gp.qq.com/api/#/searchContent/UGCInputSystem?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUGCInputSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCInputSystem&autoJump=InjectInputMapping) 触发输入事件，当点击按钮时即会触发冲刺技能。

```lua
local ActionTag = UGCGameplayTagSystem.RequestGameplayTag("Input.Action.Test")
UGCInputSystem.InjectInputMapping(self, ActionTag, 1)
```

### 轴映射输入

轴映射通常应用在摇杆的输入场景，包括角色移动、旋转等，因此使用上需要配合摇杆类UI，UI编辑器中提供的 ``技能按钮模板`` 元件具有绑定输入事件的属性，绑定后轴映射的输入值将传递给输入事件Tag，从而进入下游的业务逻辑处理。
