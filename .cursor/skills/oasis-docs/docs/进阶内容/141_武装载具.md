# 武装载具

武装载具是 [普通载具](https://developer.gp.qq.com/wikieditor/#/catalog/20024) 的升级版，不仅提供了基础的移动能力，还允许玩家在载具上进行射击战斗。这种增强型的载具能够为游戏带来更丰富的战术选择和战斗体验。本文档将详细介绍如何在编辑器中对普通载具进行武装化改造，包括武器系统的添加、配置和使用。

> 该文档为旧载具蓝图下的操作指南，目前已统一迁移至 [载具编辑器](https://developer.gp.qq.com/wikieditor/#/catalog/20245)，请使用新的方式创建武装载具

## 创建武装载具

编辑器已内置了一批武装载具蓝图可供开发者直接使用，但配置武器时无法直接修改默认蓝图，因此需要创建武装载具的子蓝图。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/tYipkimage.png)

开发者可通过 ```和平精英/载具``` 目录下查找目标载具蓝图，并在项目工程下继承创建新的载具蓝图。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/9QgRWimage.png)

创建方式：右键 ``蓝图类 -> 搜索需要创建的载具名 -> 选择目标载具``<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/nySkIimage.png)

> 武装载具的本体仍旧是载具，因此也可以基于普通载具创建子蓝图，两者武器的配置方式相同

## 载具武器配置
武装载具的武器和座位相绑定，只有对应座位的玩家才能操控座位上配置的武器，而武器的生效需要主动启用及配置挂载插槽和武器类。

### 启用武器
打开载具蓝图，在组件栏中找到 ```VehicleSeats``` 组件，查看 ```Seats``` 属性，其中0到3号为载具的座位下标序号，每组配置项下都有一个 ```EnableVehicleWeapon``` 属性，勾选上即可启用此座位的武器系统。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/HTU6Nimage.png)

以上图为例，勾选了0号位和1号位的 ```EnableVehicleWeapon``` 属性，代表0与1两个座位的武器系统被启用，玩家坐上后即可使用车载武器；而2号位与3号位的玩家无法使用武器。

![vehicle_seat.jpg](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/rbR9f11(1).jpg)

<br>

### 配置武器
武器与载具通过Socket插槽相连接，也可以理解为武器的挂载点，开发者首先需要在载具上选择一个Socket插槽，而后将Socket插槽与目标武器类相关联完成武器的配置。

#### 选择Socket插槽
进入载具蓝图，在组件栏中选择 ```Mesh``` 组件，查看属性栏中的 ```骨骼网格体``` ，双击进入。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/uKfQAimage.png)

进入网格体界面后，点击右上角骨骼选项，查看载具骨骼，在搜索框中输入 ```Socket``` 关键词，筛选出来的结果则是可使用的Socket插槽。这里我们选中其中一个插槽 ```RPGSocket``` ，在视图中可以看到这个插槽的具体位置，这个位置便是武器模型与载具模型相连接的位置。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/cyEwsimage.png)

#### 设置武器参数
选择好Socket插槽后，需要配置武器的具体参数。

进入载具蓝图，查看 ```VehicleSeats``` 组件中的 ``Seats`` 属性，涉及的参数配置如下：

- Enable Vehicle Weapon：启用此座位的武器系统
- Vehicle Weapon Config：此座位使用的武器组合，可以配置多个，但使用相同的开火键，开火时多个武器同时生效
- Vehicle Weapon Config Support：此座位的备选武器列表，可以用备选武器替换 ``Vehicle Weapon Config`` 配置的当前武器列表中的武器，具体使用参考【载具武器切换】
- Vehicle Weapon Attach Pos Offset：武器在原有挂载点基础上的相对偏移量

**配置Vehicle Weapon Config**

```Vehicle Weapon Config``` 中可以插入多条配置，每个配置为单独的武器组。每组都需要配置 ```Weapon Socket``` 和 ```Vechicle Weapon CLass```： 
 
- Weapon Socket：武器与载具相连的Socket插槽名称
- Vechicle Weapon CLass：使用的目标武器类

如图所示，此处配置了两组武器，且两个Socket插槽对应不同的武器：

0号位：使用RPGSocket，配置了RPG武器。
1号位：使用GatlinSocket，配置了加特林武器。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/vEEaNimage.png)

在该配置下，进入游戏后的主驾驶位，点击开火键，将会触发两个武器一起开火射击。

> 此处配置的武器将会作为载具默认武器，如有替换武器的诉求，需要开发者手动编写逻辑实现

---

**配置Vehicle Weapon Config Support**

如果玩法中需要替换座位上的武器，则需要在 ```Vehicle Weapon Config Support``` 中额外配置要切换的武器类型，及该武器的挂载点，配置方式与 ``Vehicle Weapon Config`` 相同。<br>

## 载具武器切换
在完成 ``Vehicle Weapon Config`` 与 ``Vehicle Weapon Config Support`` 的配置后，开发者可以通过调用以下接口进行武器的替换。

``` lua
-- 生效范围：服务端
-- @param Vehicle ASTExtraVehicleBase* @载具
-- @param SeatIndex int32 @座位ID
-- @param WeaponIndex int32 @车载武器ID
-- @param WeaponIndexSupport int32 @武器库武器ID
function UGCVehicleSeatSystem.AddVehicleWeaponFromSupportKit(Vehicle, SeatIndex, WeaponIndex, WeaponIndexSupport)
end
```

- Vehicle：需要替换武器的载具实例对象
- SeatIndex：需要替换武器的座位序号（0-3）
- WeaponIndex：被替换的武器索引，即 ```Vehicle Weapon Config``` 中配置的武器索引
- WeaponIndexSupport：进行替换的备用武器索引，即 ```Vehicle Weapon Config Support``` 中配置的武器索引

**功能演示**

为了方便演示，制作一个简单的Button按钮，点击此按钮后将会发送服务端RPC，进行载具的武器替换。

Button按钮绑定发送RPC的事件：

``` lua
function Custom_UI:OnChangeClicked()
    ugcprint("clicked~~~")

    local PlayerController = GameplayStatics.GetPlayerController(self, 0);
    UnrealNetwork.CallUnrealRPC(PlayerController, PlayerController, "ServerRPC_ChangeVehicleWeapon");
end
```

服务端RPC处理逻辑，调用 ``AddVehicleWeaponFromSupportKit`` 进行武器切换，其中第四个传参生成一个范围在0到4以内的随机数，即代表随机选择五个备选武器中的一个进行替换：
``` lua
function UGCPlayerController:ServerRPC_ChangeVehicleWeapon()
    local VehicleClass = UE.LoadClass(UGCGameSystem.GetUGCResourcesFullPath('Asset/vehicles/Test_ArmedDacia.Test_ArmedDacia_C')) -- 获取载具类
    local TargetCars = GameplayStatics.GetAllActorsOfClass(self, VehicleClass, {}); -- 根据载具类获取载具
    local PlayerController = GameplayStatics.GetPlayerController(self, 0); -- 获取玩家控制器
    for k, v in ipairs(TargetCars) do
        ugcprint("find target car~~~")
        local ChangeWeaponIndex = math.random(0, 4); -- 随机选择后备武器
        ugcprint(string.format("change weapon to [%d]", ChangeWeaponIndex))
        UGCVehicleSeatSystem.AddVehicleWeaponFromSupportKit(v, 0, 0, ChangeWeaponIndex); -- 武器替换
    end
end
```

## 载具武器动态启用
除了在蓝图配置中勾选 ```Enable Vehicle Weapon``` 选项来启用或禁用武器系统，也可以通过 ``SetPassengerVehicleWeapon`` 函数动态控制武器的可用性。

``` lua
-- 设置当前座位上的车载武器是否能使用，需要这个座位原来也配置了载具武器，且乘客正在该座位上
-- 生效范围：服务端
-- @param Vehicle ASTExtraVehicleBase* @载具
-- @param SeatIndex int32 @座位ID
-- @param bControlVehicleWeapon bool @是否能控制车载武器
function UGCVehicleSeatSystem.SetPassengerVehicleWeapon(Vehicle, SeatIndex, bControlVehicleWeapon)
end
```
