#  好友观战

好友观战是好友在大厅上选择了观战好友后进入游戏作为观战者的功能，由于该功能需要进行开发者进行针对性适配，所以默认对玩法关闭，开发者在适配完毕后可以申请开启好友观战功能。

## 接入好友观战

好友观战和局内观战共用一套机制，实际相当于创建了一个没有实体的虚拟玩家，进入游戏后自动进入观战流程。
所以比起局内观战省去了进入观战的流程，同样可以使用`UGCGameSystem.ChangeAllowOBPlayerKeys`设置可观战目标，或者`UGCGameSystem.ChangeOBPlayer`指定观战目标。

**API接口**

| 接口 | 描述 |  适用范围|
| ------ | ------ | ------ |
| PlayerController:IsPureSpectator() | 是否观战玩家（包括全局观战和好友观战） | DS和客户端 |
| PlayerController:IsFriendObserver() | 是否好友观战玩家 | DS和客户端 | 
| PlayerController:GetCurPawn() | 获取当前观战目标PlayerPawn | DS和客户端 | 
| PlayerController:OnSpectatorChange | 委托，观战目标切换后调用 | DS和客户端 |

<br>

## 注意事项

**1. 玩家登录时过滤观战玩家**

好友观战由于本质也是玩家加入，所以依然会触发PostLogin，玩家登陆事件，所以需要在该处进行过滤，通过PlayerController:IsPureSpectator()判断是否为观战玩家，将其从普通玩家中过滤出去。

**2. 游戏内部分提示和页面特殊处理**

由于没有对应的分队归属，也不是具体玩家，部分游戏提示和页面对于观战目标并不完全使用，比如结算，胜负UI等，需要通过PlayerController:IsPureSpectator()判断是否为观战玩家，进行对应的特殊化处理。

**3. 对局结束后的结算处理**

- 观战玩家和普通玩家对局结束后显示的结算信息可能会有差异，可以在结算信息广播后判断是否观战玩家显示不同的结算信息
- 观战玩家不参与玩法的数据结算，因此无需对观战玩家发送结算上报
