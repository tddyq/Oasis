# 玩家数据存档

数据存档功能允许开发者将对局中产生的玩家个人数据跨对局、持久化存储，例如玩家获得的装备物品、历史战绩等等，各玩法之间的存档数据相互隔离，因此开发者可以安全地使用存档数据。

<br>

## 存档数据结构

存档数据的类型和结构为 [LuaTable](https://www.runoob.com/lua/lua-tables.html)，table中的数据元素支持复合类型，包括数字、字符串等基础数据类型，也可以为数组、字典等复杂嵌套结构数据。

> 单玩家的存档数据容量大小不能超过256Kb

```lua
SaveData = {}

-- 基础类型元素
SaveData[1] = "Lua"
SaveData["wow"] = "abc"

-- table类型元素
local SubTableData = {}
table.insert(SubTableData,"equip")
table.insert(SubTableData,"avatar")
SaveData["attributeList"] = SubTableData
```

<br>

## 保存存档

使用 [SavePlayerArchiveData](https://developer.gp.qq.com/api/#/searchContent/UGCPlayerStateSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E8%A7%92%E8%89%B2%2FUGCPlayerStateSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCPlayerStateSystem:~:text=bool-,SavePlayerArchiveData) 接口存储玩家的存档数据，通过参数 [UID](https://developer.gp.qq.com/wikieditor/#/catalog/352) 标识数据的拥有者。

```lua   
local equipSequence = {}

local playerPawn = self:GetPlayerCharacterSafety()
local allItemData = UGCBackPackSystem.GetAllItemData(playerPawn)
for key, itemData in pairs(allItemData) do
	table.insert(equipSequence, itemData.ItemID)
end

local playerUID = UGCPawnAttrSystem.GetPlayerUID(playerPawn)
UGCPlayerStateSystem.SavePlayerArchiveData(tonumber(playerUID), equipSequence)
```

<br>

## 获取存档

调用 [GetPlayerArchiveData](https://developer.gp.qq.com/api/#/searchContent/UGCPlayerStateSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E8%A7%92%E8%89%B2%2FUGCPlayerStateSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCPlayerStateSystem:~:text=LuaTable-,GetPlayerArchiveData) 接口以获取指定玩家的存档数据。
	
```lua
local playerPawn = self:GetPlayerCharacterSafety()
	
local playerUID = UGCPawnAttrSystem.GetPlayerUID(playerPawn)
local readArchive = UGCPlayerStateSystem.GetPlayerArchiveData(tonumber(playerUID))
for i = 1, #readArchive do
	UGCBackPackSystem.AddItem(playerPawn, readArchive[i], 1)
end
```

## 存档调试

存档功能已支持编辑器内 [Debug调试](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/307) 与 [移动端测试](https://developer.gp.qq.com/wikieditor/?timeStamp=1729765192241#/catalog/372)，其中Debug调试环境下支持跨DS存储，但仅限一次调试会话内有效。

例如，接入了 [二次匹配](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/376) 功能的玩法在一次调试会话内，不同子模式（即不同DS）之间的存档数据可互通，但重启调试后存档数据无法再被读取。

> Debug调试环境下每场对局存储的数据非持久化，每次重新启动调试均会重置数据

<br>

## 注意事项

开发者在使用存档功能时需要特别注意以下事项：

- 请勿在 [发送游戏结算数据](https://developer.gp.qq.com/wikieditor/#/catalog/272) 之后调用SavePlayerArchiveData，否则会出现数据丢档的问题
- 存档数据针对玩家个人，请勿把全局玩家的数据都存储在同一个UID下
- 存档功能支持数据的读写，但不支持删除操作，如果游戏业务中确定需要清除玩家的数据，可以将存档数据置nil，历史态数据无法追溯
