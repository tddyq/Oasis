# 自定义行为统计

开发者可以制定具体的数据指标来跟踪玩家的行为模式与偏好，玩法上线后通过分析这些数据，针对性优化游戏的设计与体验，包含但不限于：
- 玩家行为：某类道具的使用次数与频率如何？
- 关卡体验：关卡难度设计是否合理？玩家游玩进度情况？
- 交互设计：有多少玩家点击或触发过某按钮或功能？

绿洲启元编辑器通过自定义行为上报的API支持了特定数据的收集，并于绿洲启元管理平台提供了 [查询统计数据](https://gpeditor.qq.com/user#operationalData/2) 的途径。

> 查询功能仅在<span style="color: red;font-weight:bold">灰度环境</span>中有效，如需在正式服上报自定义行为数据，请使用 [SendPgcGameTlog](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=SendPgcGameTlog) 接口，且正式服的统计数据需联系官方运营人员获取

<br>

## 自定义行为上报


### **SendGreyTLog**

``` lua
-- 生效范围：服务器
function UGCGameSystem.SendGreyTLog(ID, PlayerKey)
end
```

- ID：自定义行为id，含义为开发者自行定义的用户行为或事件，数据类型为整型，且范围限定在 ``[800800 - 801800]``，非整型或超出范围的id将导致上报失败
- PlayerKey：[玩家对局标识](https://developer.gp.qq.com/wikieditor/#/catalog/352)，整型或留空，无效或非整型的PlayerKey会导致上报失败

**使用规则**

1. 该接口仅在服务器端生效，客户端调用无效
2. 仅在<span style="color: red;font-weight:bold">灰度环境</span>中有效

---

### **SendPgcGameTlog**

``` lua
-- 生效范围：服务器
function UGCGameSystem.SendPgcGameTlog(Index, UID, CustomData)
end
```

- Index ：日志索引，数据类型为整型，且范围限定在 ``[800800, 801800]`` ，非整型和不合法Index都会导致上报失败
- UID：玩家UID，整型或留空，若非整型或UID无效/错误会导致上报失败
- CustomData：自定义数据，类型为表，超过512字节会进行自动截断，按照表格里的键值对进行截断

**使用规则**

1. 该接口仅在服务器端生效，客户端调用无效
2. 仅在<span style="color: red;font-weight:bold">正式服</span>中有效

<br>

## 数据统计查询

玩法中加入了行为上报后，可以在绿洲启元管理平台 ``项目数据 -> 运营数据页签 -> 运营数据(体验)`` 页面中查询统计数据。
> 行为上报非实时生成统计数据，需要玩法在抢先体验专区产生完整一天的有效数据后，才能在管理平台查询到，另外正式服的统计数据需联系官方运营人员获取

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/CK24jimage.png)

目前支持按日期时间段、玩法名、行为ID的查询条件，也支持以Excel的格式导出相关数据：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/EShqHimage.png)
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/9CHFdimage.png)

**字段说明**
- 上报次数：满足对应行为ID的总上报次数
- 上报人数：满足对应行为ID的去重玩家人数（同一玩家多次触发只算一次）
- 总参与人次：玩法在抢先体验专区的总参与人次
- 总参与人数：玩法在抢先体验专区的去重参与人数（同一玩家多次参与只算一次）

> 由于上报次数为触发行为ID的累积值，当与去重值进行百分比计算时，最后的结果可能大于100%

## 案例演示

抽奖是游戏中较为常见的一类功能，玩家通过抽奖行为获得预设的相对应的奖励，开发者要验证抽奖概率的准确性就依赖有效的玩家数据作为基准进行分析。以跟踪和分析玩家在游戏中的抽奖行为为例，做一个简单的演示。

如下所示，设计了一个具有三种奖品的奖池，每当一个奖品被抽中时，就将该事件进行上报。

|抽奖奖品|中奖概率|中奖行为ID|
|-|-|-|
|AKM|40%|800800|
|M416|30%|800801|
|AUG|30%|800803|

---

**抽奖行为上报**

``` lua
function Lottery:LotteryHandle(PlayerKey)
    -- 判断PlayerKey以及PlayerKey的有效性
    if not PlayerKey or not UGCGameSystem.GetPlayerPawnByPlayerKey(PlayerKey) then return end

    -- 自定义一个奖品列表
    local rewardInfo = {
        result001 = {
            resultId = "AKM",
            eventId = 800801,
            rate = 40
        },
        result002 = {
            resultId = "M416",
            eventId = 800802,
            rate = 30
        },
        result003 = {
            resultId = "AUG",
            eventId = 800803,
            rate = 30
        },
    }

    -- 预创建抽奖结果，随机数
    local result = rewardInfo.result001
    local rate = math.random(1,100)

    local addNum = 0
    for k,v in pairs(rewardInfo) do
        if rate <= addNum + v.rate then
            -- 真实抽奖结果赋值
            result = v
            break
                                else
                                                addNum = addNum + v.rate
        end
    end

    -- 上报抽奖结果事件
    UGCGameSystem.SendGreyTLog(result.eventId, PlayerKey)

end
```

在这段代码逻辑中，我们设计了一个简单的抽奖功能，在处理抽奖结果中上报了该行为事件。

如上所示，当抽奖结果result返回后，使用 ``UGCGameSystem.SendGreyTLog`` 将结果进行上报，其中 ``eventId`` 对应的是每次抽奖结果的事件，一共有三个抽奖奖品，对应的事件ID分别为：800800，800801，800803。

> **注意：**
行为上报只作为收集数据的方式，不应当对游戏流程产生任何的阻塞，并破坏玩家的游戏体验。

---

**数据分析**

上报的数据将会自动统计到绿洲启元管理平台，我们可以查询数据并作进一步的分析。

登录 [管理平台](https://gpeditor.qq.com/user#operationalData/2)，选择 ``运营数据-自定义埋点数据`` tab页，选择对应的游戏项目，并输入 ``800800、800801与800803``，查询这三个ID所对应的奖品在 ``最近30天`` 的中奖情况：

![Group 120.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/xVTSHGroup%20120(1).png)

从统计的数据来看，AUG（800803）被抽中的次数明显最多，可以进一步分析中奖概率是否符合预期。
