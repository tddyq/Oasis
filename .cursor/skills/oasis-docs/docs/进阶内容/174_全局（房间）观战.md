#  全局（房间）观战

## 功能介绍

全局观战也叫房间观战或者赛事观战，从房间观战席加入，可以选择任意玩家观战，还有自由视角观战等功能。

## 如何适配

全局观战是独立的观战系统，有独立的一套观战UI和机制，由于该功能需要进行开发者进行针对性适配，所以默认对玩法关闭，开发者在适配完毕后可以申请开启房间功能。

### 添加全局观战相关组件

1.打开项目内GameMode蓝图。<br>
2.细节中搜索Component，找到ComponentConfigs，点击加号添加<br>

### 玩家加入时过滤：

房间观战和好友观战，由于本质也是玩家加入，所以依然会触发PostLogin，玩家登陆事件，所以需要在该处进行过滤，通过PlayerController:IsPureSpectator()判断是否为观战玩家，将其从普通玩家中过滤出去。

### 游戏内部分提示和页面特殊处理

由于没有对应的分队归属，也不是具体玩家，部分游戏提示和页面对于观战目标并不完全使用，比如结算，胜负UI等，需要通过PlayerController:IsPureSpectator()判断是否为观战玩家，进行对应的特殊化处理。

### 对局结束后的结算界面

观战玩家和普通玩家对局结束后显示的结算信息可能会有差异，可以在结算信息广播后判断是否观战玩家显示不同的结算信息，

### 全局观战UI特殊处理

由于全局观战UI是一个固定模板，部分玩法可能不需要全局观战UI的某些部分，可以获取OBUI对对应部分UI进行隐藏处理。

### 游戏中相机视角相关调整：

如果玩法调整了玩家相机视角，可以通过PlayerController.OnSpectatorChange委托，确认观战玩家切换了观战目标，并通过PlayerController:GetCurPawn()获取当前观战目标，然后调整观战目标对应的镜头参数。

## 相关接口

| 接口 | 描述 |  适用范围|
| ------ | ------ | ------ |
| PlayerController:IsPureSpectator() | 是否观战玩家（包括全局观战和好友观战） | DS和客户端 |
| PlayerController:IsGlobalObserver() | 是否全局观战玩家 | DS和客户端 | 
| PlayerController:GetCurPawn() | 获取当前观战目标PlayerPawn | DS和客户端 | 
| PlayerController:OnSpectatorChange | 委托，观战目标切换后调用 | DS和客户端 |
| PlayerController:IsCurrentSpectatorFreeView | 参数，是否自由视角 | DS和客户端 |

## 部分代码示例

### 获取观战UI

```lua
local Mgr = GameFrontendHUD:GetLogicManagerByName("observe")
if mgr then
      local OBUI = InGameUIManager.GetWidgetFromName(Mgr, "OBUI_UIBP")
end
```

### 全局观战自由视角下调整摄像头位置

全局观战分为自动视角（跟随视角）和自由视角，在自由视角下如果有切换关卡功能可能需要自由视角跟随到新关卡，以下提供设置自由视角下摄像头位置的方法：

```lua
local ViewTarget = Controller:GetViewTarget()
ViewTarget:SetActorLocation({X=新位置x, Y=新位置y, Z=新位置z })
```
