#  使用枚举类型

绿洲启元编辑器支持枚举类型，但绿洲启元编辑器枚举类型的创建和使用与虚幻引擎稍有不同，下面会逐步介绍。
<br>
## 基本介绍

### 1. 枚举是什么？

![企业微信截图_16868353788232.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868353788232.png)

**枚举** （英：Enum或Enumeration）用于为整数值命名，通常用于在复数状态下（≥3）进行分类判断（对应布尔值仅支持0和1）。

例如，一个基本的游戏菜单状态可以表示为名称并编程为整数。

```js
– 0 = Mainmenu
– 1 = Game
– 2 = Pause
– 3 = Gameover
```

### 2. 绿洲枚举类型与虚幻原生枚举类型对比

所有游戏引擎都可以使用枚举，因为它们是基本的编程功能。虚幻引擎可以在c++和蓝图中使用枚举，绿洲启元编辑器继承了此特性，并支持在lua中使用枚举。<br>

| 项名 | 编辑器中创建 | 不允许与系统枚举重名 | 不允许创建同名枚举 | lua中添加自动映射 |
| ------ | ------ | ------ | ------ | ------ |
| 虚幻 | ✔ | ✖ | ✖ | ✖ |
| 绿洲（2022年4月版本） | ✔ | ✖ | ✔ | ✖ |
| 绿洲（2022年6月版本） | ✔ | ✔ | ✔ | ✔ |

<br>

## 在绿洲启元编辑器下创建枚举

### 1. 创建枚举

1.1 在【内容浏览器】的空白处右击，找到`蓝图` -`枚举` ，创建一个枚举类型产并为其命名（如“MyEnum”）。

![企业微信截图_16868353906119.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868353906119.png)
  
1.2  枚举命名注意

- 避免与系统保留枚举重名：[此API页面](https://developer.gp.qq.com/api/#/?classDetailShow=false&isSelect=1)记录了所有和平**全局保留的枚举类型**，请开发者在创建自定义枚举类型时避免与此页面下的枚举重名（2022年6月版本绿洲启元编辑器固定不允许玩家创建与系统同名的枚举）。
- 避免同一个工程下重名：同一个玩法工程下也不允许创建**同名枚举**，即便同名的枚举分别在不同的目录下。
   
    ![企业微信截图_16868354142099.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868354142099.png)
		
1.3 编辑枚举值

双击创建好的枚举类型资产，打开枚举值编辑界面，点击【新建】按钮为创建的枚举添加不同的枚举值，并为其命名和添加描述，点击保存来应用更改

 ![企业微信截图_16868354316452.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868354316452.png)
 
1.4 自动映射

枚举创建完毕后，绿洲启元编辑器会自动为枚举类型添加映射，写入至项目目录下`\Script\Common\ue_enum_custom.lua`文件，你可以手动将此映射require至你需要使用枚举的lua脚本以使用枚举值（2022年6月版本此映射将在玩法运行时自动加载，无需手动require）

![企业微信截图_16868354408863.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868354408863.png)

### 2. 使用枚举

可参照如下示例使用枚举类型

2.1 蓝图设置

先创建一个要调用枚举值的Actor（如“MyActor”）。若要以蓝图变量方式存储枚举值，可在创建的Actor下新建一个变量，类型为用户自行创建的枚举类型（如“MyEnum”），你可以随时更改此变量的枚举值。

`注意：以蓝图变量方式存储枚举值非必须选项，开发者可根据具体需求选择是否需要以蓝图变量方式存储枚`

![企业微信截图_16868354621148.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868354621148.png)

2.2 lua脚本设置

你也可以在lua脚本中定义变量以存储和改变枚举值。

```lua
--Edit Below--
local MyActor = {
    MyEnumration = MyEnum.Banana
};
```

枚举值在计算和使用时**被视为整数**，例如定义如下枚举：

```lua
---@enum MyEnum
MyEnum = { 
    Apple = 0,
    Orange = 1,
    Banana = 2,
};
```

可通过如下代码打印：

```lua
function MyActor:ReceiveBeginPlay()
    print("MyActor:ReceiveBeginPlay");
    print(MyEnum.Apple); -- 输出的值为0
end
```

这里贴出完整的示例工程代码，方便开发者参考使用，另附示例工程压缩包，请前往文章末尾的附件进行下载。

```lua
---@class MyActor_C:Actor
---@field Box UBoxComponent
---@field DefaultSceneRoot USceneComponent
---@field MyEnumeration TEnumAsByte<MyEnum>

require('Script.Common.ue_enum_custom') -- 2022年6月版本无需手动require

--Edit Below--
local MyActor = {}; 

function MyActor:ReceiveBeginPlay()
    print("MyActor:ReceiveBeginPlay");
    print(MyEnum.Apple); -- 输出的值为0
    print(self.MyEnumeration); -- 输出的值与MyActor蓝图中的枚举值一致
    if self:HasAuthority() == true then
        --仅在服务器运行
        self.Box.OnComponentBeginOverlap:Add(self.BeginOverlap, self);
        self.Box.OnComponentEndOverlap:Add(self.EndOverlap, self);
    end

end

function MyActor:BeginOverlap()
    print("MyActor:BeginOverlap");
    self.MyEnumeration = MyEnum.Banana; -- 输出的值为2
    print(self.MyEnumeration);
end

function MyActor:EndOverlap()
    print("MyActor:EndOverlap");
    self.MyEnumeration = MyEnum.Orange; -- 输出的值为1
    print(self.MyEnumeration);
end

return MyActor;
```
