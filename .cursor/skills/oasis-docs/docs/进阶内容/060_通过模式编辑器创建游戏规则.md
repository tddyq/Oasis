# 通过模式编辑器创建游戏规则

# 模式编辑器

模式编辑器是绿洲启元编辑器中，用来制作游戏玩法模式的工具。模式编辑器通过创建和管理触发器的方式，来实现游戏逻辑。

<br>

## 模式编辑器概述

### 触发器

触发器（Trigger）是用户定义在游戏中的规则，是一类由事件驱动的特殊过程。触发器的功能是在特定的事件发生时，对规则的条件进行检查，如果条件成立，就执行规则中的动作，否则不执行。所以，触发器结构由3部分组合成：

**事件（Event）**

在游戏工程中，不同游戏对象之间需要互相通信、共享信息，彼此存在依赖关系。而事件就是一种游戏对象之间一对多通信的实现方式。
当某个游戏对象发生改变时，可以发出一个事前约定好的事件，这样与它存在依赖关系的所有对象都能收到通知，并针对通知进行处理。
在触发器结构中，指定触发器需要监听的事件，那么当被监听的事件被发送出来之后，事件会遍历所有监听该事件的触发器，并一并激活。

**条件（Condition）**

条件是触发器动作的门槛，用来判断触发器被激活时，是否应该执行动作的逻辑。

**动作（Action）**

动作是触发器激活时，实际执行的逻辑，是触发器所代表规则的逻辑主体。

---

### LevelDirector

在绿洲启元编辑器中使用模式编辑器制作玩法模式，实质上是创建了一个名为LevelDirector的模块。

![企业微信截图_16868115439046.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868115439046.png)

LevelDirector是开发者在工程中自定义的所有触发器的管理器。在对局开始后，游戏会根据工程设置中配置的LevelDirector子类，来创建LevelDirector实例。然后LevelDirector会根据保存的自定义事件和触发器数据，逐一创建自定义事件和触发器，然后根据触发器监听事件，建立触发器和事件的关系，将触发器注册到监听中。

<br>

## 使用流程

### 配置LevelDirector

打开工程内的 ``UGCGameMode`` 蓝图，在 ``GMDataManager`` 属性组下找到 ``GMData Source Dynamic Object`` 组件，通常确保将路径 ``Asset/Blueprint`` 下的 ``UGCLevelDirector`` 蓝图关联到 ``Level Director Class`` 属性即可。

### 编辑自定义事件

点击 ![企业微信截图_16868117316062.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868117316062.png) 按钮，可以打开自定义事件面板。点击右上角的加号，可以创建新的自定义事件；点击事件右侧的叉号可以删除事件。

![企业微信截图_16868117499227.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868117499227.png)

点击事件右侧的加号，可以为自定义事件添加参数，并指定参数的类型。

![企业微信截图_16868117599127.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868117599127.png)

若想在游戏中激活自定义的事件，可以使用模式编辑器的事件接口来派发事件。

```
LuaQuickFireEvent([事件名称], [发出事件的对象], [事件参数]);
```

### 条件的参数配置

对于传入参数的方式，这里模式编辑器提供了多种不同方式。

![企业微信截图_16868120345995.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868120345995.png)

**固定值**

勾选Value则需要手动输入来为参数指定值，而参数的取值是固定值，运行中不会改变。参数默认将使用固定值。

![企业微信截图_16868120454795.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868120454795.png)

**事件参数**

可以将触发器所监听的事件中传入的参数配置给Condition。

勾选EventParam后，在参数配置界面上，直接在下拉菜单中，选择触发器监听事件的参数。下拉菜单只能读取到与定义的参数<span style="color: #EE0000">**相同类型**</span>的参数。

![企业微信截图_1686812055439.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_1686812055439.png)

### 为触发器添加动作

点击 ![企业微信截图_16868120856265.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868120856265.png) 按钮，为触发器添加动作。

![企业微信截图_16868120979558.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868120979558.png)

一个触发器可以有多个动作，每个动作直接并列执行，没有先后顺序。

> 不建议为动作添加延迟执行时间，可能因缓存机制导致功能异常

![企业微信截图_16868121218517.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868121218517.png)

**动作的参数配置**

Action的参数配置方式与Condition一致，这里不再赘述。

**动作的Config配置**

系统预制的Action中，都会有是否开启tick和Action的执行范围两个参数。

- 是否开启tick：开启后，Action将保持执行，并在每个tick时，调用Action的Update
- 执行范围：当前只支持在服务端执行

![企业微信截图_16868122376135.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868122376135.png)

除此之外，每个不同的系统预制Action可能还需要配置不同的Config，以添加Buff的Action举例：添加Buff的Action中，需要指定给玩家添加的Buff相关的参数，例如Buff的Id，层数等等。

![企业微信截图_16868122478622.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16868122478622.png)
