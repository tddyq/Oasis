#  游戏结束时的逻辑处理

游戏的战斗阶段结束后，需要将玩家的结算数据上报，而后关闭服务器并断开客户端连接，这是游戏结束时必须要添加的逻辑；如果没有正确处理这部分逻辑，DS服务器超过有效时长将被异常关闭，导致玩家数据未被正确保存而丢失，以及客户端异常等表现。

<br>

## DS关闭时机

每个DS进程有固定的有效存活时间，超过时长将被强制杀进程以关闭DS，因此每个玩法需要确保在有效时间内被正确关闭。

> DS默认有效时长为30分钟，如有需要可联系绿洲官方运营人员提出调整申请

**DS剩余时长** 

开发者可以调用 [``GetDSRemainingTime``](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=GetDSRemainingTime) 查询DS关闭前的剩余时长，以及时处理结束前的收尾逻辑。

```lua
local RemainTime = UGCGameSystem.GetDSRemainingTime()
print(string.format("当前DS剩余关闭时间：%s 秒", RemainTime))
```

**DS关闭通知**

[``SetDSCloseNotify``](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=SetDSCloseNotify) 提供了监听DS关闭前通知的机制，通过此函数可以注册希望通知的时间点，系统将在指定的时间点通过 ``UGCGenericMessageSystem.UserDefinedMessages.UGC.UGCDSShutDownManager.DSCloseNotify`` 事件进行广播通知，在监听事件的回调中处理游戏阶段的逻辑。

```lua
-- 注册通知点，离关闭前60秒将触发事件
UGCGameSystem.SetDSCloseNotify({60})

-- 监听关闭通知事件，执行数据存档
UGCGenericMessageSystem.ListenGlobalMessage(
	UGCGameSystem.GameState,
	UGCGenericMessageSystem.UserDefinedMessages.UGC.UGCDSShutDownManager.DSCloseNotify,
	self,
	function()
		UGCPlayerStateSystem.SavePlayerArchiveData(UID, self.PlayerData)
  end
)
```

## 结束处理步骤

### 1. 显示结算界面

游戏对局结束后，通常需要弹出独立的结算界面，让玩家可以清楚了解到获得的分数与奖励，同时界面中需要保留返回大厅的功能，可以让玩家顺利返回和平大厅。

返回大厅功能形式，可以参考以下两种：

- 返回大厅按钮：在结算界面，设置返回大厅按钮，玩家点击按钮后，即可顺利返回大厅。
- 倒计时返回大厅(可选)：进入结算界面后，启动倒计时程序，在界面中显示倒计时时间，时间结束后，玩家自动返回大厅。

### 2. 触发结束游戏事件

当游戏对局结束后，需要处理发送结算以及关闭DS服务器等逻辑，为此需要创建一个用于监听游戏结束的触发器。

首先创建自定义事件，点击模式编辑器，点击编辑事件，新建一个事件，取名为GameEnd。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pMlxHimage.png)

而后，创建一个新的流程触发器，取名为结束游戏，事件类型(Event Type)选择刚刚创建好的自定义事件【GameEnd】。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/KkteIimage.png)

当游戏对局结束后，使用编辑器的事件接口来触发【GameEnd】事件
> 接口：LuaQuickFireEvent([事件名称], [发出事件的对象], [事件参数]);
	
在确定对局结束的代码逻辑中，调用此接口，以下是示例：

``` lua
if UGCGameSystem.GameState.CurrentRound >= self.MaxRound then --当前记录的对局数已经达到最大对局数，游戏结束
	LuaQuickFireEvent(“GameEnd”, self); -- 触发【GameEnd】事件
end
```

在触发【GameEnd】事件后，将自动执行绑定在事件之下的动作（Action）。

---

### 3. 结算与退出DS

关闭DS前需要将每个玩家的数据发送给后台进行结算上报，该数据会影响玩家完成绿洲启元相关的任务和数据统计，模式编辑器将发送结算与关闭DS的逻辑封装成一个动作脚本 ``UGC请求退出DS``，开发者仅需在触发【GameEnd】事件之后，指定执行该动作即可。

> 无需再单独调用 [``SendPlayerSettlement``](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E6%B8%B8%E6%88%8F%E9%80%9A%E7%94%A8%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=SendPlayerSettlement) 做结算上报

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/vS6h1image.png)

---

### 4. 断开客户端链接

DS关闭后，需要同步关闭客户端对服务器的长链接检测，否则玩家客户端会弹出无法连接到服务器的报错信息。可以在客户端监听OnGameEnd事件并调用 [``DisconnectClient``](https://developer-test.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E6%B8%B8%E6%88%8F%E9%80%9A%E7%94%A8%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=DisconnectClient)。

代码示例：

``` lua
-- 当监听到游戏结束事件时，触发断开客户端链接
function MainUI:OnGameEnd(GameResultData)
    ugcprint("MainUI:OnGameEnd GameResultData: ", tostring(GameResultData))
		
		-- 断开客户端链接
		UGCGameSystem.DisconnectClient()

    -- TODO 其他逻辑
end
```

## 特殊情况：无存活玩家

在DS服务器中，如果所有玩家全部处于死亡状态（无存活Pawn），服务器会开启超时检测：若在一定时间内无有效存活玩家，将会强制关闭。

如果玩法设计中存在所有玩家全部死亡，且游戏流程仍需继续的场景，则需要在UGCGameMode中修改检测时间，以避免DS服务器被强制关闭而导致游戏被中断，如图所示：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/eZIlximage.png)

``Check Period`` 默认设定的时间为120s，开发者可以根据需要修改设定时间

**注意**
> 虽然可以使用这个方法来避免DS服务器被强制关闭，但我们还是建议，在游戏玩法设计时，尽可能避免出现所有玩家死亡或其他极端的情况，原则上尽可能不要修改检测超时的参数。
