# 团队账号功能使用说明

绿洲启元编辑器和管理平台上传提审玩法工程时，账号必须要归属于一个团队才能正常进行操作，您可以通过自己创建团队（团队内允许仅有一个成员），或者加入已有团队的方式实现。（注：**本地调试可以不用归属团队也可进行**）。

本文将详细介绍当前团队账号的主要功能和常见问题。

<br>

## 一、有无团队在使用编辑器时的区别
编辑器上传主要分为两种：

- **测试上传：不需要加入团队**。主要是开发人员调试使用，上传后在调试客户端“我的上传”中可以看到。<span style="color:red;font-weight:bold">``测试上传``</span> 的工程版本在管理平台提审选择版本时不会出现。<span style="color:red;font-weight:bold">``测试上传``</span> 前需要填写游戏账号UID白名单，<span style="color:red;font-weight:bold">``测试上传``</span> 的玩法仅白名单人员可见。

- **发布上传：需要加入团队**。通过 <span style="color:red;font-weight:bold">``发布上传``</span> 的工程版本才能在管理平台进行提审。如果未加入团队，点击 <span style="color:red;font-weight:bold">``发布上传``</span> 编辑器弹窗会提示开发者” [账号未加入团队，请点击前往绿洲启元管理平台创建或加入团队”。](https://gpeditor.qq.com/user#myprojects)可点击跳转到管理平台，选择创建团队或加入已有团队（具体操作方式按照以下第二部分说明）。

## 二、创建团队或加入已有团队
使用绿洲启元管理平台（下文中简称管理平台）相关功能，账号必须要属于团队才能完整使用，因此请按照如下指引进行操作：

开发者首次登录 [绿洲启元管理平台](https://gpeditor.qq.com/user#myprojects)，登录后页面默认会提示“您的账号没有加入任何团队，暂无法查看，请前往账号管理了解”，可以通过点击「账号管理」页签跳转到账号管理页面，可选择创建团队或加入已有团队。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/APzVGimage.png)

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/DjH0Eimage.png)

这两种的区别以及具体操作流程如下：

**1、如果您属于个人开发者或者团队、机构管理员，我们建议您选择“创建团队”**。创建成功后，您当前的登录账号将默认为该团队的主账号，可以管理其他账号。

点击“创建团队”后，按照要求填写相关信息则可以成功创建团队，创建人账号则默认成为团队的主账号，可对团队信息进行管理，可以邀请其他账号加入到当前团队，与团队内成员一起管理项目。具体的团队信息和成员的管理可通过“账号管理”页面操作。

**相关说明：**

*●作品提审后的一些重要节点官方会通过邮件形式发送，请确保相关信息正确填写；*

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/izjEJimage.png)

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/ItvpZimage.png)

**2、如果您只需要与已有团队共同维护项目，您可以选择加入已有团队**。加入团队后默认为该团队的子账号，在团队中拥有的权限依赖于被主账号设置的角色。

通过点击“获取账号ID”或者点击平台右上角个人头像，可以看到账号ID，然后把账号ID发送给要加入团队的主账号，团队主账号通过“账号管理-添加共同运营子账号” 功能来邀请其他账号加入团队。

## 三、添加（邀请）团队子账号

团队主账号可通过“添加共同运营子账号” 来邀请其他成员加入团队。添加时需要填写“账号ID”来进行邀请，当被邀请的账号接受邀请后，才能成功加入到团队中，并成为团队子账号。

添加子账号时，需要设置该账号的角色（管理员、开发、运营），不同角色对应的权限在页面上有对应说明，下文也会详细说明，<span style="color:red;font-weight:bold">请主账号根据需要谨慎设置子账号相关角色</span>（后续主账号也可随时调整修改）。

**特别说明：**

**1、创建团队的账号默认为团队主账号，其他被添加的账号都默认为团队子账号，一个团队只有一个主账号，子账号可以没有，也可以为多个。**

**2、管理员角色已经拥有较大权限，当子账号角色选择管理员时，则无法选择其他角色；除管理员外的其他角色可以多选。**

## 四、支持团队成员共同上传玩法

**团队内所有成员都可对该团队下的项目进行提审、信息编辑等**。假设该工程团队成员A通过编辑器提审上传，团队内其他成员（比如B、C等）也可以在管理平台提审同一玩法，**不再要求编辑器上传和提审的账号为同一个**，上传玩法变得更加方便。

玩法在管理平台提审时相关变化说明：

**● 玩法工程可选列表**：在玩法第一次绑定工程时，可选择的工程列表，包括团队内所有成员通过编辑器 <span style="color:red;font-weight:bold">``发布上传``</span> 的最新3个未绑定过玩法的工程（不再是仅自己账号下 <span style="color:red;font-weight:bold">``发布上传``</span> 的工程）

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/oNQt1image.png)
**● 玩法版本可选列表**：选择工程后，可选的版本列表包括团队内所有成员上传同一工程最新3个未绑定过的玩法版本（不再是仅自己账号下 <span style="color:red;font-weight:bold">``发布上传``</span> 的版本）

## 六、团队成员删除
团队主账号可以通过账号管理进行删除团队成员操作。删除团队成员时，该成员之前创建的玩法并不会被一起删除，**玩法项目是与团队绑定的并不是跟个人绑定。**

被删除的成员登录管理平台后，无法在管理和操作到团队内的项目内容，因此，团队内成员离职后，建议尽快从团队内删除。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Kz3ZFimage.png)

## 七、团队主账号移交

团队成员由于工作上的变动或其他原因导致主账号需要发生变更时，团队内主账号可以通过“主账号换绑”功能来完成团队所有权的转移。
<span style="color:red;font-weight:bold">（注意：主账号拥有对团队内成员的管理权和所有页面的访问权限，包括代表团队内成员及作品与官方进行签约的权限，且一个团队只有一个主账号，请谨慎进行移交操作）

点击主账号换绑后，需要选择团队内的一个子账号，将主账号权限进行移交。主账号换绑完成后，被移交的子账号将变为团队主账号（注意：团队只会有一个主账号），原先的主账号则默认变为团队子账号。
	
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/s7qN4image.png)

团队账号主体的归属说明：

● 如果团队为个人开发者性质：默认当前团队主账号的注册人为该团队账号的使用者，对于团队账号的使用权归属以当前团队主账号的注册人为准，因此请谨慎进行主账号的移交操作。

● 如果团队为公司性质：原则上该团队账号使用权归属于公司所有。除此之外，如果发生使用权归属不明确的情况，官方将结合建立团队账号的初始主账号情况、公司主体的官方发函等综合认定。

<br>
	
## 八、团队账号使用中的特别说明
	
**1、玩法签约时，使用了团队成员A的账号登录的绿洲启元管理平台进行签约上传，现在成员A离职了，需要重新发起签约吗？**
	
**``不需要``**。玩法签约是公司与公司之间发生的签约，不管团队内成员变化或者离职，只要玩法所属的公司主体没有发生变更，相关签约信息就一直有效，不会随着团队内成员变化而变化。只有当玩法所属的主体发生变化，或者需要更换时，才需要对原有签约解约，再重新发起签约。

**2、有了团队账号，团队内任意成员是不是就可以操作其他成员提审的项目或者提审其他成员创建的项目？**
	
**``可以``**。团队账号支持团队玩法共享，支持团队成员创建或提审其他成员的玩法项目。并登记团队成员相关的操作记录，相关操作包括：玩法的版本提审、撤销提审、玩法信息更新提审、玩法信息更新提审撤销。
	
玩法项目共享相关操作包括：“添加新版本、开启测试、关闭测试、更新版本、编辑、提审、撤销提审、玩法内购、徽章、段位等”。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/rHXa4image.png)
	
**3、团队成员离职后，离职成员上传的项目会有影响吗，需要做哪些操作？**
	
**玩法工程管理上**，团队成员离开后，玩法工程通过离线发送或者以SVN，GIT的形式进行版本管理。在同一个团队下，同名玩法工程上传会被认定在原项目下，因此原工程创建人的离开不会影响后续项目，其他团队成员都可以上传同一玩法工程。

**在绿洲启元管理平台上**，主账号及时将离开团队的成员账号从团队中删除。由于玩法项目是与团队绑定的，所以即使团队成员离职玩法项目也不会消失，也不会影响已上线的玩法。从团队中删除后的成员，将无法登录管理平台查看、操作团队内的项目。

## 九、团队账号使用中可能会遇到的问题

**1、团队添加子账号提示 “不能添加已经加入团队的用户”**
	
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/vrftkimage.png)
	
提示原因： 一个账号目前暂时只支持加入到一个团队，当邀请的子账号已经归属于其他团队时，会出现该提示。
	
解决方法： 分以下两种情况不同的处理方式：
	
**情况1：该账号是其他团队的子账号**
	
被邀请的账号需要先从其他的团队先退出，退出方式为：需要该账号所在团队的主账号在“账号管理“页面把该子账号从团队删除即可；确认从其他团队移出后，当前团队可以再一次进行添加子账号，则不会再出现该提示。
	
**情况2：该账号是其他团队的主账号**
	
a. 如果主账号不再保留原团队，需要直接从原团队退出，则需要该账号需要向官方申请，通过官方操作从原团队退出
	
● 如原团队中没有项目资产或团队的项目资产确认不再保留，在向官方申请时，需要通过邮件提供以下信息：申请公司名称/申请人姓名、申请人账号ID、申请人在绿洲启元管理平台的“账号管理”页面截图、申请从团队退出且账号下项目资产不再保留的相关文字说明等 （如为公司主体，则需要在以上信息附上公司盖章扫描后提供）
	
● 如团队资产需要保留，因为资产必须归属于一个团队，因此建议按照以下原团队需要保留的方式操作，目前暂不支持两个团队之间资产合并或者转移的操作。
	
b. 如果原团队需要保留，则可以在原团队先添加其他子账号，然后将主账号转移给新的账号（该账号默认成为子账号），然后再按照子账号的方式退出原团队

**2、团队成员权限变更后相关刷新提示**
	
团队主账号可以修改团队成员角色、删除/邀请团队成员、移交主账号等操作，当被修改的账号因以上操作，权限发生变化后，如果处于绿洲启元管理平台页面登录状态时，页面会提示：“您所在团队或账号权限信息发生变化”，这时候点击刷新按钮，页面刷新后则拥有最新的权限。
	
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/BJW87image.png)
	
**3、团队邀请过期提示**

当子账号同时收到多个团队邀请时，如果接受了其中一个团队邀请并成功加入团队，此时再点击“接受”其它团队的邀请，则会提示：”邀请消息已过期“。
	
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/huVegimage.png)

**4、主账号重复邀请子账号**

团队主账号邀请账号加入团队时，如果对方未处理，则无法再次邀请同一个账号，重新邀请时会提示“已经邀请过该用户，且未处理，请不要重复邀请”，这时候只要让被邀请账号登录绿洲启元管理平台处理之前的邀请信息即可，不需要重复发起邀请。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/NPa3fimage.png)
