# 调试日志说明

## 简介
在游戏开发过程中，调试是非常重要的一环，往往开发时间只占3成，调试时间占了7成，掌握快速调试的技巧，能大大提升开发效率和定位问题的效率，做到事半功倍。本篇教学会讲解绿洲启元编辑器下常用的调试技巧。

<br>

## 通用技巧
使用绿洲启元编辑器开发，通常会分两个阶段，编辑器Debug阶段以及手机真机调试阶段。但无论哪个阶段，都有一些通用的方法和技巧，下面会逐一进行介绍。

### 日志打印函数ugcprint
我们可以使用`ugcprint`函数进行日志打印，此函数功能等同于Lua中的`print`函数，但会根据游戏当前运行的环境，决定日志是否打印。使用编辑器Debug以及配套的研发用安卓Apk时，`ugcprint`会进行日志打印。当游戏运行在正式服的和平精英中时，`ugcprint`函数则不会进行打印。

可以把`ugcprint`函数看作开发环境中才生效的打印函数，而`print`函数则在正式环境中也会生效，所以尽量用于错误日志的打印，或者一些关键流程日志的打印。此外，`ugcprint`函数打印的日志会在开发环境的游戏中显示出来，如下所示，点击左上角问号按钮，会显示`ugcprint`函数打印的日志

![001.f0287a05.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/001.f0287a05.png)

以下为示例代码：
```lua
function TestCube:ReceiveBeginPlay()
    ugcprint("TestCube:ReceiveBeginPlay")
    self.SuperClass.ReceiveBeginPlay(self);
    self.Box.OnComponentBeginOverlap:Add(self.Box_OnComponentBeginOverlap,self)
end

function TestCube:Box_OnComponentBeginOverlap(OverlappedComponent, OtherActor, OtherComp, OtherBodyIndex, bFromSweep, SweepResult)
    ugcprint("TestCube:Box_OnComponentBeginOverlap")
end
```

可以看到，界面中显示的日志带有不同的标签，其中`UGCClientLog`代表客户端日志而`UGCServerLog`代表服务器日志，可以通过选项开关和过滤相关日志。

![002.89e2f63c.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/002.89e2f63c.png)

细心的同学应该会注意到`TestCube:ReceiveBeginPlay`的日志只有客户端的但是没有服务器打印，而我们的代码明明是双端运行，这是怎么回事呢？

这里是因为部分服务器Actor的BeginPlay运行时间较早，此时我们的调试客户端还没启动完毕，所以日志无法正确发送到我们的调试客户端，如果有需要确认服务器的BeginPlaye是否正确运行，建议在日志文件中搜索更为稳妥。


### 打印对象类型和对象路径
在实际的调试中，往往需要知道一个对象的类型和路径，此时可以使用`UE.GetFullName(Obj)`来获取对象的类型和路径。如下代码所示，打印了进入的区域的Component，以及Actor的类型。

```lua
function TestCube:Box_OnComponentBeginOverlap(OverlappedComponent, OtherActor, OtherComp, OtherBodyIndex, bFromSweep, SweepResult)
    ugcprint("TestCube:Box_OnComponentBeginOverlap")
    ugcprint("TestCube:Box_OnComponentBeginOverlap:"..UE.GetFullName(OverlappedComponent))
    ugcprint("TestCube:Box_OnComponentBeginOverlap:"..UE.GetFullName(OtherActor))
end
```

可以看到，日志顺利的打印出对象类型，所在地图，以及实例名称。

```lua
UGCServerLog: TestCube:Box_OnComponentBeginOverlap:BoxComponent /ColorChange/UGCmap.UGCmap:PersistentLevel.TestCube_14.Box
UGCServerLog: TestCube:Box_OnComponentBeginOverlap:UGCPlayerPawn_C /ColorChange/UGCmap.UGCmap:PersistentLevel.UGCPlayerPawn_C_0
```
![003.96630569.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/003.96630569.png)

类似`UE.GetFullName`的函数还有`UE.GetName(Obj)`,`UE.GetPathName(Obj)`等，可以在LuaHelper`LuaIntergration.lua`文件中查看到相关函数定义。


### 快速搜索错误日志
熟练掌握日志打印后，还需要快速的搜索自己的日志以便定位问题，往往在实际的调试过程中，代码中途遇到错误阻塞了流程，这时我们自己添加的日志因为被阻塞导致没有被打印，但日志文件非常大，不好定位到底哪里报错了，这时需要我们掌握一些常用错误关键字帮我们快速定位问题。

`stack traceback`，`a nil value`，`LuaException`，`call a nil`，通过这些关键字，大部分情况可以找到对应的报错代码，大家也可以在调试过程中总结遇到的常见问题，积累自己的常见错误关键字，帮助快速调试。

## 更方便的自定义日志打印工具
普通的日志打印往往需要拼接字符串，且针对多种不同类型的对象的打印，需要写多行打印代码来打印，十分不方便，并且在拼接字符串的过程中，也极容易出错，例如某些对象为nil，但直接进行字符串拼接，则会导致日志打印那行代码报错，日志也没打印出来，反向查找费时费力。

所以，建议针对`ugcprint`进行二次封装，且针对不同的使用场景，不同的对象，进行自动的tostring和拼接，让打印日志很简单更轻松，下面是一个封装后的打印示例。

```lua
function TestCube:Box_OnComponentBeginOverlap(OverlappedComponent, OtherActor, OtherComp, OtherBodyIndex, bFromSweep, SweepResult)
    --较为繁琐的日志打印方式
    ugcprint("TestCube:Box_OnComponentBeginOverlap")
    ugcprint("TestCube:Box_OnComponentBeginOverlap:"..UE.GetFullName(OverlappedComponent))
    ugcprint("TestCube:Box_OnComponentBeginOverlap:"..UE.GetFullName(OtherActor))

    --封装工具简化日志打印
    UGCLog.Log("TestCube:Box_OnComponentBeginOverlap")
    --各种内容混排,无需手动拼接字符串
    UGCLog.Log(OtherActor, OverlappedComponent)
    UGCLog.Log("TestCube:Box_OnComponentBeginOverlap", 123, {x=1, y="haha", z= {v = OtherActor}}, OtherActor)
    --空对象传入也不会影响
    local value = nil
    UGCLog.Log("TestCube:Box_OnComponentBeginOverlap test nil", value, 123)
end
```

工具代码：

```lua
UGCLog = UGCLog or {}


UGCLog.IsServerCached = nil;

function UGCLog.IsServer()
    if UGCLog.IsServerCached == nil then
        UGCLog.IsServerCached = UGCGameSystem.IsServer();
    end

    return UGCLog.IsServerCached;
end


local function tostringex(v, len)
    if len == nil then len = 0 end
    local pre = string.rep('\t', len)
    local ret = ""
    if type(v) == "table" then
        if len > 5 then return "\t{ ... }" end
        local t = ""
        local keys = {}
        for k, v1 in pairs(v) do
            table.insert(keys, k)
        end
        
        for k, v1 in pairs(keys) do
            k = v1
            v1 = v[k]
            t = t .. "\n\t" .. pre .. tostring(k) .. ":"
            t = t .. tostringex(v1, len + 1)
        end

        if t == "" then
            ret = ret .. pre .. "\n{ }\t(" .. tostring(v) .. ")"
        else
            if len > 0 then
                ret = ret .. "\t(" .. tostring(v) .. ")\n"  
            else
                ret = ret .. "\n(" .. tostring(v) .. ")"    
            end

            if len == 0 then
                ret = ret .. "\n" .. pre .. "{" .. t .. "\n" .. pre .. "}"  
            else
                ret = ret .. pre .. "{" .. t .. "\n" .. pre .. "}"  
            end
        end
    elseif type(v) == "userdata" then
        if UE.IsValid(v) then
            local Nextpre = string.rep('\t', len+1)
            ret = ret .. pre .. tostring(UE.ToTable(v)) .. "\t(" .. type(v) .. ")"
            ret = ret .. "\n" .. pre .. "{\n" .. Nextpre .. "ClassAndPath:\t" .. UE.GetFullName(v) .. "\n" .. pre .. "}"
        elseif v ~= nil then
            ret = ret .. pre .. tostring(UE.ToTable(v)) .. "\t(" .. type(v) .. ")"
        else
            ret = ret .. pre .. "[NullActor]"
        end
        
    else
        ret = ret .. pre .. tostring(v) .. "\t(" .. type(v) .. ")"
    end
    return ret
end

local function GetName(Actor)
    if UE.IsValid(Actor) then
        return string.format("\n%s:\n{\n ClassAndPath:\t%s\n}", tostring(UE.ToTable(Actor)), UE.GetFullName(Actor))
    elseif Actor ~= nil then
        return type(Actor)
    end

    return "[NullActor]";
end

local LogTag = function()
    local GameStr = "[UGC]";

    if UGCLog.IsServer() then
        GameStr = GameStr .. "[服务器]"
    else
        GameStr = GameStr .. "[客户端]"
    end

    return GameStr;
end

local GetLogStr = function(...)
    local data = {}
    local index = 1
    for k, v in pairs({...}) do
        if k > index + 1 then
            data[k - 1] = "nil"
        end

        local typev = type(v)
        if typev == "table" then
            data[k] = tostringex(v)
        elseif typev == "userdata" then
            data[k] = GetName(v)
        else
            data[k] = tostring(v)
        end

        index = k
    end

    local out = table.concat(data, "\t")
    if out:len() >= 10 * 1024 - 100 then
        out = out:sub(1, 10 * 1024 - 100) .. "..."
    end

    return out
end

local BaseLog = function(...)
    --优化，判断开关，开关状态打开才进行字符串序列化
    if UGC_DEV_LOG then
        if _G.bEnableUGCDebugLog == nil or _G.bEnableUGCDebugLog > 0 then
            ugcprint(LogTag() .. GetLogStr(...))
        end
    end
end

local BaseLogShipping = function(tag, ...)
    print(LogTag() .. tag .. GetLogStr(...))
end

function UGCLog.LogIf(Cond, ...)
    if Cond then
        BaseLog(...)
    end
end

function UGCLog.LogErrorIf(Cond, ...)
    if Cond then
        BaseLogShipping("[Error]", ...)
    end
end

function UGCLog.Log(...)
    BaseLog(...)
end

function UGCLog.LogError(...)
    BaseLogShipping("[Error]", ...)
end

function UGCLog.LogUEObj(Obj)
    UGCLog.Log("[UEObj]", UE.ToTable(Obj))
end

function UGCLog.LogUEObjList(Lst)
    print("[UGC][UEObj] UGCLog.LogUEObjList");
    for k, v in pairs(Lst) do
        UGCLog.Log("[UEObj]", GetName(v))
    end
end

function UGCLog.LogUEPawnList(Lst)
    local TableHelper = require("Script.Common.TableHelper");
    for k, v in pairs(Lst) do
        UGCLog.Log("[Pawn]", v.PlayerKey, GetName(v))
    end
end

function UGCLog.LogNetStatus(PlayerKey)    
    local PS = UGCGameSystem.GetPlayerStateByPlayerKey(PlayerKey)

    if PS == nil then
        UGCLog.Log("[Net]还没有还没有PlayerState:", PlayerKey)
        return;
    end

    UGCLog.Log("[Net]PlayerKey:", PlayerKey, "是否断线：", PS.isLostConnection, "是否重连：", PS.isReconnected)
end

UGCLog.GameFlowStep = 1;
function UGCLog.LogGameFlow(...)
    BaseLogShipping("[GameFlow]", "["..UGCLog.GameFlowStep.."]", ...)
    UGCLog.GameFlowStep = UGCLog.GameFlowStep + 1;
end

return UGCLog;
```
