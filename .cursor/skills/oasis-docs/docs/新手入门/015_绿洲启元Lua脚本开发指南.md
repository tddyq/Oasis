# 绿洲启元Lua脚本开发指南

绿洲启元编辑器使用Lua作为脚本语言，可以使用大部分Lua库和自带接口。但Lua脚本和引擎参数和接口交互，有一套自己的规则，下面将逐步介绍这些规则。

<br>

## 基础操作

### 1. 基础语法说明

- 对象成员属性访问使用 **点(.)** 访问；
- 对象成员属性修改使用 **点(.)** 访问后在 **等号(=)** 赋值即可；
- 对象成员方法使用 **冒号(:)** 调用；
- 类静态方法使用 **点(.)** 调用；

<details open>
<summary> <font face="楷体">示例：基础操作</font> </summary>

```lua
-- 类静态方法调用,大部分UE4全局类，如UGameplayStatics都可以直接把前缀删除后直接使用。
local PlayerPawn = GameplayStatics.GetPlayerCharacter(self,0) 
-- 对象属性访问，可以访问该对象C++/蓝图/Lua属性。
local PlayerName = PlayerPawn.PlayerName	
-- 对象属性修改，可以修改该对象C++/蓝图/Lua属性。
PlayerPawn.PlayerName = "Test"
-- 对象方法调用：使用默认值参数。
PlayerPawn:K2_SetActorLocation(Vector(0,0,0)) 	
--PS：由于C++的默认参数是编译期生效，Lua脚本是运行时生效，所以无法直接使用C++接口默认值，这里的默认值使用0x0，number型为0，bool类型为fasle，指针为nil，如果是结构体那么内部参数按照以上默认值规则。
```

</details>

### 2. 可访问数据说明

- 绿洲启元编辑器中Lua导出/反射都是通过UE4反射系统实现，因此只能访问 UProperty 数据和 UFunction 方法。
- BP中属性说明:
  
  - BP中变量/方法/代理列表中的数据都可在lua中访问;
  - BPActor中Component默认都是变量，即都可以在lua中访问;
  - BPUMG中WidgetTree中的子控件: 勾选`IsVariable`则会显示在变量列表中，可在lua中方位；未勾选，则变量列表中没有，也不可在lua中访问；

### 3. 可重写函数说明

- C++中 `UFUNCTION `宏中有`BlueprintNativeEvent`和`BlueprintPure`标记为可重写函数。
- 蓝图中定义的函数均为可重写函数。

<details open>
<summary> <font face="楷体">示例：重写函数示例</font> </summary>

```lua
local UGCGameMode = {}; 
function UGCGameMode:ReceiveBeginPlay()
    self.SuperClass.ReceiveBeginPlay(self)	--作用等同于C++的Super::ReceiveBeginPlay()
end
```

</details>

### 1. 数据类型说明

lua访问UE数据时：基础类型数据直接值传递；复合数据类型做了自定义封装，达到成员变量的引用放访问效果，并且能继续调用数据类型的成员方法.

使用方式：枚举名 点 枚举值：`Enum.EnumItem`，该使用方式仅限于C++声明的枚举。

<details open>


### 3. 使用结构体

- lua中访问到的结构体为 `userdata`(带元表)，引用传递类型数据;
- 访问结构体属性使用基础语法访问即可;
- lua中给结构体赋值兼容 `userdata`类型(推荐)和 `table`类型(兼容支持) 赋值;
  - table 类型赋值时：Key为字段名, Value为字段值; 可部分赋值， 未赋值字段使用内存`0x0`填充;

<details open>
<summary> <font face="楷体">示例：数据类型-结构体</font> </summary>

```lua
local object = CreateUIWidget(bp_lua, "/Game/Demo/DemoUI.DemoUI_C");
local t1 = object.ColorAndOpacity;
local t2 = object.ColorAndOpacity.R;
local t3 = t1.R;

print(type(t1));        -- userdata

object.ColorAndOpacity = t1;                    -- 使用 userdata 类型赋值;
object.ColorAndOpacity = {R=1, G=2, B=3, A=1};  -- 使用table类型赋值; = FLinearColor(1,2,3,1);
object.ColorAndOpacity = {R=1, A=1};            -- 使用table类型赋值; = FLinearColor(1,0,0,1);
```

</details>


### 4. 使用容器

- 容器类型包括：`TArray`/`TSet`/`TMap`;
- lua中访问到的容器类型为 `userdata`(带元表);
- 常用操作：
  - 可使用 操作符`[]` 用 `Index`/`Key` 访问成员;
  - 可使用 操作符`[]` 用 `Index`/`Key` 进行赋值修改;
  - 可使用 函数`pairs()` 遍历;
- 关于索引:
  - 为保持lua使用习惯：`TArray`/`TSet` 在lua中操作时索引区间为 **[1,N]**，~~非[0, N-1]~~;

### 5. 使用代理

- 代理分为单播代理和多播代理;
- 单播代理：只能绑定一个回调函数; 多次绑定会覆盖，C++声明前缀为`DECLARE_DELEGATE`，如Button.OnMouseButtonDownEvent
  - 添加绑定：`Bind(CallBackFunc [, param])`;
  - 移除绑定：`UnBind()`;
  - 触发代理：`Execute(...)`;
- 多播代理：可绑定多个回调函数; 相同的绑定会自动去重，C++声明前缀为`DECLARE_DYNAMIC_MULTICAST_DELEGATE`，如Button.OnClicked
  - 添加绑定：`Add(CallBackFunc [, param])`:
    - `CallBackFunc` 作为Key: 同Func不同param会覆盖去重;
  - 添加绑定：`AddInstance(CallBackFunc[, param])`:
    - `CallBackFunc && param` 共同作为Key:同Func不同Param不会去重; 只有同Func同Param才会去重；
  - 触发代理：`Broadcast(...)`; (此处参数为代理的声明参数)
    - 移除绑定： `Remove(CallBackFunc [, param])`  or  `RemoveAll()` ;
    - 触发代理： `Broadcast(...)` ; (此处参数为代理的声明参数)
- 创建代理：创建代理这里主要用于SetTimer，代理的生命周期跟随Object，如果频繁的创建需要注意手动销毁；
  - 创建绑定：`ObjectExtend.CreateDelegate(Object, CallBackFunc)`
    - 删除代理：`ObjectExtend.DestroyDelegate(Delegate)`

<details open>
<summary> <font face="楷体">示例：数据类型-代理</font> </summary>

```lua
---@class MainUI_C:UserWidget   
---@field TestButton UButton            --该UMG有一个TestButton的按钮控件
local MainUI = {}; 
function MainUI:Construct()
    self.TestButton.OnMouseButtonDownEvent:Bind(self.TestButton_OnMouseButtonDown, self)   --绑定TestButton_OnMouseButtonDown函数
    self.TestButton.OnMouseButtonDownEvent:Bind(                                           --绑定匿名函数，单播委托此处绑定会直接覆盖上次绑定，这里仅做示范
        function(MyGeometry,MouseEvent) 
            print("MainUI:TestButton_OnMouseButtonDown")
        end, self)  

    self.TestButton.OnMouseButtonDownEvent:UnBind()                     -- 移除绑定
    self.TestButton.OnMouseButtonDownEvent:Execute(nil, nil)            -- 触发代理（控件事件不需要主动触发,此仅做用法示例）

    self.TestButton.OnClicked:Add(self.TestButton_OnClicked, self)   --绑定TestButton_OnClicked函数
    self.TestButton.OnClicked:Add(                                   --绑定匿名函数，注意绑定匿名函数无法通过Remove移除绑定
        function() 
            print("MainUI:TestButton_OnClicked")
        end, self)  

    self.TestButton.OnClicked:AddInstance(self.TestButton_OnClicked, self)  --此次绑定由于是同名同参，会被覆盖
    self.TestButton.OnClicked:Remove(self.TestButton_OnClicked, self)       --移除绑定
    self.TestButton.OnClicked:RemoveAll()                                   --移除所有绑定
    self.TestButton.OnClicked:Broadcast()                                   --触发广播代理（控件事件不需要主动触发,此仅做用法示例）

    self.TimerDelegate = ObjectExtend.CreateDelegate(self,                  --创建Delegate
        function()
            print("TimerExecute")
            ObjectExtend.DestroyDelegate(self.TimerDelegate)                --触发后可以手动销毁
        end
    )
    self.TimerHandle = KismetSystemLibrary.K2_SetTimerDelegateForLua(self.TimerDelegate, self, 3, false)    --执行Timer，定时3秒，不循环
end

function MainUI:TestButton_OnMouseButtonDown(MyGeometry,MouseEvent)
    print("MainUI:TestButton_OnMouseButtonDown")
end

function MainUI:TestButton_OnClicked()
    print("MainUI:TestButton_OnClicked")
end
```

</details>

## 方法调用

方法基础操作参考上文中【基础语法说明】板块， 注意成员方法与静态方法调用方式即可，普通的参数正常使用即可，无需特别注意。此部分主要针对特殊类型和标记的参数(引用/const等)。

- 方法返回值指：1.C++方法的return; 2.蓝图方法的Outputs;
- 返回值都是值拷贝方式返回， 即便声明为引用返回也是拷贝方式;
  - 拷贝是指对数据进行内存复制，然后返回复制的内存数据；
  - UObject类型实际都是指针类型UObject*:，返回指针数据值，不会复制UObject整体;
- 非const引用参数会作为返回结果输出，依次位于return值后面;

<details open>
<summary> <font face="楷体">示例：方法返回值</font> </summary>

```c++
// C++
// 简要声明:忽略各种标签宏说明等说明;
class UDemoWidget : public UUserWidget
{
    int32 Param;
    int32 GetParam()      { return Param; }
    int32& GetParamRef()  { return Param; }
	bool GetDoubleParam(int32& OutParam1, int32& OutParam2)
	{ 
		OutParam1 = Param + OutParam1;
		OutParam2 = Param + OutParam2;
		return true;
	}
};
// /Game/Demo/DemoUI 继承于 UDemoWidget 创建;
```

```lua
-- lua
local DemoWidget = CreateUIWidget(bp_lua, "/Game/Demo/DemoWidget.DemoWidget_C");

local Param = DemoWidget:GetParam();
local ParamRef = DemoWidget:GetParamRef();      -- 等价与 GetParam 调用;
ParamRef = 123;   -- 声明引用也是拷贝形式返回：赋值只更新lua变量值, 不会更新DemoWidget中的Param;

local OutParam1,OutParam2;
local Result;
Result,OutParam1,OutParam2 = DemoWidget:GetDoubleParam(OutParam1,OutParam2)	 --非const引用参数会作为返回结果输出,依次位于return值后面
```

</details>

## 全局API

对于c++提供的全局方法、非UFUNCTION的成员方法、自定义扩展等提供全局库形式的封装。

### 1. LuaGlobalAPI

- **`CreateUIWidget(bp_lua, ClassFullPath)`**
  - 创建UI对象;
  - `bp_lua`: 当前bp_lua对象;
  - `ClassFullPath`: UI类全路径;
- **`CreateUIWidgetWithClass(bp_lua, Class)`**
  - 创建UI对象;
  - `bp_lua`: 当前的bp_lua对象;
  - `Class`: 对象类UClass\*;
- **`LoadObject(ObjectFullPath)`**
  - 加载对象;
  - UE全局API封装：`T* LoadObject(...)`;
  - `ObjectFullPath`：对象全路径;
- **`LoadClass(ClassFullPath)`**
  - 加载类;
  - UE全局API封装：`T* LoadClass(...)`;
  - `ClassFullPath`：类全路径;
- **`IsObject(Object, bEvenNullptr)`**
  - 判断传入值是否为UObject对象;
  - `Object`: 检查的数据;
  - `bEventNullPtr`: 是否将nullptr视作UObject;
- **`GetClassName(Object)`**
  - 获取Object的ClassName;
- **`WeakObjectPtr(Object)`**
  - 创建弱指针对象指向Object;
- **`CheckData(flag)`**
  - 类型C++的check(): SHIPPING版空执行返回true; Dev版本flag为false时弹窗提示返回false，为true返回true;
- **`GetWorldContext(InObj)`**
  - 辅助 WorldContext 参数：需要此参数的地方可使用该接口返回值传入；
  - `InObj`: 指定的WorldContect, 有效则返回InObj, 无效返回一个通用的WorldContext对象(目前为 GameFrontHUD);
- **_`ShowPlatformMessageBox(Title, Message)`_**
  - 调试功能; 显示系统弹窗; 主要用于开发模式下的提示;
  - Shipping版本调用为空逻辑;
- **_`totable(Data)`_**
  - 调试功能; 主要用于将 UStruct/TArray/TMap/TSet 转为table类型，然后打印输出进行数据查看;
  - Dev版本：支持全部结构体/容器的转换;
  - **Shipping版本：仅支持容器的转换**



### 2. UE 库

- **`UE.IsA(Object, Class)`**
  - 检查对象是否为指定类的实例;
  - UObject成员API封装：`bool IsA(OtherClassType SomeBase);`
  - `Object`：对象;
  - `Class`：类型Class;
- `UE.FindObject(UClass, UPackage, ObjectName, bExactClass)`
  - 查找对象;
  - 全局API封装: `UObject* StaticFindObject( UClass*, UObject*, TCHAR*, bool);`
- `UE.FindObjectFast(UClass, UPackage, ObjectName, bExactClass)`
  - 查找对象;
  - 全局API封装: `UObject* StaticFindObjectFast(UClass*, UObject*, FName, bool, ...);`
- `UE.LoadObject(ObjectFullPath)`
  - 等价 `_G.LoadObject`
- `UE.StaticLoadObject(UClass, Outer, ObjectName)`
  - 加载对象
  - 全局API封装: `UObject* StaticLoadObject(UClass*, UObject*, TCHAR*, ...);`
- `UE.LoadClass(ClassFullPath)`**
  - 等价 `_G.LoadClass`
- `UE.GetName(Object)`**
  - 访问对象名字;
  - UObject成员API封装：`FString GetName();`
- `UE.GetFullName(Object)`
  - 访问对象名字;
  - UObject成员API封装：`FString GetFullName();`
- `UE.LuaGetPathName(Object)`
  - 访问对象名字;
  - UObject成员API封装：`FString LuaGetPathName();`
- `UE.GetOuter(Object)`
  - 访问对象Outer;
  - UObject成员API封装：`UObject* GetOuter();`
- `UE.IsValid(Object)`
  - 检查对象是否Valid;
  - 全局API封装：`bool IsValid(const UObject*);`
- `UE.IsObject(Object)`
  - 检查UObject是否为空指针; 辅助 `_G.IsValid()`;
  - `Object`: UObject数据;
- `UE.ToTable(Data)`
  - 将结构体/容器转为table; 辅助 `_G.totable`;

##  重点说明
	
1. （内存）lua 引用的 UObject 变量，请在销毁前手动 set nil，缓存到lua后被UE销毁的对象，在lua访问到的是invalid Object，而不是nil，判断的时候要用UE.IsValid。

```lua
local Actor = SpawnActor(...)
Actor:K2_DestroyActor()
--这里建议销毁后手动设置Actor = nil
if Actor == nil then end   --Actor不为nil
if UE.IsValid(Actor) == false then end  --UE.IsValid(Actor)为false
```
	
2. （功能）lua调用C++方法时默认参数请主动填入，不要省略，除非需要传入的值为`0x0`。

```c++
void CPPFunction(int Param = 1);
```

```lua
CPPFunction();--这里会使用0x0的值，也就是Param会是0
CPPFunction(1); --√
```

5. （功能）totable 仅用于调试及特殊需求，Shipping下仅支持数组容器操作，会导致调试功能和真机运行功能不符。
6. （性能）lua中频繁传递结构体参数到UE4，尽量使用结构体而非table，会有额外的转换开销。

```lua
void CPPFunction(Vector Param);
......
CPPFunction({X=1,Y=2,Z=3});		--×
CPPFunction(Vector.New(1,2,3));	--√
```

8. （性能）请缓存续频繁使用到的数据，每次访问C++/蓝图参数都有额外的查询开销。
9. （性能）lua容器缓存UE管理的c++指针数组时，会存在指针被UE释放掉的情况，从而引发DS崩溃，避免使用Copy的方式对指针数组进行缓存，可以使用ToTable的方式缓存，**<span style="color:red;">以下为错误案例：</span>**
```lua
function BP_Worship:RecordBeginAreaPlayer(CheckActorArray)
	self.TotalBeginAreaPlayer = CheckActorArray.OverlapActorList:Copy()
end
```
