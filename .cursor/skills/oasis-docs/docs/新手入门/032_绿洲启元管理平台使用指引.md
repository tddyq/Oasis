# 绿洲启元管理平台使用指引
绿洲启元编辑器为开发者提供了玩法创作的空间，而 [绿洲启元管理平台](https://gpeditor.qq.com/auth?authsrc=user#myprojects) 则承担了对玩法工程的管理职责，也是玩法创作过程中高频使用的场景，例如真机测试、玩法提审等等。除了对工程的管理之外，账号管理、项目数据、日志提取和玩家反馈等功能也都集成在管理平台上，实现对项目全生命周期的可视化管理。下面将对绿洲启元管理平台做基本介绍，为开发者的日常使用提供指引。
|功能模块|简要说明|
|-|-|
|项目管理|提供项目与工程的关联及版本管理，查看项目收益情况。|
|项目数据|查看玩法的运营数据、性能数据和上报的异常信息。|
|日志提取|提供真机测试期间的DS日志下载功能。|
|玩家反馈|查看玩家的真实反馈信息。|
|账号管理|提供项目关联账号的管理能力，支持子账号。|

<br>

## 项目管理

项目管理模块主要包含两部分：玩法项目列表和项目收益。在玩法项目列表中可以对项目工程进行管理、提审和开启测试等操作；项目收益提供玩法上线后开发者所获得结算收益的信息展示。

### 玩法项目列表

进入项目管理下的 [玩法项目列表](https://gpeditor.qq.com/auth?authsrc=user#myprojects) 页面，可以查看创建过的所有项目列表信息，包括各项目的名称、开发者和签约状态等。一般情况下，当初次进行真机测试或者玩法提审的时候，需要将工程与项目关联，此时可以通过【添加新项目】来创建项目，具体的操作流程可以参考 [玩法提审与更新](https://developer.gp.qq.com/wikieditor/#/catalog/378)。

---

### 项目收益

开发者获得的结算收益情况会展示在 [项目收益](https://gpeditor.qq.com/auth?authsrc=user#operationalData) 页面，可以结合时间筛选条件查询具体的收益明细情况，另外结算详情还提供了相关的开票指引。

<br>

## 项目数据

项目数据模块提供了玩法的运营数据和性能数据视图，开发者需要在不同的时间节点关注相应数据的动态变化，以便及时对玩法进行调整。

### 运营数据

当玩法正式上线后，开发者可以通过 [运营数据](https://gpeditor.qq.com/auth?authsrc=user#operationalData) 页面查看玩法的表现效果。页面上提供了玩法参与率、参与率排序、人均对局数和场均时长等多项关键指标，帮助开发者了解玩法表现的变化趋势。

---

### 性能数据
玩法在灰度期以及正式上线后的性能统计数据会上报到 [性能数据](https://gpeditor.qq.com/auth?authsrc=user#ratingData) 页面中，包含FPS、内存峰值等等多项指标，帮助开发者全面了解玩法的性能表现情况。

---

### 异常上报

[异常上报](https://gpeditor.qq.com/auth?authsrc=user#crash_report) 页面是对 [性能数据](https://gpeditor.qq.com/auth?authsrc=user#ratingData) 中异常统计数据的补充，这里展示了发生异常情况时的具体错误信息和错误堆栈，开发者可以根据堆栈信息来定位和排查具体的问题原因。

<br>

## 日志提取

当开发者进行真机测试时，有时需要通过DS日志来辅助排查问题，但DS日志存储在远端服务器中，为此提供了日志提取的解决方案。

### 项目日志

[项目日志](https://gpeditor.qq.com/auth?authsrc=user#getProjectlog) 页面上的DS日志以game_id命名，可以通过客户端日志查找game_id进而匹配对应的DS日志，有关客户端日志及搜索方法可以参考 [绿洲启元调试说明和技巧](https://developer.gp.qq.com/wikieditor/#/catalog/354) 中的「获取手机日志」部分内容。

> 该功能仅作为玩法开发期间的辅助调试使用，不提供灰度及正式上线后的日志提取。

<br>

## 玩家反馈

来自用户的声音往往能提供有效的反馈信息，开发者可以从反馈中发现潜在的问题，也可以结合用户的建议挖掘出玩法的优化方向。因此，当玩法灰度或正式上线后开发者需要重点关注用户反馈的信息，而 [玩家反馈](https://gpeditor.qq.com/auth?authsrc=user#player_feedback) 页面则提供了获取反馈的途径。

<br>

## 账号管理

玩法提审的前置条件是需要确保项目关联的账号归属于一个团队，[账号管理](https://gpeditor.qq.com/auth?authsrc=user#account) 页面提供了创建团队账号、管理子账号、移交主账号和角色权限修改等核心功能，具体的操作可以参考 [团队帐号使用说明](https://developer.gp.qq.com/wikieditor/#/catalog/380)。
