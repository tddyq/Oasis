# 阶段匹配

常规的 [匹配机制](https://developer.gp.qq.com/wikieditor/#/catalog/127) 依据玩法中对所有队伍玩家总人数的设定，需满足总人数条件才能匹配成局，但是当同时参与匹配的玩家人数不足时，可能出现匹配时间过长，甚至难以成局的现象。

为了优化匹配效率、解决成局率的问题，提供了阶段匹配功能，允许开发者设置多种匹配阶段，通过梯度式动态降低匹配人数条件，提高匹配成局的成功率。

<br>

## 启用阶段匹配

阶段匹配功能同时支持单模式与 [二次匹配](https://developer.gp.qq.com/wikieditor/#/catalog/376) 模式，两种模式下配置方式相同，下面以单模式为例介绍如何启用及设置阶段匹配功能。

点击编辑器菜单栏 ``编辑 -> 工程设置 -> Match Setting`` 打开匹配设置界面。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/a93mdimage.png)

勾选【是否开启阶段匹配】将切换为阶段匹配的设置项，配置形式为多元素数组，每一个元素中的数据单元为一个阶段的成局条件，阶段匹配设置即制定各阶段下允许成局的人数条件。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/ARivqimage.png)

<br>

## 阶段匹配时长

为确保各玩法执行阶段匹配的统一与标准化，各阶段的匹配时长目前由官方统一调控，以下为当前匹配时长规则：
- 第一阶段：5s
- 第二阶段：5s
- 第三阶段：2s
- 第四阶段：2s
- 后续其他阶段：1s

<br>

## 注意事项

- 匹配系统按照各阶段的设置逐次尝试匹配，建议配置上逐渐降低成局人数要求，即 ``队伍数量 * 小队玩家数量`` 梯度式递减才能发挥优化效用
- 启用阶段匹配后，由于玩家可能因符合任何阶段的成局条件而进入对局，因此玩法机制上需要适配不同开局人数，开发者需自行支持相关逻辑
- 单阶段的成局人数设置必须小于20人
- 在 [真机测试](https://developer.gp.qq.com/wikieditor/#/catalog/372) 环境中，为了方便测试加入了30秒的保底策略，即若在30秒内未能成功匹配，则尝试将符合 [匹配设置](https://developer.gp.qq.com/wikieditor/#/catalog/127) 条件的玩家强制拉入对局
