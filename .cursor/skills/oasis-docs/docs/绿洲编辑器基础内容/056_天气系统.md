# 天气系统

天气系统是一个高自由度、可配置化的光照集成系统，可以对关卡地图的全局光照、雾效、光晕等环境光进行设置，以调整整体的氛围效果，目前主要包含定向光源、天空光照、天空大气和高度雾四部分：
- [定向光源](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/directional-lights?application_version=4.27)：模拟太阳光的平行照射光线，属于全局的主光照光源
- [天空光照](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/sky-light?application_version=4.27)：通过捕捉关卡环境为场景提供均匀的光照和反射，以模拟环境光，为场景提供间接光照
- [天空大气](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/sky-atmosphere?application_version=4.27)：基于物理的天空和大气渲染，模拟不同天气、光照条件下的视觉效果
- [高度雾](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/exponential-height-fog-user-guide?application_version=4.27)：模拟现实生活中的雾气效果，增强场景的真实感和氛围感

除了基础的静态光照设置之外，天气系统还提供了天气序列功能，可以让开发者轻松地实现随时间流逝的动态天气效果，以及在多个天气之间来回切换的更精细化控制天气的能力。

<br>

## 使用天气系统

编辑器提供了天气控件蓝图，只需将该蓝图放置在关卡地图中，即可使用天气系统的功能。

>天气控件蓝图路径： 可在资源目录内搜索 ``BP_STExtraSkyBox``

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/WftTVimage.png)

天气系统核心配置模块分为两部分：Planner和Artist。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/wBH7vimage.png)

- Planner：运行时配置，DS启动时将加载该天气配置，对所有玩家都生效，且支持多个天气的加权随机
- Artist：加载指定的天气配置数据资产，支持对配置参数的二次修改及预览
	* TimeOfDay：自然天时间点（0~24）对应的天气
	* WeatherObjects：内置的光照及雾效等组件配置，一般无需修改
	* Preview：视口内预览天气序列效果的配置区，可通过MinTime及MaxTime指定预览的时间区间，并可调节播放速率和进度
	* Parameters：可以设置光照、雾效、大气等基础配置参数，同时支持曲线的方式配置自然天内（24小时）的天气动态变化参数（时变曲线将覆盖基础参数的效果）

> 天气系统集成了定向光源、天空光照、天空大气和高度雾的功能和设置，属于对天空盒系统的功能升级，因此如果关卡中存在旧版本的普通天气Actor可以全部删除，包括：Sky_Sphere、SkyLight、DirectionalLight、ExponentialHeightFog及各类PostProcess后处理盒，以免造成配置效果的冲突。<br>

### 使用预制的天气效果

编辑器内置了一批丰富的预制天气资源可供使用，天气资源以Asset数据资产的形式存储相关配置，只需在 ``SaveLoadSourceAsset`` 属性中选取你希望使用的Asset资源即可。

## 天气序列

除了单天气数据资产配置，天气控件蓝图也支持加载天气序列数据，多种天气组合出更丰富、变幻莫测的环境效果，既可以模拟随时间平滑变化的天气环境，也可以根据玩法规则动态切换天气效果。

### 创建与配置天气序列

**1. 创建子天气**

由于天气序列旨在将不同的天气组合起来，因此需要前置配置若干子天气数据，可参考上文 [基础天气](/基础天气/) 部分进行配置。

**2. 创建天气序列**

在工程Asset任意目录下右键：各种各样->数据资源，创建一个数据资产，资源类别选择 ``WeatherSequence``。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/2WKNQimage.png)

**3. 配置天气序列**

**基础序列配置**

天气序列以一个KeyFrame作为单天气的播放周期，而不同天气之间以 ``KeyFrame -> Link -> KeyFrame`` 的形式进行交叉组合，因此需要配置序列中各KeyFrame，以及两个KeyFrame之间的Link。

![企业微信截图_17216489169325.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Civme%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_17216489169325.png)

KeyFrame配置：
- Style：子天气数据资产
- StartTime：该天气启动的时间，以上一个子天气持续时间和过渡时间进行自动计算，无需主动填写
- HourOfDay：该天气所处的时间点（0-24），为取得最好的效果，建议将此值配置在9-18之间
- Duration：该天气持续的时间（单位：秒）

Link配置：
- BlendTime：进入下一个天气前的过渡时间
- BlendCurve：过渡曲线，不配置则默认为线性过渡

> 虽然天气序列提供了多个KeyFrame插槽，但并不代表开发者必须配置多个KeyFrame，也可以只配置一个KeyFrame来做成单天气效果，只需将Duration参数尽可能调大即可。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/oC1qnimage.png)

**时间轴特效**

除了基础的天气变换，还支持配置一些已经定制好的屏幕特效，用以加强天气氛围感，例如：沙尘暴或者下雨等。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/rQ76limage.png)
![企业微信截图_17216496265419.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/oHzjI%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_17216496265419.png)

- StartTime：屏幕特效开始时间
- EndTime：屏幕特效结束时间
- ScreenAppearancePaths：指定的屏幕特效资源
	- 启用低画质特效：在性能较差的设备上或者低端画质设置下，启用该配置将会读取低画质特效，以保证游戏流畅性
	- LowQualityScreenAppearancePaths：配置的低画质屏幕特效资源

> **注意：** PIE视口不支持预览屏幕特效

**多天气优先级**

开发者可以在玩法内一次性加载多个天气序列，通过指定天气序列优先级可以调整播放时的顺序，且播放时不会卸载上一个天气序列。

**启用循环**

勾选此项将循环播放该天气序列。

**材质混合时长**

若相邻的KeyFrame天气对应的材质不同，则在指定的时间跨度内完成材质的混合。

### 使用天气序列

天气系统提供了两种天气序列的使用方式：全局天气序列和动态天气序列。

> 不同天气序列使用方式以 ``多天气优先级`` 的设置来决定覆盖关系；若优先级相同，则以最近加载的天气序列进行覆盖

<br>

**全局天气序列**

找到关卡中的天气控件蓝图，在 ``Planner`` 模块下通过 ``WeatherSequence`` 属性配置目标天气序列资产，这种方式将在DS启动时加载并播放序列，因此对所有玩家都生效，达到全局统一的天气效果。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/ZzfrIimage.png)

此外，天气控件支持加载多个天气序列，通过 ``weight`` 来调整各个天气序列在DS启动时的出现权重，游戏开始时将会按权重随机出一个序列，并自动为所有玩家加载此天气序列，达到随机天气的效果。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/xm7U1image.png)

**动态天气序列**

天气系统提供了加载/卸载天气序列的API，让开发者可以在玩法逻辑中动态加载不同的天气序列，为玩家构建更个性化的天气环境，丰富玩法元素。

天气序列相关API：

``` lua
-- 加载天气序列
-- 生效范围：S
-- @param PlayerController 玩家的PC
-- @param WeatherSequence 目标天气序列资源
UGCWeatherSystem.LoadWeatherSequence(PlayerController, WeatherSequence)

-- 卸载天气序列
-- 生效范围：S
-- @param PlayerController 玩家的PC
-- @param WeatherSequence 目标天气序列资源
UGCWeatherSystem.UnloadWeatherSequence(PlayerController, WeatherSequence)
```

代码示例：

``` lua
-- 加载天气序列
function UnitTest.S_LoadWeatherSequence(Tester, param)
    if param == "" then 
        param = "Asset/TestSequence.TestSequence"
    end
    local Path = UGCMapInfoLib.GetRootLongPackagePath()..param
    local WeatherSequence = LoadObject(Path)
    if WeatherSequence == nil then 
        return Path.." is not a valid asset path"
    end
    UGCWeatherSystem.LoadWeatherSequence(Tester, WeatherSequence)
    return "ok"
end

-- 卸载天气序列
function UnitTest.S_UnloadWeatherSequence(Tester, param)
    if param == "" then 
        param = "Asset/TestSequence.TestSequence"
    end
    local Path = UGCMapInfoLib.GetRootLongPackagePath()..param
    local WeatherSequence = LoadObject(Path)
    if WeatherSequence == nil then 
        return Path.." is not a valid asset path"
    end
    UGCWeatherSystem.UnloadWeatherSequence(Tester, WeatherSequence)
    return "ok"
end
```
