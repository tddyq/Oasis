# 超大地图关卡加载方式

#  超大地图关卡加载方式：World Composition（基于大地图模板建立的地图）
**World Composition：** 和平大地图模板地图加载方式（海岛、沙漠、雨林、雪地、山谷）。

<br>

## 超大地图加载（World Composition）

### 1.  什么是超大地图加载：

- World Composition。适用于超大地图多关卡管理，这种模式下的关卡加载方式可以大大节省内存的占用；
- 激活World Compositiom加载后，该关卡就变成了Persistent,并且, 会遍历该关卡所在目录及其子目录的所有关卡文件, 将他们作为子关卡自动添加到该关卡下。

### 2.  激活方式：
- 可通过菜单栏中“设置”→“世界设置”窗口进行管理，勾选Enable World Composition<br>

### 3.  子关卡的增减

- A.常规编辑：创建 编辑 保存都通用基本操作<br>
- B.模板原有子关卡：<span style="color: #ff0000">**和平大地图模板中原有的子关卡，都不可单独编辑修改**，</span>但是可以选择不加载某一关卡。不想加载的子关卡需要填入项目设置的黑名单中<br>

### 4.  子关卡的加载方式

World Compositiom的模式下，子关卡有两种加载方式：按距离加载&蓝图加载
- 设置子关卡加载方式：关卡编辑窗口选中要编辑的关卡，选择“分配给层”，会出现关卡加载方式选项<br>
![企业微信截图_16866492696915.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16866492696915.png)

- 按距离加载：可设置不同的加载距离，当玩家和该关距离满足条件时，关卡就会被加载
  自定义关卡加载距离：在关卡面板中点击“世界构成”，然后可以按着自己需求新建不同距离的加载层级，
  Streaming distance:100=1M<br>
![企业微信截图_16866492826732.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16866492826732.png)

- 蓝图加载：把关卡加载方式设置为LoadByBP,然后自己写加载逻辑
  范例：场景中摆放一个名叫“LoadLevel” 的触发器actor，当有玩家进入该触发器范围的时候，就加载名为“NewMap01”的关卡，并且显示“NewMap01”关卡下的所有内容。玩家退出触发器范围后，卸载掉“NewMap01”关卡，示例如下：
	
```lua
local LoadLevel = {}

function LoadLevel:ReceiveBeginPlay()
    ugcprint("LoadLevel:ReceiveBeginPlay");
    if self:HasAuthority() == true then
        self.Box.OnComponentBeginOverlap:Add(self.OnBeginOverlap, self);
        self.Box.OnComponentEndOverlap:Add(self.OnEndOverlap, self);
    end
end


function LoadLevel:OnBeginOverlap()
    GameplayStatics.LoadStreamLevel(self, "NewMap01", true, false)
end

function LoadLevel:OnEndOverlap()
    GameplayStatics.UnloadStreamLevel(self, "NewMap01")
end
```
