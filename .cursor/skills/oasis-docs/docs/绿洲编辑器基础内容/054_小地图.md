#  小地图

游戏中的小地图可以让玩家快速了解周围的地形、队友位置和敌人动向，使玩家得以了解战场信息、制定合理的战术策略。

绿洲启元编辑器提供了小地图自动化生成工具，帮助开发者快速构建小地图功能，该工具通过触发器生成一张全局的地图贴图，然后将该贴图的信息导入小地图系统，实现显示玩家的地理位置及变化的效果。开发者在生成贴图时可以选择性过滤部分场景建筑，如宝箱、陷阱等，也可以导出贴图进行二次美化。

<br>

## 生成小地图

首先在场景中放置一个 ```触发器体积组件``` （trigger volume），调整触发器的大小以覆盖需要生成小地图的完整场景，如果对整张关卡图生成小地图，则和 ```level bounds``` 大小一致。

> 触发器体积组件默认单边长200厘米，建议按正方体进行等比缩放

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/yx18pimage.png)

在世界浏览器中搜索选中 ```触发器体积组件```，为其添加一个名为“minimap”的Tag标签：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pnaq3image.png)

点击编辑器菜单栏 ``窗口 -> 小地图``，调出小地图生成器：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/bURW1image.png)

点击【生成】按钮，系统将会根据触发器的覆盖范围自动生成小地图贴图，贴图会默认保存在项目工程的根目录下。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/QdtC1image.png)

如果希望场景中的一些建筑或者物品不要在小地图中生成，或者替换成其他显示样式，可以在小地图生成器的【设置】中进行配置，例如将场景中的“海岛物件15”添加入隐藏数组中，小地图贴图生成时将会忽略此物件。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/DkgpLimage.png)

## 小地图美化

如果对小地图需要进行更多标记或者效果修改，可以将生成的小地图贴图资源导出，在图片编辑软件中进行美化，而后再导入编辑器。

**1. 导出小地图**

调出小地图生成器，点击【导出】按钮，可以将小地图资源导出到外部目录。
> 保存时的贴图文件名称需要与工程名保持一致

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pAmvPimage.png)

**2. 导入编辑器**

在外部对小地图完成美化后，调出小地图生成器，点击【导入】按钮，选择要导入的图片，即可覆盖旧的小地图。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Al8y7image.png)

小地图导入后可以按需设置大小，双击打开小地图贴图，在属性搜素框内搜素 ```texture``` 关键词，``MaximumTextureSize`` 为小地图精度属性，一般情况下设置1024即可，修改完点击保存。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/rqtUpimage.png)

## 配置小地图

### 添加BP_MiniMapStandardPoint蓝图

``BP_MiniMapStandardPoint`` 蓝图是游戏中实时显示小地图的必备条件，在资源目录中搜索“BP_MiniMapStandardPoint”找到该蓝图，并将其放置在关卡场景中。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/YoQEUimage.png)

设置 ```BP_MiniMapStandardPoint``` 蓝图的世界坐标，确保其与生成小地图的 ```触发器体积组件``` 世界坐标一致
> 蓝图实例的旋转值Z轴一定要设置为-90

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/M9Ilqimage.png)

- Level Bound Extent：关卡的边界大小（单位：厘米），一般情况下与 ``Each Tile Extent`` 保持一致
- Each Tile Extent：用于生成小地图的 ```触发器体积组件``` 的边长（单位：厘米），以60倍缩放为例，边长即为12000厘米
- Level Name：关卡地图的名称

---

### 小地图缩放比例

编辑器菜单栏点击 ``编辑 -> 工程设置``，在小地图配置项中设置以下参数：

- 缩放比率：表示右上角小雷达图视角显示时，对于小地图图片文件的缩放比率，最小配100
- 滑块缩放比率：表示小地图拖动滑块时的缩放比率
- Map Image Path：默认填写1

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/12Llvimage.png)

---

### 小地图功能参数

小地图的战争迷雾、图例等功能设置集成在 [GamePart模块](https://developer.gp.qq.com/wikieditor/#/catalog/20090?autoJump=%E5%90%AF%E7%94%A8%E6%A8%A1%E5%9D%97%E5%8C%96%E5%8A%9F%E8%83%BD) 中，点击编辑器菜单栏的【玩法通用设置】按钮，打开 ``DA_GameModeGeneral`` 数据资产蓝图，启用“UGCMinimap_Game_Part_Config”模块功能。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/5S2qhimage.png)

展开GamePart配置项，小地图的功能参数分为基础信息、细节配置和图例配置三部分。

**基础信息**

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/goheuimage.png)

- 默认地图ID：进入游戏时默认使用的小地图ID，该ID必须为细节配置中的有效ID
- 图例最大显示数量：小地图最多可显示的全局图例数量上限
- 图例显示范围(半径)：图例对象与玩家距离小于该值才显示图例（单位：厘米）

**细节配置**

支持为玩法配置多组小地图，而细节配置是每一组小地图的具体配置信息。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pn8rdimage.png)

- 地图ID：小地图的唯一标识ID
- 缩放比率：该小地图的实际缩放比率，会覆盖工程设置中的比率设置
- 地图中心点：该小地图的中心点位置，必须与 ``触发器体积组件`` 的世界坐标一致
- 地图真实尺寸：该小地图的大小，必须与 ```BP_MiniMapStandardPoint``` 蓝图的关卡边界大小一致
- 地图文件路径：若配置，将覆盖小地图默认的底图
- 是否开启迷雾：该小地图是否开启迷雾效果
	- 队伍共享迷雾区域：全队是否共享迷雾视野，即队友探索过的迷雾区域是否也会同步给自己
	- 消散时间：迷雾被探索驱散后重新聚合的时间，-1则永久不消散
	- 探索半径：迷雾的探索半径（单位：厘米）

**图例配置**

把静态或者动态对象指定为图例后，该对象将以指定的样式和规则显示在小地图上，每种图例需要通过Key与具体配置关联，该Key值是需要启用图例功能的蓝图类。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/xeKreimage.png)

- 图标：图例对应的图标icon
- 优先级：图例的优先级，当小地图上显示图例数量超过 ``最大显示数量`` 时，会根据优先级排序筛选需要显示的图例
- 最大显示数量：该类型图例的最大显示数量
- 是否动态：该类型图例是否会动态改变位置，静态对象不建议勾选，可能影响性能
- 子类是否生效：派生蓝图类是否也需要启用图例功能
- 是否常显：若常显，则该类型图例不会出现在图例筛选列表里
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/0bf8Dimage.png)
- 是否可以点击：该类型图例是否支持点击，若支持，则在点击后会显示图例详情面板
	- 标题：图例详情面板的标题文本
	- 详情图：图例详情面板的大图，启用点击后生效
	- 描述：图例详情面板的描述文本，启用点击后生效
	- 点击触发行为：图例详情面板的额外点击行为配置项，启用点击后生效
		- 无行为：无任何点击行为
		- 传送：点击后将传送到图例对象的对应位置
		- 自定义：点击触发行为由开发者自定义实现
- 边框颜色：图例的边框颜色

**自定义图例点击行为**

打开图例对象的蓝图，点击菜单栏【类设置】按钮切换到类设置的细节面板。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/QAaFDimage.png)

在 ``接口`` 属性组下，点击【添加】按钮并搜索“UGCActorMarkExtraActionInterface”，为蓝图添加该接口。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/7EQRCimage.png)

打开蓝图的Lua脚本，添加并实现 ``OnMapMarkShortcutButtonClicked`` 函数，即可在点击该图例时触发此函数逻辑。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/J1BhPimage.png)

完成以上配置后，调试进入游戏，客户端界面右上角即会显示小地图，点击小地图会按比例展开大地图。

![QQ2026123-145158-HD-ezgif.com-video-to-gif-converter.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/MD53fQQ2026123-145158-HD-ezgif.com-video-to-gif-converter.gif)

## 多地图切换

小地图生成器默认显示 ``默认地图ID`` 中指定的小地图，如果玩法创建了多张小地图，且需要切换小地图，则需要于脚本合适的时机中调用 [``ChangeMapByMapID``](https://developer.gp.qq.com/api/#/searchContent/UGCWidgetManagerSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2FUI%20%E7%95%8C%E9%9D%A2%2FUGCWidgetManagerSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCWidgetManagerSystem&autoJump=ChangeMapByMapID) 函数进行切换。
