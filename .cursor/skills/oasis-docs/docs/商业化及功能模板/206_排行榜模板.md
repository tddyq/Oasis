# 排行榜模板

游戏中通常利用基于玩家等级、能力、积分等不同维度的排名机制，给与玩家明确的竞争目标，以此提升玩家的留存与粘性，是最重要的系统功能之一，排行榜模板支持按指定数据维度实时排名，同时提供了一套标准的榜单样式模板。

> 排行榜模板已经历过重构优化，不再向后兼容，开发者如果曾经下载过旧版本，需从 [内容浏览器](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/124?autoJump=%E5%86%85%E5%AE%B9%E6%B5%8F%E8%A7%88%E5%99%A8) 右键删除 ``ExtendResource`` 目录下对应的模板文件夹，再下载最新版本并按照本文档配置使用

<br>

## 模板结构

排行榜模板支持自定义多种榜单、设置榜单起止时间、查看历史榜单数据、配置排名奖励及玩家隐私设置等能力。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/lYjsqimage.png)

**A. 榜单名称**

支持配置多种榜单，不同榜单通过页签的方式进行切换

- 榜单名称限制：5个字以内
- 页签数量限制：启用状态的榜单才会进入页签，最多同时启用10个榜单
- 榜单数量限制：最多可配置30个榜单

**B. 排序名称**

参与排序的数据名称，例如：击杀数、积分、通关时间等

**C. 上榜玩家**

进入榜单的玩家排名及昵称信息，可配置允许上榜的人数

- 上榜人数限制：10~200，即前十名~前200名玩家可上榜
- 前三名固定展示金牌/银牌/铜牌标识，其余名次展示排名序号
- “我的”模块展示玩家本人数据，若玩家上榜则显示名次，且在榜单内高亮显示；否则显示“未上榜”

**D. 榜单时间**

榜单的有效时间，默认为配置的 **开始时间** 至 **结算时间**，且榜单支持按日榜/周榜/月榜重置。

- 日榜：每天0点重置，展示形式：2024.10.01 00:00:00 - 2024.10.01 23:59:59
- 周榜：每周一0点重置，展示形式：2024.09.30 00:00:00 - 2024.10.06 23:59:59
- 月榜：每月1号0点重置，展示形式：2024.10.01 00:00:00 - 2024.10.31 23:59:59

**E. 榜单说明**

配置此榜单的具体说明内容，无字数限制，点击将弹窗展示完整内容

**F. 历史榜单数据**

仅 **重置型榜单** 支持查看历史数据功能：

- 若当前榜单为第一期，则没有历史数据，将默认隐藏该按钮
- 历史榜单数据跟随榜单的重置时间同步刷新

**G. 隐私设置**

模板提供的玩家匿名上榜功能，默认非匿名上榜，由玩家自行设置。

![企业微信截图_17317476918749.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Qibfn%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_17317476918749.png)

**H. 排名奖励**

开发者可以为上榜玩家按排名区间提供不同的奖励

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/jtMAAimage.png)

- 每个排名区间最多可以配置5种奖励
- 对于非重置型榜单，玩家可领取奖励的时间为榜单的 **结算时间** 至 **结束时间** 之间，逾期无法补领
- 对于重置型榜单，玩家通过 ``历史榜单数据`` 界面领取奖励，且根据重置类型历史榜单将额外保留展示一天/一周/一个月

## 配置依赖

排行榜功能的运行依赖于 ``RankingListComponent`` 组件 和 ``GamePart`` 的特定配置，需要前置将其正确配置好。

### GamePart

排行榜功能已做了 [GamePart模块化](https://developer.gp.qq.com/wikieditor/#/catalog/20090) 封装，因此需要启用“GP_RankingListManager”模块功能。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/319gjimage.png)

- Request Time Interval：DS定时向后台请求最新分数及排名的时间间隔，期间榜单数据非实时数据（单位：秒）
- Update Time Interval：DS定时向后台发起更新分数的时间间隔，期间会积累待更新的请求，依据每次调用更新请求的类型（覆写式/累加式），统一计算本批次待更新的最终值并发送给后台，触发更新前从DS获取到的分数非最新值（单位：秒）

	- 百分比：分数显示为xxx%

**注意：**
> 不支持对已启用且已产生有效数据的榜单做删除或表格的任意修改等操作

### 榜单时间配置

榜单的有效时间由开始时间和结束时间决定，而根据重置类型的不同结算形式和时间有所差异。

![企业微信截图_17317560436223.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/j8KVK%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_17317560436223.png)

**周期类型**

- 每日重置：榜单每天凌晨00:00重置，重置后玩家可前往上期榜单领取前一天的排名奖励
- 每周重置：榜单每周一凌晨00:00重置，重置后玩家可前往上期榜单领取前一周的排名奖励
- 每月重置：榜单每月1号凌晨00:00重置，重置后玩家可前往上期榜单领取前一个月的排名奖励
- 不重置：固定周期的榜单，根据配置的结算时间进行排名结算

> 非重置型榜单的有效时间不能超过365天

**开始时间**

开始计榜的时间，开始时间必须早于结算时间和结束时间。

**结算时间**

停止计榜的时间，到达该时间后会最后刷新一次排名，并锁定排名结果，如果配置了排名奖励，则玩家可以开始领取奖品。所有榜单的结算时间都必须早于结束时间。
> 结算时间仅针对非重置型榜单生效

**结束时间**

超出该时间后，榜单将被隐藏，玩家无法再领取排名奖励，对于重置型榜单有不同的配置规则：

- 日榜：结束时间必须是任意天的23:59:59，到达结束时间后，榜单展示额外延长一天，期间玩家仍可领奖
- 周榜：结束时间必须是任意周日的23:59:59，到达结束时间后，榜单展示额外延长一周，期间玩家仍可领奖
- 月榜：结束时间必须是任意月末（30或31号）的23:59:59，到达结束时间后，榜单展示额外延长一个月，期间玩家仍可领奖

## 排行榜调试

> 目前排行榜在移动端的测试，暂只支持【测试上传】的方式

模板内置了一个简易的调试控件，方便开发者通过可交互的方式进行功能调试，此外开发者也可以结合提供的API进行更复杂的排行榜功能验证。
> 控件蓝图路径：ExtendResource/RankingList/OfficialPackage/Asset/RankingList/Blueprint/WBP_RankingListBtn

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/qdU44image.png)

将控件通过 [``AddToViewport``](https://developer.gp.qq.com/api/#/searchContent/UUserWidget?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUUserWidget.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UUserWidget&autoJump=AddToViewport) 添加到屏幕视口中，启动PIE调试后出现调试的功能按钮。

```lua
local RankListBtnClass = UE.LoadClass(UGCMapInfoLib.GetRootLongPackagePath().. "ExtendResource/RankingList/OfficialPackage/Asset/RankingList/Blueprint/WBP_RankingListBtn.WBP_RankingListBtn_C");
local PlayerController = STExtraGameplayStatics.GetFirstPlayerController(self);
local RankListBtn = UserWidget.NewWidgetObjectBP(PlayerController, RankListBtnClass);
RankListBtn:AddToViewport(1000);
```

输入玩家UID、排行榜ID及分数，点击【更新排行榜分数】按钮即会将玩家分数添加到榜单中，通过多次更新分数操作以验证排行榜数据的准确性。

## API参考

榜单领取的奖励本质为虚拟物品，可以通过 ``VirtualItemManager`` 提供的功能函数进行管理。

|函数库/类名|函数类型|函数功能范围|
|-|-|-|
|[RankingListManager](https://developer.gp.qq.com/api/#/searchContent/RankingListManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FRankingListManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=RankingListManager)|对象函数|请求/更新玩家分数及获取玩家信息|
|[VirtualItemManager](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager)|对象函数|虚拟物品的添加、删除、查询等数据管理|

模板中提供了部分额外的封装函数可供调用，需主动引用 ``RankingListManager`` 类文件：

``` lua
UGCGameSystem.UGCRequire("ExtendResource.RankingList.OfficialPackage.Script.RankingList.RankingListManager")
```
