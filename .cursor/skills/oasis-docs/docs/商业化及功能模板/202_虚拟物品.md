# 虚拟物品

# 虚拟物品

[商业化系统](https://developer.gp.qq.com/wikieditor/#/catalog/20008) 引入了物品与商品的概念，商品是物品被附加了商业化属性后的特殊形式，因此物品是商业化功能中的基础要素，当玩家在游戏中产生物品数据的交互时，通过虚拟物品管理物品数据的流转变化。

<br>

## 虚拟物品概述

不同于 [物资系统](https://developer.gp.qq.com/wikieditor/#/catalog/20101) 中具备场景具象化、可与背包/仓库交互特征的实体物品，基于 [物品表](https://developer.gp.qq.com/wikieditor/#/catalog/20008?autoJump=%E7%89%A9%E5%93%81%E8%A1%A8%EF%BC%88UGCObject%EF%BC%89) 定义创建的物品仅有概念层面的业务含义，没有与场景和背包的数据交互，因此属于一类虚拟物品。

使用物品表的功能都涉及到虚拟物品的应用，例如 [商城模板](https://developer.gp.qq.com/wikieditor/#/catalog/20026) 中购买商品后即以虚拟物品的形式获得该物品，又如 [抽奖模板](https://developer.gp.qq.com/wikieditor/#/catalog/20027) 中抽取的奖品也是以虚拟物品的形式发放到玩家账户等等，这些功能通过 ``物品ID + 数量`` 的表结构形式表示虚拟物品的实例数据，同时虚拟物品与背包物品之间也允许存在单向的映射关系。

## 虚拟物品管理

由于虚拟物品应用场景的通用性，对虚拟物品的功能进行了 [GamePart模块化](https://developer.gp.qq.com/wikieditor/#/catalog/20090) 封装，开发者可以调用虚拟物品GamePart提供的API实现对虚拟物品数据的管理。

### 启用虚拟物品GamePart

在编辑器菜单栏上点击【玩法通用设置】按钮，在 ``DA_GameModeGeneral`` 数据资产蓝图的 ``Active Game Part Configs`` 属性下添加“GP_VirtualItemManager”模块，以启用虚拟物品的模块功能。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/2FpkJimage.png)

---

### 管理虚拟物品数据

虚拟物品GamePart通过 [``VirtualItemManager``](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager) 类函数提供数据管理的API，与常规的实体物品一样支持添加、查询、扣除等基础操作，有所区别的是虚拟物品的数据默认支持持久化存储，因此玩家拥有的虚拟物品可以跨对局存在。

以添加虚拟物品场景为例，调用 [``AddVirtualItem``](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager&autoJump=AddVirtualItem) 可以给玩家发放指定的虚拟物品，通过 [``GetOwnedVirtualItems``](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager&autoJump=GetOwnedVirtualItems) 可以查询玩家当前所拥有的所有虚拟物品。

**代码示例**

```lua
-- DS发放虚拟物品
local GamePartActor = UGCGamePartSystem.GetGamePartGlobalActor("VirtualItemManager");
if GamePartActor then
	GamePartActor:AddVirtualItem(self, 1001, 3);
end

-- 客户端查询拥有的虚拟物品数据
local GamePartActor = UGCGamePartSystem.GetGamePartGlobalActor("VirtualItemManager");
if GamePartActor then
	local OwnedItems = GamePartActor:GetOwnedVirtualItems()
  log_tree(OwnedItems)
end 
```

## 虚拟物品应用

### 功能模板中的虚拟物品

虚拟物品被广泛应用在使用了物品表的各类功能模板中，例如商城中购买或者签到获取的礼包，即是以虚拟物品的形式发放到玩家账户中，而礼包打开后获得的子物品也同样是虚拟物品，因此通过功能模板获得的物品默认都持久存储在玩家的账户中，需要通过 [``VirtualItemManager``](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager) 提供的API管理玩家的虚拟物品数据。

> 商城模板使用的GamePart模块默认引用了虚拟物品依赖，因此可以理解为从商城购买获得的物品实质均为虚拟物品

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/UQ2p7image.png)

### 虚拟物品转换背包物品

> <span style="color:red;font-weight:bold;">当前版本该功能存在bug，在背包满格的时候无法正常转换数据，建议开发者临时通过 [``RemoveVirtualItem``](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%95%86%E4%B8%9A%E5%8C%96%E4%B8%8E%E5%8A%9F%E8%83%BD%E6%A8%A1%E6%9D%BF%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager&autoJump=RemoveVirtualItem) 和 [``AddItemV2``](https://developer.gp.qq.com/api/#/searchContent/UGCBackpackSystemV2?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E7%89%A9%E5%93%81%E4%B8%8E%E8%83%8C%E5%8C%85%2FUGCBackpackSystemV2.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCBackpackSystemV2&autoJump=AddItemV2) 组合实现等同效果，等后续版本修复后再启用</span>

当功能模板中的物品涉及物资系统的物品时，存在虚拟物品向背包物品数据转换的需求场景。例如每日签到领取枪械，为实现该功能开发者需要监听获得虚拟物品的委托，调用 [``AddItemV2``](https://developer.gp.qq.com/api/#/searchContent/UGCBackpackSystemV2?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUGCBackpackSystemV2.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCBackpackSystemV2&autoJump=AddItemV2) 发放枪械物品，同时触发 [``OnAddItemV2``](https://developer.gp.qq.com/api/#/searchContent/UBackpackComponentV2?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUBackpackComponentV2.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UBackpackComponentV2&autoJump=OnAddItemV2) 回调后，再通过 [``RemoveVirtualItem``](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager&autoJump=RemoveVirtualItem) 移除对应的虚拟物品。为了简化这类场景的实现，提供了配表的转换方式。

**创建物品映射表**

当启用虚拟物品GamePart时，会自动在 ``Asset/Data/Table`` 目录下生成名为“UGCObjectMapping”的物品映射表：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Sd2udimage.png)

如果缺失该映射表，可以通过编辑器菜单栏 ``窗口 -> Create System Default Table -> Create VirtualItem Table`` 重新创建物品表与映射表；或者在 ``Asset/Data/Table`` 目录下右键 ``各种各样 -> 数据表格`` 基于 ``UGCVirtual Item Mapping Data`` 结构体创建该映射表。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/fQWpuimage.png)
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/g2qqSimage.png)

**配置物品映射关系**

打开 ``UGCObjectMapping`` 映射表，分别填入 [物品表](https://developer.gp.qq.com/wikieditor/#/catalog/20008?autoJump=%E7%89%A9%E5%93%81%E8%A1%A8%EF%BC%88UGCObject%EF%BC%89) 中定义的物品ID与对应的物资系统中的 [物品ID](https://developer.gp.qq.com/wikieditor/#/catalog/20101?autoJump=%E5%9F%BA%E7%A1%80%E9%85%8D%E7%BD%AE)，系统将为任意途径获取的虚拟物品自动完成向背包数据的转换，相关物品将进入背包，且持久化存储该物品，对虚拟物品的数量更新、移除等操作也会自动同步至背包物品。

> 虚拟物品与背包物品的映射转换为单向1:1的绑定关系，即只存在虚拟物品转为背包物品，且一种虚拟物品对应一种背包物品

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/4er0zimage.png)
