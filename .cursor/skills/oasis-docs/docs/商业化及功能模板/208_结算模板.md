# 结算模板

结算模板是为游戏开发者提供的一套完整的对局结算解决方案，包含结算界面展示、数据统计、胜负判定、结算动画等核心功能。该模板支持单挑、小队对抗、小队排名板和个人排名板四种主流结算模式，帮助开发者快速实现游戏结算系统。

<br>

|属性名称|说明|
|:-:|-|
|Game Mode Name|游戏模式名，对应结算界面的“玩法标题”|
|Scoreboard Type|结算界面类型，决定结算板的整体样式，可选Duel(单挑)、SquadDuel(小队对抗)、TeamLeaderBoard(小队排名板)或SoloLeaderBoard(个人排名板)|

### 结算数据配置

结算界面的数据需通过调用BeginSettlement接口进行传入。主要包含以下两个数据表：PlayerResults（玩家结算数据）和 TeamResults（队伍结算数据）。

#### PlayerResults（玩家结算数据）

PlayerResults的类型为table，每一项对应一名玩家的全部结算数据。每项数据包含PlayerKey和Results，PlayerKey的类型为number，对应该玩家的Key；Results的类型为table，保存该玩家的各项结算数据，其中数据的类型均为string。

![企业微信截图_17592180107823.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Oouei%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_17592180107823.png)

Results中的键名必须与蓝图“BP_GameResult”中【Scoreboard Headers】、【Player Result Headers】和【Player Info Result Headers】属性组内配置的Key字段相匹配，以确保数据能被正确读取并显示。

![结算模板绘图-数据2.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/wKllq%E7%BB%93%E7%AE%97%E6%A8%A1%E6%9D%BF%E7%BB%98%E5%9B%BE-%E6%95%B0%E6%8D%AE2.drawio.png)
![结算模板绘图-数据.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pwo9I%E7%BB%93%E7%AE%97%E6%A8%A1%E6%9D%BF%E7%BB%98%E5%9B%BE-%E6%95%B0%E6%8D%AE.drawio.png)

若【Scoreboard Type】选择了TeamLeaderBoard，PlayerResults表应按玩家排名从高到低进行排列，即排名第一的玩家结算数据置于首位，排名最低的玩家结算数据置于末位。

---

#### TeamResults（队伍结算数据）

TeamResults的类型为table，每一项对应一个队伍的结算数据。每项数据包含TeamID和DeterminingResult，TeamID的类型为number，对应该队伍的ID；DetermingResult的类型为string，为队伍胜负判定数据，该字段的值将显示在结算界面的“胜负判定依据”区域。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/sN9lkimage.png)
![结算模板绘图-位置演示.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/XksFx%E7%BB%93%E7%AE%97%E6%A8%A1%E6%9D%BF%E7%BB%98%E5%9B%BE-%E4%BD%8D%E7%BD%AE%E6%BC%94%E7%A4%BA.drawio.png)

根据【Scoreboard Type】的不同，TeamResults的传入数据规则如下：
- 若【Scoreboard Type】为Duel或SquadDuel，且调用BeginSettlement时bDraw参数设为false，则将获胜方数据置于第一项，失败方数据置于第二项
- 若【Scoreboard Type】为TeamLeaderBoard，TeamResults表应按队伍排名从高到低进行排列，即排名第一的队伍结算数据置于首位，排名最低的队伍结算数据置于末位
- 若【Scoreboard Type】为SoloLeaderboard，队伍结算数据须为空。

## 结算调试

在玩法进程中，可以调用BeginSettlement开始进行结算，调用EndSettlement结束结算。

以下为使用示例，界面设计如下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/a9ZqRimage.png)

为按钮添加对应功能逻辑，部分代码如下：

``` lua
--Main.lua部分代码

--UI控件的初始化函数
function MainUI:Construct()
	--给按键绑定点击事件回调
	self.ButtonBeginSettlement.OnClicked:Add(self.ButtonBeginSettlementOnClicked, self);
	self.ButtonEndSettlement.OnClicked:Add(self.ButtonEndSettlementOnClicked, self);
end

function MainUI:ButtonBeginSettlementOnClicked()
	--获取PlayerController
	local PlayerController = UGCGameSystem.GetLocalPlayerController();
	--调用PlayerController的ServerRPC函数
	UnrealNetwork.CallUnrealRPC(PlayerController, PlayerController, "BeginSettlement");
	return nil;
end

function MainUI:ButtonEndSettlementOnClicked()
	--获取PlayerController
	local PlayerController = UGCGameSystem.GetLocalPlayerController();
	--调用PlayerController的ServerRPC函数
	UnrealNetwork.CallUnrealRPC(PlayerController, PlayerController, "EndSettlement");
	return nil;
end

--UGCPlayerController.lua部分代码

--接收RPC调用
function UGCPlayerController:GetAvailableServerRPCs()
	return
  "BeginSettlement",
  "EndSettlement"
end

function UGCPlayerController:BeginSettlement()
	--获取所有的PlayerController
  local AllPlayerController = UGCGameSystem.GetAllPlayerController();

  --构造玩家结算数据
  local PlayerResults = {
		--第一位玩家的数据
		{
			PlayerKey = UGCGameSystem.GetPlayerKeyByPlayerController(AllPlayerController[1]);
			Results = {
				Kill = "35",
        Death = "5",
        Assist = "0",
        Time = "1分30秒",
        Val5 = "123",
        Val6 = "321",
        Val7 = "132",
        Val8 = "923",
        Val9 = "292"
			}
		},
    --第二位玩家的数据
		{
			PlayerKey = UGCGameSystem.GetPlayerKeyByPlayerController(AllPlayerController[2]);
			Results = {
				Kill = "25",
        Death = "20",
        Assist = "0",
        Time = "1分20秒",
        Val5 = "123",
        Val6 = "321",
        Val7 = "132",
        Val8 = "923",
        Val9 = "292"
			}
		}
	};

	--构造队伍结算数据
  local TeamResults = {
		--第一位玩家所在的队伍为胜利方
    {
    	TeamID = UGCPlayerControllerSystem.GetTeamID(AllPlayerController[1]);
      DetermingResult = "40";
    },
    --第二位玩家所在的队伍为失败方
    {
    	TeamID = UGCPlayerControllerSystem.GetTeamID(AllPlayerController[2]);
      DetermingResult = "30";
    }
	};

	--开始结算
  GameResultManager:BeginSettlement(PlayerResults,TeamResults,false);
end

function UGCPlayerController:EndSettlement()
	--结束结算
  GameResultManager:EndSettlement();
end

```

点击第一个按键，开始结算，出现结算界面与结算动画。

## API参考

### **Functions**

#### **BeginSettlement**

开始结算

