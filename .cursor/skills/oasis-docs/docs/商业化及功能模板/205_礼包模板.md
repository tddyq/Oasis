# 礼包模板

开发者可以将商品、货币或者其他各类道具整合成礼包的形式进行促销售卖，也可以作为游戏内刺激玩家持续追逐目标的激励物，礼包模板提供了便捷的配置方案，支持常规礼包与自选礼包两种礼包形式，是适用于 [商城](https://developer.gp.qq.com/wikieditor/#/catalog/20026)、[抽奖](https://developer.gp.qq.com/wikieditor/#/catalog/20027)、[签到](https://developer.gp.qq.com/wikieditor/#/catalog/20060) 等多种应用场景的通用解决方案。

<br>

## 模板结构

礼包分为常规礼包和自选礼包两类，不同类型礼包的模板结构略有差异。

### 常规礼包

玩家无法在常规礼包中主动选择奖励物品，当使用礼包后即获得礼包中的全部物品。

![QQ20241115-21289-HD.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/uikY6QQ20241115-21289-HD.gif)

**使用礼包界面**

当打开礼包时会自动弹出使用礼包的二次确认界面，该界面上会展示礼包相关的详情信息。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Qdognimage.png)

**A. 礼包弹窗标题**

配置礼包使用时的弹窗标题（非礼包名称），字数限制14个字以内

**B. 礼包详情**

礼包自身也是一种物品，与普通物品一样具有物品名称、物品描述及物品的icon等属性

【礼包名称】

配置在 [物品表](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/20008?autoJump=%E7%89%A9%E5%93%81%E8%A1%A8%EF%BC%88UGCObject%EF%BC%89) 中的物品名称，字数限制7个字以内

【礼包描述信息】

配置在 [物品表](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/20008?autoJump=%E7%89%A9%E5%93%81%E8%A1%A8%EF%BC%88UGCObject%EF%BC%89) 中的物品描述，字数限制同物品描述

【礼包库存】

当前已拥有的礼包数量

【礼包icon图】

配置在 [物品表](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/20008?autoJump=%E7%89%A9%E5%93%81%E8%A1%A8%EF%BC%88UGCObject%EF%BC%89) 中的小icon，尺寸规格：55*55 

**C. 数量选择器**

模板自带数量选择器，当库存数量>1时可以选择当次需要使用的礼包数量，此时获得的物品数量为 **``使用的礼包数量 × 单礼包中各物品的数量``**

---

### 自选礼包

自选礼包支持玩家单次选择其中一份奖励，因此比常规礼包多一级交互，当玩家选择了目标奖励物品后才获得对应的物品。

![QQ20241115-21419-HD.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/RTb59QQ20241115-21419-HD.gif)

**可选物品界面**

弹窗中展示礼包下所有的可选物品，各物品包含 [物品表](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/20008?autoJump=%E7%89%A9%E5%93%81%E8%A1%A8%EF%BC%88UGCObject%EF%BC%89) 中配置的物品名称、物品icon图及设定的物品数量，其中物品名称默认只展示7个字长度的内容，超出部分以 ...替代，长按物品可显示完整名称及物品描述信息。

![企业微信截图_1731659697623.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/R9cyq%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_1731659697623.png)

当自选礼包库存数量>1且一次性使用多个礼包时，需要手动选择满足 **``物品选择累计总数量 = 使用礼包数量``** 条件的物品：

## 下载模板

礼包模板通过 [资源商店](https://developer.gp.qq.com/wikieditor/#/catalog/336) 的形式提供下载，可以在资源商店中搜索“礼包模板”找到该模板，点击“下载资源”把模板下载至本地工程。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/m6n7Limage.png)

下载后的模板将以 ``ExtendResource`` 资源包的形式追加到工程里，其中 ``GiftPack/Arts_UI`` 和 ``GiftPack/Blueprint`` 文件夹内为核心配置资源：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/A9NqYimage.png)

- Arts_UI：UI相关配置的控件类均放置在 ``UIBP`` 子目录下
- Blueprint：礼包关联的子组件蓝图存放目录

## 配置依赖

礼包功能的运行依赖于 ``GiftPackComponent`` 组件 和 ``Gamepart`` 的特定配置，需要前置将其正确配置好。

### GamePart

礼包本质为虚拟物品，而虚拟物品已做了 [GamePart模块化](https://developer.gp.qq.com/wikieditor/#/catalog/20090) 封装，因此需要启用“GP_VirtualItemManager”模块功能。

## 礼包配置

``GiftPack_ApplyProp_Popup_UIBP`` 、``GiftPack_Complex_Popup_UIBP``、``GiftPack_Singular_Popup_UIBP`` 和 ``GiftPack_ItemGet_UIBP`` 为礼包交互的核心UI控件蓝图，开发者可以简单修改礼包弹窗标题，也可以完全自定义UI的风格；礼包及物品的关联数据配置在 ``物品表``、 ``礼包表`` 和 ``掉落表`` 中。

### 弹窗标题配置

礼包弹窗的标题可通过 ``GiftPack_ApplyProp_Popup_UIBP`` 控件蓝图的 ``CanvasPanel_UIPopup`` 画布下的 ``Title`` 属性进行修改。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Qbkjnimage.png)

---

### 物品表配置

除了礼包内的物品外，礼包自身也需要作为物品添加到物品表中，用以与商城、抽奖等其他模板进行关联索引，详细的配置方式可参考商业化系统文档中 [物品表配置](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/20008?autoJump=%E7%89%A9%E5%93%81%E8%A1%A8%EF%BC%88UGCObject%EF%BC%89) 部分内容。

### 掉落表配置

礼包中可以包含单个物品，也可以包含固定多种或者随机某几种的物品，因此礼包模板也是通过“掉落”的概念描述礼包的产出物，掉落表的具体配置方式可参见 [通用掉落](https://developer.gp.qq.com/wikieditor/#/catalog/20058)。

### 礼包表配置

礼包表决定礼包的类型、礼包的打开方式及如何产出物品，是礼包功能的核心数据。

> 创建方式：右键点击 ``各种各样 -> 数据表格 -> Gift Pack Data结构体``，创建的礼包表将自动添加到 ``Asset/Data/Table`` 目录下

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/8fx3dimage.png)

打开礼包表（UGCGiftPack），插入一条行数据，配置方式如下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/NyYSTimage.png)

- ID：礼包ID，该礼包的唯一索引
- ItemID：配置在物品表中的该礼包ItemID
- type：礼包类型，Normal(常规礼包)/Optional(自选礼包)
- OpenWay：礼包的打开方式，ManuallyOpen(手动打开)/AutoOpen(自动打开)，自动打开仅对常规礼包生效
- DropID：掉落表ID，与DropGroupID互斥，优先级更高
- DropGroupID：掉落组表ID，与DropID互斥

## 礼包调试

模板内置了一个简易的调试控件，方便开发者功能调试，该方式需要前置拥有一个礼包，可参考 [商城模板](https://developer.gp.qq.com/wikieditor/#/catalog/20026) 配置一个礼包型商品，通过购买的方式获得。

> 控件蓝图路径：ExtendResource/GiftPack/OfficialPackage/Asset/GiftPack/Blueprint/WBP_GiftPackBtn

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/3Phnximage.png)

将控件通过 [``AddToViewport``](https://developer.gp.qq.com/api/#/searchContent/UUserWidget?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUUserWidget.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UUserWidget&autoJump=AddToViewport) 添加到屏幕视口中，启动PIE调试后出现测试UI界面，在输入框内填写获得的礼包ID，点击【使用礼包】按钮即打开对应的礼包。

```lua
local GiftPackUIClass = UE.LoadClass(UGCGameSystem.GetUGCResourcesFullPath("ExtendResource/GiftPack/OfficialPackage/Asset/GiftPack/Blueprint/WBP_GiftPackBtn.WBP_GiftPackBtn_C"));
local PlayerController = STExtraGameplayStatics.GetFirstPlayerController(self);
local GiftPackUI = UserWidget.NewWidgetObjectBP(PlayerController, GiftPackUIClass);
GiftPackUI:AddToViewport(12000);
```

## API参考

礼包的数据可以通过 ``VirtualItemManager`` 提供的功能函数进行管理。

|函数库/类名|函数类型|函数功能范围|
|-|-|-|
|[VirtualItemManager](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager)|对象函数|虚拟物品的添加、删除、查询等数据管理|

模板中提供了部分额外的封装函数可供调用，需主动引入 ``GiftPackManager`` 类文件：

```lua
UGCGameSystem.UGCRequire("ExtendResource.GiftPack.OfficialPackage.Script.GiftPack.GiftPackManager")
```

|值|说明|
|-|-|

|Normal|常规礼包|
|Optional|自选礼包|

### **Functions**

#### **GetGiftPackageType**

查询礼包类型

```lua
-- 生效范围：客户端&&服务端
-- @param ItemID number 物品ID
function GiftPackManager:GetGiftPackageType(ItemID):GiftPackageType
end
```

---

#### **OpenNormalGiftPackage**

打开常规礼包，通过 ``OnOpenGiftPackageDelegate`` 委托获取打开结果

```lua
-- 生效范围：客户端&&服务端
-- @param GiftPackID number 礼包ID
-- @param UseNum number 打开的礼包数量
-- @param PlayerController UGCPlayerController 玩家控制器
function GiftPackManager:OpenNormalGiftPackage(GiftPackID, UseNum, PlayerController)
end
```

---

#### **OpenOptionalGiftPackage**

打开自选礼包，通过 ``OnOpenGiftPackageDelegate`` 委托获取打开结果

```lua
-- 生效范围：客户端&&服务端
-- @param GiftPackID number 礼包ID
-- @param UseNum number 打开的礼包数量
-- @param ItemList LuaTable<ItemID，Num> 选择的物品列表
-- @param PlayerController UGCPlayerController 玩家控制器
function GiftPackManager:OpenOptionalGiftPackage(GiftPackID, UseNum, ItemList, PlayerController)
end
```

---

#### **GetPackageDropItems**

获取礼包可掉落物品列表

```lua
-- 生效范围：客户端&&服务端
-- @param GiftPackID number 礼包ID
function GiftPackManager:GetPackageDropItems(GiftPackID):LuaTable<ItemID，Num>
end
```

---

#### **AddGiftPackage**

发放礼包

```lua
-- 生效范围：服务端
-- @param GiftPackID number 礼包ID
---@param Num number 发放数量
---@param PlayerController UUGCPlayerController 玩家控制器
function GiftPackManager:AddGiftPackage(GiftPackID, Num, PlayerController):bool
end
```
