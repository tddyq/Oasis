# 商城模板

绿洲启元商城模板提供了一套通用的商城结构框架与统一的配置规范，开发者能够基于此模板，结合 [商业化系统](https://developer.gp.qq.com/wikieditor/#/catalog/20008) 功能快速打造绿洲玩法的个性化商城。

> 商城模板已经历过重构优化，不再向后兼容，开发者如果曾经下载过旧版本，需从 [内容浏览器](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/124?autoJump=%E5%86%85%E5%AE%B9%E6%B5%8F%E8%A7%88%E5%99%A8) 右键删除 ``ExtendResource`` 目录下对应的模板文件夹，再下载最新版本并按照本文档配置使用

<br>

## 模板结构

模板结构划分为五个部分：商城标题、货币类型、商品页签、商品橱窗及商品详情。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/nFYj5image.png)

**A. 商城标题**

配置商城名称及商城的icon图
- 名称限制：7个字以内
- icon规格：31*31（支持jpg、png格式），2MB以内

**B. 货币类型**

商品支持绿洲币购买，开发者也可以自定义专属货币，且货币属于物品类别，因此货币的配置复用 [物品表](https://developer.gp.qq.com/wikieditor/#/catalog/20008?autoJump=%E7%89%A9%E5%93%81%E8%A1%A8%EF%BC%88UGCObject%EF%BC%89)
> 自定义货币定位为二级货币，获取渠道需要设计为通过绿洲币购买获得

**C. 商品页签**

开发者可以对售卖商品进行分类，每种类别对应一个页签，例如：“武器”、“消耗品”等
- 页签字数限制：4个字以内
> 至少需要配置一种页签

**D. 商品橱窗**

橱窗是向玩家展示可售卖商品的区域，每件商品的展示内容包含：商品基础信息、限购信息、限时售卖信息、折扣信息及售卖状态。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/7MmEjimage.png)

【商品基础信息】

商品本质上是售卖的物品，因此每件商品的基础信息实际就是物品的信息，物品ID、物品名、物品描述以及物品icon图等均配置在 [物品表](https://developer.gp.qq.com/wikieditor/#/catalog/20008?autoJump=%E7%89%A9%E5%93%81%E8%A1%A8%EF%BC%88UGCObject%EF%BC%89) 中

【限购信息】

支持限购类型及限购数量的配置：
- 每日限购：每天限购固定数量，0点刷新
- 每周限购：每周限购固定数量，每周一0点刷新
- 永久限购：玩法内限购固定数量

【限时售卖信息】

如果商品的“是否售卖”属性选择了 ``限时售卖``，且配置了商品的上下架时间，当前处于有效时间内则会在商品的左下角显示剩余时间：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/fz1CRimage.png)

|时间粒度|显示内容|备注说明|
|-|-|-|
|天|剩X天下架|天数向下取整|
|小时|剩X小时下架|<1天，小时数向下取整|
|分钟|剩X分钟下架|<1小时，分钟数向下取整|
|<1分钟|剩<1分钟下架|-|

> 商品下架后将在橱窗取消该商品的展示，如果下架前未关闭商城界面，则显示“商品已下架”

【折扣信息】

当商品配置了折扣点及有效的折扣时间段时，会显示折扣标签及剩余折扣时间：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/kBYv4image.png)

|时间粒度|显示内容|备注说明|
|-|-|-|
|天|折扣剩X天|天数向下取整|
|小时|折扣剩X小时|<1天，小时数向下取整|
|分钟|折扣剩X分钟|<1小时，分钟数向下取整|
|<1分钟|折扣<1分钟|-|

> 1. 折扣结束时间 < 默认值“5000.01.01”即视为有效的时间段
> 2. 当限时售卖时间与折扣时间同时生效时，优先显示折扣剩余时间

【售卖状态】

商品的购买按钮会根据售卖价格、限售数量以及折扣情况的不同展示对应的按钮样式：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/RkVlwimage.png)

【品质色】

商品品质色由高至低分为7种

![未标题-1.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/97vgw%E6%9C%AA%E6%A0%87%E9%A2%98-1.png)

![image.31.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/dZCVoimage.31.png)

支持标准（长）尺寸：202x289

![image.33.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/186egimage.33.png)

**E.商品详情**

详情栏会根据对应的物品配置，展示物品的icon图、物品名称、物品描述、物品价格及售卖状态等信息：

## 下载模板

商城模板通过 [资源商店](https://developer.gp.qq.com/wikieditor/#/catalog/336) 的形式提供下载，可以在资源商店中搜索“商城模板”找到该模板，点击“下载资源”把模板下载至本地工程。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/B2jWximage.png)

下载后的模板将以 ``ExtendResource`` 资源包的形式追加到工程里，其中 ``Shop/Arts_UI`` 和 ``Shop/Blueprint`` 文件夹内为核心配置资源：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/GTvYLimage.png)

- Arts_UI：UI相关配置的控件类均放置在 ``UIBP`` 子目录下
- Blueprint：商城关联的蓝图类和组件类存放目录

## 配置依赖

商城功能的运行依赖于 ``ShopV2Component`` 组件和 ``GamePart`` 的特定配置，需要前置将其正确配置好。

### GamePart

商城的购买流程相关功能已做了 [GamePart模块化](https://developer.gp.qq.com/wikieditor/#/catalog/20090) 封装，因此需要启用“GP_CommodityOperationManager”模块功能。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/9xIEOimage.png)

---

### ShopV2Component

ShopV2Component组件用于同步玩家的购买及限购记录数据至客户端，因此需要将其配置在PlayerController上：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/cwm8Oimage.png)

- Show Test Button：是否默认启动展示商城界面
- Main UIClass Path：商城主界面UI蓝图控件类
- Test Button Path：测试按钮蓝图控件类
- Item Quality Table Path：[物品品质色](https://developer.gp.qq.com/wikieditor/#/catalog/20050?autoJump=%E7%89%A9%E5%93%81%E5%93%81%E8%B4%A8%E8%89%B2%E9%85%8D%E7%BD%AE) 表格路径

## 商城配置

商城以 ``Shop_MainUI_UIBP`` 为主界面，商城标题、商品页签、货币类型的配置都集成在该控件类中，而商品橱窗及商品详情的数据关联 ``物品表（UGCObject）`` 和 ``商品表（UGCShop）`` 的配置。

### 商城标题配置

商城标题由 ``TitleIcon`` 和 ``TitleText`` 两个控件组成，通过调整控件的属性来实现标题的修改。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pBP9uimage.png)

---

### 货币类型配置

绿洲币作为商品购买的默认货币类型，开发者可以通过 ``ShowOasisCoin`` 布尔属性控制是否在商城主界面上显示绿洲币余额：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Gb4t3image.png)

除了官方货币，开发者可以配置自定义的货币展示，由于二级货币也属于一类物品，因此每种货币都具有唯一的ItemID标识，货币的UI蓝图类为 ``Shop_Currency_UIBP``。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/3EVNNimage.png)

基于Shop_Currency_UIBP创建货币子类控件，并将该子控件添加在 ``Shop_MainUI_UIBP`` 蓝图中的 ``CurrencyBar`` 水平框下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/SMMJjimage.png)

---

### 物品品质色配置

商城支持为商品添加品质色，目前从高阶到低阶共分七种级别的品质色：

![image.28.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/uwQBRimage.28.png)


品质色通过配表实现，内容浏览器右键选择 ``各种各样 -> 数据资源``，基于 ``ShopV2ItemQuality`` 结构体创建物品品质表。

![5.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/jTGie5.png)
![image.25.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/PJPOOimage.25.png)

品质与物品关联，因此同类物品将显示相同的品质色。

|字段|说明|

将品质表配置到 ``UGCPlayerController``蓝图的 ``ShopV2Component`` 组件中。

### 商品页签配置

由于商城可能具有多个类别的商品，因此页签数据是一个“ID + 名称”的数组结构，开发者可以按需添加页签数组的元素，但需要确保至少有一个页签元素：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/35c7Timage.png)

- TabID：页签ID，可自行定义此ID，需要与商品表的 ``商城页签ID`` 对应
- TabName：页签名称
- SelectedTabID：默认选中状态的页签ID

---

### 橱窗及详情配置

商品相关数据均读取自物品表和商品表，详细的配置方式可参考 [商业化系统](https://developer.gp.qq.com/wikieditor/#/catalog/20008) 文档中 ``商业化数据配置`` 部分内容。

## 商城调试

模板内置了一个简易的调试控件，方便开发者通过可交互的方式进行功能调试。

> 控件蓝图路径：ExtendResource/ShopV2/OfficialPackage/Asset/ShopV2/Blueprint/ShopV2_OpenShopButton_UIBP

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/eOufPimage.png)

将控件通过 [``AddToViewport``](https://developer.gp.qq.com/api/#/searchContent/UUserWidget?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUUserWidget.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UUserWidget&autoJump=AddToViewport) 添加到屏幕视口中，启动PIE调试后出现调试控件，点击【打开商城】按钮并执行购买验证。

```lua
local PlayerController = STExtraGameplayStatics.GetFirstPlayerController(self);
local ShopMainUIClass = UE.LoadClass(UGCGameSystem.GetUGCResourcesFullPath('ExtendResource/ShopV2/OfficialPackage/Asset/ShopV2/Blueprint/ShopV2_OpenShopButton_UIBP.ShopV2_OpenShopButton_UIBP_C'));
local ShopMainUI = UserWidget.NewWidgetObjectBP(PlayerController, ShopMainUIClass);
ShopMainUI:AddToViewport(1000);
```

![QQ202525-205013-HD.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/bMUvuQQ202525-205013-HD.gif)

## API参考

商城购买获得的物品为虚拟物品，因此可以通过 ``VirtualItemManager`` 管理玩家购买的物品数据。

|函数库/类名|函数类型|函数功能范围|
|-|-|-|
|[CommodityOperationManager](https://developer.gp.qq.com/api/#/searchContent/CommodityOperationManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FCommodityOperationManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=CommodityOperationManager)|对象函数|商品购买及数据查询，支持绿洲币及自定义货币购买|
|[VirtualItemManager](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2FOthers%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager)|对象函数|虚拟物品的添加、删除、查询等数据管理|

模板中提供了部分额外的封装函数可供调用，需主动引入 ``ShopV2Manager`` 类文件：

```lua
UGCGameSystem.UGCRequire("ExtendResource.ShopV2.OfficialPackage.Script.ShopV2.ShopV2Manager")
```
