# 占点团竞玩法

> <span style="color: #ff0000;font-weight:bold">由于该模板的实现存在一定的学习门槛，目前已推出《经典团队竞技2.0》玩法模板进行替代，开发者请以2.0的实现为准进行参考学习，不建议再使用此模板</span>

占点竞技玩法是FPS游戏中一种团队对抗模式，玩家分队争夺地图上的特定区域（占点），占领这些区域可以获得积分或胜利。该玩法强调团队合作与战术执行，竞争激烈，带来紧张刺激的战斗体验。

如果开发者希望在此模式的基础上创建自己的独特玩法，可以基于该模板创建工程并实现自己的创意。

打开绿洲启元编辑器，在玩法示例下找到  ``经典占点竞技玩法`` 工程，选择该工程，并在下方输入新的工程名字，点击新建工程，编辑器将会以此玩法模板为基础，创建新的独立工程。<br>

## 游戏模式配置
[GameMode](https://developer.gp.qq.com/wikieditor/#/catalog/352) 作为玩法全局管理模块，用于定义游戏的规则、流程和逻辑，控制游戏的整体行为。在占点模板中，不同于和平精英经典的大逃杀玩法，占点模式需要重新设置游戏模式（Game Mode)。

找到编辑器上方窗口栏，在窗口栏中找到【世界设置】按钮，点击弹出世界设置窗口。在世界窗口中找到 ``Game Mode`` 配置，检查游戏模式重载是否为 ``OccupyTeamDeathMatchGameMode`` 。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/OrxEAimage.png)

## 游戏规则配置
占点团竞模板将与游戏规则有关的所有具体配置（包括站点配置、队伍配置、军备武器配置及战斗规则配置等）集成在 ``OccupyTeamDeathMatchParamSetting`` 蓝图中，开发者可以高效地在此调整各配置项，从而实现想要的效果。

首先，找到项目工程下 ``Assert/Blueprint`` 目录，找到并双击打开 ``OccupyTeamDeathMatchGameMode`` 蓝图。<br>

打开后，在【细节面板】中找到 ``Param Setting Class`` 属性，查找配置的 ``OccupyTeamDeathMatchParamSetting`` 蓝图。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/7asg9image.png)

``OccupyTeamDeathMatchParamSetting`` 蓝图中集成了全部的游戏规则，在 ```GameMode``` 中配置完成后，然后在 ``Assert/Blueprint`` 目录下打开该蓝图，对规则进行配置。<br>

### 队伍配置
进入 ```OccupyTeamDeathMatchParamSetting``` ，找到【细节面板】中 ``队伍设置`` 部分，在此可以配置队伍参数。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/j4pnCimage.png)

**队伍人数**

在此配置两个队伍的人数，其中0为红队，1为蓝队，每队最多支持4人。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Dl9ICimage.png)

**出生区域关联**

在此配置两个队伍的出生区域，每个区域下有八个出生点（BP_STPlayerStart），出生区域蓝图为 ``BP_PlayerSpawnArea`` ，设置参数 ``Born Point ID`` 与队伍进行关联，如此处红队配置为2，蓝队配置为3。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/OraCnimage.png)

>**注意：** 出生区域ID需大于1

查看场景中BP_PlayerSpawnArea的配置，共有两个BP_PlayerSpawnArea，分别对应红队和蓝队，其BornPointID属性分别设置为2和3，以此和队伍的出生区域相关联。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Ar104image.png)

**角色相关参数**

- 生命值配置区间：5~30000
- 移动速度配置区间：0.1~3.0
- 重力缩放配置区间：0.5~2.0

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/wRZ75image.png)

> 以上属性如果只配置一条，则代表对两队都通用

**呼吸回血Buff**

### 装备配置

找到【细节面板】中 ``TeamMatch`` 配置，在此可以配置游戏中玩家可以选择使用的武器装备，目前游戏提供了两种配置方式，分别为自选装备和队伍装备。
- 自选装备：可以让玩家自由搭配装备
- 队伍装备：玩家只能使用开发者提前设定好的武器装备，不能更换。

对于这两种模式，可以设置不同的开局等待时间，如自选装备模式需要更久一些的等待时间供玩家选择装备，而队伍装备模式则不需要太长时间。

> 每个队伍都必须至少配置一种装备，否则玩家可能无法获得装备

**自选装备列表**

勾选此选项，开发者可以将装备模式设置为自选装备，若不勾选，将使用队伍装备。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/x8AHvimage.png)

开发者可以在此处自定义添加想要的装备数据，添加的装备在游戏中将能被玩家使用

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/OTOZYimage.png)

> 此处为两队的通用配置

自选列表目前默认拆分为10个独立的列表，每个独立列表代表一种装备类型，如0号列表为突击步枪类型装备，具体可参考如下。<br>
|  序号 | 装备类型 |
| :--: | :--:|
| 0 | 步枪 |
| 1 | 冲锋枪 |
| 2 | 狙击枪 |
| 3 | 霰弹枪 |
| 4 | 特殊武器 |
| 5 | 近战 |
| 6 | 配件 |
| 7 | 子弹 |
| 8 | 装备 |
| 9 | 道具 |

**队伍装备**

如果没有勾选开启自选装备，或勾选了但没有配置自选装备列表，将会使用队伍装备作为默认装备配置。

队伍装备列表有0和1两个列表，其中0代表红队装备，1代表蓝队装备，配置完成后，玩家进入游戏分队后将默认使用相对应列表的装备。目前每个队伍装备列表有15个插槽，其中0-9已经使用，10-14未使用，开发者可以根据需要增加、删除、修改装备数据。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/ygTWFimage.png)

>注意：如果对其中某队没有配置装备列表，并且也没有勾选是否可以自选装备，该队进入游戏将不会获得任何装备。

**装备数据单元**

无论是队伍装备配置，还是自选装备配置，基础的装备数据单元都是一样的，分为三部分：物品表ID，数量。

- 物品表ID：需要插入的装备物品ID（包括枪械、投掷物、配件等），可在编辑器内置的物品表中查询。
- 数量：插入的装备数量。
- 配件列表：此处可以为主武器加入配置，如三倍镜，消声器等，不能配备子弹、背包、药品、手雷等其他道具装备，**此处配置对自选装备列表无效**。

如图所示，此处插入了一个AKM突击步枪装备。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/1yjo9image.png)

> 1. 在添加装备数据时，需要使用装备物品ID，详细ID可在资源目录内置的物品表 ``UGCItemOpen`` 中查询。
> 2. 队伍装备上限：玩家可携带的装备拥有上限，与背包容量有关，游戏开始时，系统将会从0开始依次向下将装备插入玩家背包，达到上限后，后续装备不再发放。开发者在配置完成队伍装备列表后，需手动调试游戏，确认装备发放情况。

找到【细节面板】中 ```TeamMatch_Occupy``` 配置项，在此可以配置游戏中的据点信息。<br>

模板中已经配置好了三个据点信息，每个据点信息如下：

- 据点ID：唯一，与场景中的据点实例关联
- 是否开启：配置该据点是否启用

配置完据点信息后，需要将据点蓝图放入场景中，模板内场景中已经放置了三个据点实例，如图所示：<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/W1zQ8image.png)

如果需要增加更多的据点，可在资源目录内找到占点区域蓝图 ``BP_BattleArea``，并将其放入场景中。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/1DsLPimage.png)

> 据点数量最多只能配置五个，在场景中分别以A、B、C、D、E标识

#### 占点区域蓝图
占点区域蓝图放入场景中实例化后，还需要和 ```TeamMatch_Occupy``` 配置项中的据点数据相关联。在占点区域实例的配置项中找到 ```Battle Area ID``` 属性，填入 ```TeamMatch_Occupy``` 中的据点ID。<br>

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/fPRp5image.png)

除此之外，开发者还可以对占点区域实例添加其他配置，参考如下：

- Battle Area ID：据点ID
- Source Per Sec：占领该据点的队伍每秒可以增加的积分
- Occupied Time：玩家占领该据点需要的总时间
- Unlock Time：游戏开始后该据点启用的解锁时间
- Battle Area Color Setting：激活/占领/争夺三种状态下的区域颜色
- State Change Event：自定义据点状态变化的委托触发回调事件名
- Event Enable：自定义启用据点的事件名
- Event Disable：自定义禁用据点的事件名

**据点状态变化**

据点拥有四种状态，分别为：中立、占领中、争夺中、已占领，开发者可以自定义事件名用于监听这四种状态的变化。<br>

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/uxKT4image.png)

使用 ```UGCTemplateActorEventManager``` 对这四个状态绑定委托回调函数：

``` lua
UGCTemplateActorEventManager.TemplateActerDelegate:Add(Callback(eventName), DelegateWrapper) -- 绑定委托
UGCTemplateActorEventManager.TemplateActerDelegate:Remove(Callback(eventName), DelegateWrapper) -- 委托解绑
```

示例代码：

``` lua
local UGCTemplateActorEventManager = require("ugc.TemplateActor.UGCTemplateActorEventManager")

function UGCScoreBall:ReceiveBeginPlay()
    if self:HasAuthority() then
        UGCTemplateActorEventManager.TemplateActerDelegate:Add(self.OnReceiveEvent, self)
    end
end

function UGCScoreBall:ReceiveEndPlay()
    if self:HasAuthority() then
        UGCTemplateActorEventManager.TemplateActerDelegate:Remove(self.OnReceiveEvent, self)
    end
end

function UGCScoreBall:OnReceiveEvent(Param)
    if self:HasAuthority() then
        if Param and type(Param) == "string" then
            if Param == "Contention" then -- 判断事件名
                print("据点争夺中")
            end
        end
    end
end
```

**启动/禁用据点事件**

``Event Enable`` 和 ``Event Disable`` 定义脚本中发送启用与禁用的事件，方便动态控制据点区域的状态。以下方配置为例，定义“EnableArea”为启用据点的事件，“Disable”为禁用据点的事件：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/BIa0simage.png)

脚本中调用 ```UGCTemplateActorEventManager::SendEvent``` 函数可以发送指定事件用以控制玩家是否可占领指定区域，示例代码：

``` lua
local UGCTemplateActorEventManager = require("ugc.TemplateActor.UGCTemplateActorEventManager") 
-- 发送自定义事件，禁用该区域
UGCTemplateActorEventManager:SendEvent("DisableArea") 
```

### 战斗规则配置
找到【细节面板】中 ``流程规则`` 和 ``淘汰设置`` 两个配置项，这两个配置项包含了主要的游戏规则，如回合时间、目标分数、重生时间等。

**流程规则**
- 是否支持队伍自定义选择： 勾选【是否支持队伍自定义选择】时，玩家进入游戏可自由选择队伍。不勾选则随机分队。
- 回合准备时间： 游戏开始前的备战时间，在此期间玩家可以换队、配置武器装备等，配置区间为10s~60s。
- 回合战斗时间： 单回合时间，时间到了回合结束，配置区间为90s~600s。
- 目标分数： 单回合的目标分数，分数达标时，回合结束。配置为-1则不启用该配置，配置区间为10分~200分。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/IZeQRimage.png)

**注意：**
>占点竞技游戏规则中，目标分数为首要获胜条件，如果一方队伍率先达到目标分数条件，即可获得胜利；
如果在规定时间内双方队伍都未达到目标分数，则分数高的一方获胜。

**淘汰设置**
- 重生等待时间： 角色死亡之后，等待x秒复活，配置区间为0.5s~5s。
- 重生无敌时间： 角色重生后，获得x秒的无敌buff，配置区间为0~5s。

## 模板调试

配置完工程后，可以在编辑器内进行调试，以测试配置项是否达到预期。

**1. 添加玩家人数**

点击调试选项菜单，可以设置调试需要接入的客户端数量，客户端总数 = 每队玩家数量 × 队伍数量，建议与队伍配置的总人数保持一致。

如果填写人数超过对局总人数，游戏依旧可以启动成功，但是多出的玩家将无法选队以及获得装备。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/omqxUimage.png)

点击调试按钮即可启动游戏的调试。<br>
