# 军备团竞玩法

> <span style="color: #ff0000;font-weight:bold">由于该模板的实现存在一定的学习门槛，目前已推出《经典团队竞技2.0》玩法模板进行替代，开发者请以2.0的实现为准进行参考学习，不建议再使用此模板</span>

军备团竞是一种常见的FPS团竞类游戏模式，相比于 [经典团竞](https://developer.gp.qq.com/wikieditor/#/catalog/20015)，军备团竞增加了一些新的玩法元素：每个玩家可以通过累计击杀敌人数量来升级自己的武器，这种玩法更加强调团队合作和策略，增加了游戏的挑战性和趣味性。

如果开发者希望在此模式的基础上创建自己的独特玩法，可以基于该模板创建工程并实现自己的创意。

## 新建模板工程
打开绿洲启元编辑器，在玩法示例下找到  ``Template_ArmsTeamCompetition`` 工程，选择该工程，并在下方输入新的工程名字，点击新建工程，编辑器将会以此玩法模板为基础，创建新的独立工程。<br>
![企业微信截图_1723548974523.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/0a9jd%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_1723548974523.png)

## 游戏模式配置
[GameMode](https://developer.gp.qq.com/wikieditor/#/catalog/352) 作为玩法全局管理模块，用于定义游戏的规则、流程和逻辑，控制游戏的整体行为。在团竞模板中，不同于和平精英经典的大逃杀玩法，团队竞技模式需要重新设置游戏模式（Game Mode)。

找到编辑器上方窗口栏，在窗口栏中找到【世界设置】按钮，点击弹出世界设置窗口。在世界窗口中找到 ``Game Mode`` 配置，检查游戏模式重载是否为 ``ArmsTeamDeathMatchGameMode`` 。<br>
![企业微信截图_17235499101494.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/UASm3%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_17235499101494.png)

## 游戏规则配置
军备团竞模板将与游戏规则有关的所有具体配置（包括队伍配置、军备武器配置及战斗规则配置等）集成在 ``ArmsTeamDeathMatchParamSetting`` 蓝图中，开发者可以高效地在此调整各配置项，从而实现想要的效果。

找到项目工程下 ``Assert/Blueprint`` 目录，找到并双击打开 ``ArmsTeamDeathMatchGameMode`` 蓝图。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/ue8QFimage.png)

打开后，在【细节面板】中找到 ``Param Setting Class`` 属性，查找配置的 ``ArmsTeamDeathMatchParamSetting`` 蓝图。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Ydcl7image.png)

``ArmsTeamDeathMatchParamSetting`` 蓝图中集成了全部的游戏规则，在 ``Assert/Blueprint`` 目录下打开该蓝图，对规则进行配置。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/F4YYHimage.png)

### 队伍配置
进入 ```ArmsTeamDeathMatchParamSetting``` ，找到【细节面板】中 ``队伍设置`` 部分，在此可以配置队伍参数。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/j4pnCimage.png)

**队伍人数**

在此配置两个队伍的人数，其中0为红队，1为蓝队，每队最多支持4人。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Dl9ICimage.png)

**出生区域关联**

在此配置两个队伍的出生区域，每个区域下有八个出生点（BP_STPlayerStart），出生区域蓝图为 ``BP_PlayerSpawnArea`` ，设置参数 ``Born Point ID`` 与队伍进行关联，如此处红队配置为2，蓝队配置为3。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/OraCnimage.png)

>**注意：** 出生区域ID需大于1

查看场景中BP_PlayerSpawnArea的配置，共有两个BP_PlayerSpawnArea，分别对应红队和蓝队，其BornPointID属性分别设置为2和3，以此和队伍的出生区域相关联。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Ar104image.png)

**角色相关参数**

- 生命值配置区间：5~30000
- 移动速度配置区间：0.1~3.0
- 重力缩放配置区间：0.5~2.0

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/wRZ75image.png)

> 以上属性如果只配置一条，则代表对两队都通用

**呼吸回血Buff**

规则配置蓝图中内置了一个Buff功能-呼吸回血，效果为玩家每秒可以自动恢复部分血量，开发者可以根据需要设置触发的时间间隔以及恢复的血量值。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/iOOqdimage.png)

从内容浏览器查看 ``Assert/BluePrint/UGCBuffs`` 目录，打开MyBuffList蓝图，具体参数配置可查找 ``BuffActions`` 属性，其中 ``Interval Time`` 为触发时间，默认为1s， ``Dot Value`` 为扣除的血量值，默认为-20（代表恢复血量），开发者可以根据需要自行进行修改。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/KqjBcimage.png)

> 回血Buff的生效依赖于PlayerController上的 ``HealthControlComponent`` 子组件，需要将 ``HealthRecoverBuffName`` 属性配置为对应的回血Buff名字：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/4DQ6Jimage.png)

**角色描边**

游戏中可以通过描边的效果来区分敌方玩家和友方玩家：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/2ZkS8image.png)

要想实现这个效果，需要在关卡场景里添加 ```PostProcessMgr``` 后处理组件，模板工程默认已经内置了这个组件蓝图。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Z8ndeimage.png)

> 如果没有此组件蓝图，可以在资源库中搜素“PostProcessMgr”找到并放置在关卡场景内。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/3Khxbimage.png)

```PostProcessMgr``` 组件蓝图提供了包含描边颜色、描边粗细度和描边范围等配置，开发者可以按需调整目标效果。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/30Ajqimage.png)

除了组件蓝图的配置，还需要手动调用API对需要增加描边效果的目标进行设置，描边支持角色Pawn，也支持普通Actor物件：

``` lua
-- 针对物体设置外发光效果
STExtraBlueprintFunctionLibrary.EnableActorOutline(Actor:AActor,bEnable:bool)

-- 针对玩家角色设置外发光效果
STExtraBlueprintFunctionLibrary.EnablePlayerAvatarOutline(PlayerPawn:ASTExtraBaseCharacter,bEnable:bool)
```

### 军备武器配置

找到【细节面板】中 ``TeamMatch_Arms`` 配置，在此可以配置游戏中不同队伍玩家的武器装备的升级列表。

**装备升级列表**

装备升级系统，玩家从第一套装备开始，按照列表顺序逐次向后解锁新的装备，在达到新的装备解锁要求后，将自动解锁装备，在这套系统中，需要用到的便是装备升级列表。其中，解锁装备的要求为积分，玩家每击败一次队手即可获得一点积分，如图所示，装备栏左边是目前正在使用的装备，右边是下一次即将升级的装备，右上角便是目前拥有的积分与需要获得的目标积分。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/txQegimage.png)


在 ``TeamMatch_Arms`` 下，有0和1两个列表，分别对应不同的两个队伍，开发者可以对每个队伍单独配置一套装备升级列表。如图所示，目前已经为每个队伍配置了十套装备，每套装备包含一把枪械武器以及其他相对应的道具，如背包，头盔等。

其中序号0对应的装备默认作为玩家的初始装备，后续序号为依次可以升级的装备。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/1Eg2Wimage.png)

每个装备插槽都有两个配置信息：积分以及物品列表。
 - 积分：从当前组武器升级到下一组武器需要的积分数量，达到积分数后，自动升级。
 - 物品列表：相对应的装备信息（枪械武器+道具配件）

如图所示，从序号0装备开始，升级到序号1装备，需要两个积分，从序号1装备升级到序号2装备，也需要两个积分。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pT5Azimage.png)

**物品列表配置**

开发者可以在物品列表中配置装备信息，列表信息分为三部分：物品表ID，数量，配件列表。

- 物品表ID：需要插入的枪械物品ID，唯一，可在编辑器内置的物品表中查询，注意，此处只能插入枪械武器ID。
- 数量：插入的装备数量。
- 配件列表：其他需要提供给玩家的非枪械装备，如手雷，背包，二级盔等。

如图所示，序列0物品列表配置的主武器为AKM突击步枪，配件列表中配置的其他道具为三级背包，特种部队头盔等物品。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/mkjKSimage.png)

> 1. 在添加装备数据时，需要使用装备物品ID，详细ID可在资源目录内置的物品表 ``UGCItemOpen`` 中查询。
> 2. 物品列表中每个物品插槽外部物品表ID（非配件列表），必须配置枪械武器ID。
> 3. 目前可配置的装备中，不支持投掷物。

### 战斗规则配置

找到【细节面板】中 ``流程规则`` 和 ``淘汰设置`` 两个配置项，这两个配置项包含了主要的游戏规则，如回合时间，目标分数，重生时间等。

**流程规则**

- 是否支持队伍自定义选择：勾选【是否支持队伍自定义选择】时，玩家进入游戏可自由选择队伍。不勾选则随机分队。
- 回合准备时间：游戏开始前的备战时间，在此期间玩家可以换队、配置武器装备等，配置区间为10s~60s。
- 回合战斗时间：单回合时间，时间到了回合结束，配置区间为90s~600s。
- 目标分数：枪械升级总积分。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/IZeQRimage.png)

**注意：**
> 军备团竞游戏规则中，队伍里如果任意一名玩家率先达到枪械升级总积分，即可获得胜利。

**淘汰设置**

- 重生等待时间：角色死亡之后，等待x秒复活，配置区间为0.5s~5s。
- 重生无敌时间：角色重生后，获得x秒的无敌buff，配置区间为0~5s。

## 模板调试

配置完工程后，可以在编辑器内进行调试，以测试配置项是否达到预期。

**1. 添加玩家人数**

点击调试选项菜单，可以设置调试需要接入的客户端数量，客户端总数 = 每队玩家数量 × 队伍数量，建议与队伍配置的总人数保持一致。

如果填写人数超过对局总人数，游戏依旧可以启动成功，但是多出的玩家将无法选队以及获得装备。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/LHdGtimage.png)

**2. 启动调试**

点击调试按钮即可启动游戏的调试。<br>
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/XC7nzimage.png)

> 更多和调试有关的信息可查看 [启动调试游戏](https://developer.gp.qq.com/wikieditor/#/catalog/307) 与 [调试说明和技巧](https://developer.gp.qq.com/wikieditor/#/catalog/354)
