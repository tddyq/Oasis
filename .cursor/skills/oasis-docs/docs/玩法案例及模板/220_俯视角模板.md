# 俯视角模板

俯视角模板是专门为俯视角游戏玩法设计的开发框架，提供了一套完整的控制、视角和射击系统解决方案。该模板基于双摇杆操作范式构建，能够快速创建俯视角射击、动作等类型的游戏。

<br>

## 模板功能说明

区别于“空模板”，俯视角模板预置了2.5D类游戏所需的3C系统设置参数，让开发者能跳过基础3C的繁杂实现过程，仅需简单的参数配置就能快速进入俯视角玩法的创作。

|特性维度|2.5D俯视角模板|普通空模板|
|-|-|-|
|核心相机|预置固定角度俯视角相机|默认为可自由旋转的3D跟随相机，需手动调整至俯角|
|控制方案|内置多套摇杆控制逻辑（双摇杆、单摇杆、联动）|需从零开始编写或集成移动、旋转的输入与控制逻辑|
|射击体验|集成俯视角专用射击系统（瞄准线、右摇杆开火、粗略命中模式）|使用标准的第一/三人称射击逻辑，在俯视角下可能出现体验问题（如子弹与视角不符）|
|UI与交互|预置适配俯视角的动态准星、瞄准线及UI布局。|需自行设计并适配俯视角下的UI交互逻辑|

### 输入设置
此部分配置影响玩家使用键盘鼠标等输入设备时的操作手感和响应逻辑。

- 武器类型Pitch限制：是一个Map类型的配置，Key表示武器类型，Value表示这个类型的武器相关的俯仰角配置
  - Pitch下限/上限：表示持有这个武器时候，实际瞄准的俯仰角范围
  - 手雷速度比率下限/上限：手持手雷的时候，右摇杆会控制手雷的投出速度，摇杆的长度会和这个上下限对应，以实现摇杆长度控制手雷速度的机制
  - 鼠标输入距离下限/上限：当在电脑上时候，鼠标距离角色的远近多少才会影响输入
- 默认Pitch限制：当初始化的时候，角色没有持有任何武器，这时候的Pitch相关的配置
- 手雷起点偏移：在俯视角下，投掷手雷的时候，手雷抛出的起点相对于角色的偏移
- 朝向计算死区半径：用来控制摇杆或者鼠标输入控制角色旋转的死区，一般来说，这个值小的话，一些轻微的摇杆移动就可以让角色产生旋转
- 显示Debug参考线：勾选之后会在场景中显示枪械的调试瞄准线，一般不需要勾选

---

### 视角设置
此部分配置控制游戏中摄像机的摆放位置、角度和视野。

- 模式视角朝向：相机相对于角色的旋转角度
- 模式视角距离：相机与角色之间的距离
- 模式视角FOV：相机的视野范围

---

### 枪械设置
此部分配置管理游戏中所有枪械的通用射击、瞄准和命中判定规则。

- 射击起点类型：支持从角色位置或枪口位置发射射线
- 射击朝向类型：支持枪口朝向或角色朝向两种瞄准方式
- 粗略命中模式：简化命中检测，提高游戏流畅度
- 射击目标距离：全局射线检测最远距离
- 瞄准线配置：瞄准线蓝图配置
- 是否允许开镜：俯视角模版中不允许开镜
- 使用UI左摇杆：勾选将会永久隐藏和平自带的左摇杆

另外，此蓝图中还提供了两个接口供开发者自定义某个枪械对象使用的瞄准线资产以及瞄准线的长度和角度，创建蓝图之后打开对应的Lua文件，可以看到已经有了这两个函数。
```lua
---@class BP_UGCCustomSightLine_TopDown_C:UUGCCustomModeSightLine
local BP_UGCCustomSightLine_TopDown = {}

--更新枪械瞄准线特效
---@param ShootWeapon ASTExtraShootWeapon @枪械对象
---@param Range float @瞄准线长度
---@param Angle float @瞄准线角度
---@return UParticleSystemComponent @瞄准线特效组件
function BP_UGCCustomSightLine_TopDown:UpdateEffect(ShootWeapon, Range, Angle)
    local Effect = BP_UGCCustomSightLine_TopDown.SuperClass.UpdateEffect(self, ShootWeapon, Range, Angle)
    local Character = ShootWeapon:GetOwnerPlayerController():GetControlledPawn()
    local bSwitchWeapon = UGCPlayerPawnSystem.HasPawnState(Character, EPawnState.SwitchWeapon)
    Effect:SetActive(!bSwitchWeapon, false)
    return Effect
end

--根据枪械对象获取瞄准线特效资产模板
---@param ShootWeapon ASTExtraShootWeapon @枪械对象
---@return UParticleSystem @瞄准线特效模板
function BP_UGCCustomSightLine_TopDown:FindSightLineTemplateByShootWeapon(ShootWeapon)
    return BP_UGCCustomSightLine_TopDown.SuperClass.FindSightLineTemplateByShootWeapon(self, ShootWeapon)
end

return BP_UGCCustomSightLine_TopDown
```
其中UpdateEffect重载函数为每帧调用（只要枪械在玩家槽位上就会执行，不管是否手持），用来更新瞄准线的范围和角度的；FindSightLineTemplateByShootWeapon重载函数是针对某个特定对象的枪械，使用哪个粒子模板，默认会返回蓝图中配置的数据，当装备枪械的时候会调用一次。

当前工程中默认提供了两个类型的瞄准线：

- 射线类型指示器：一个特效，有一个名称为LaserBeamTarget的参数，类型为Vector，表示这个射线的终点位置，若开发者有自定义需求，则可以在UpdateEffect函数中，通过手动设置特效的这个参数来实现自定义射线终点的需求。射线起点固定为客户端枪口位置

- 扇形类型指示器：一个特效，有一个参数，名称为Angle，类型为float，表示这个特效的展开角度，对应霰弹枪的散射，若开发者有自定义需求，则可以在UpdateEffect函数中，修改这个特效的缩放(射程)和这个参数（散射角度），来实现自身需求。

若不手动修改Lua代码，则使用的射线类型瞄准器中必须有LaserBeamTarget参数，类型为vector，来表示射线终点；使用的扇形类型瞄准器中必须有Angle参数，类型为float，表示扇形角度。若没有，则瞄准线的表现将会出问题。

当前特效默认挂载在枪械模型上，位置为枪械的枪口位置，所以在枪械开火或者换弹等操作时候，可能会出现瞄准线特效异常抖动的问题，开发者可以在UpdateEffect函数中，手动修改瞄准线的挂载组件为玩家角色根组件，然后修改其相对坐标，以实现瞄准线稳定显示的效果，代码如下所示：
```lua
--更新枪械瞄准线特效
---@param ShootWeapon ASTExtraShootWeapon @枪械对象
---@param Range float @瞄准线长度
---@param Angle float @瞄准线角度
---@return UParticleSystemComponent @瞄准线特效组件
function BP_SD:UpdateEffect(ShootWeapon, Range, Angle)
    local Effect = BP_SD.SuperClass.UpdateEffect(self, ShootWeapon, Range, Angle)
    local Controller = ShootWeapon:GetOwnerPlayerController()
    local Character = UGCPlayerControllerSystem.GetPlayerCharacter(Controller)
    Effect:K2_AttachToComponent(Character.RootComponent, "NAME_None", EAttachmentRule.KeepRelative, EAttachmentRule.KeepRelative, EAttachmentRule.KeepWorld, true)
    Effect:K2_SetRelativeLocationAndRotation(KismetMathLibrary.MakeVector(80, 0, 40), KismetMathLibrary.MakeRotator(0, 0, 0), false, false)
    -- local bSwitchWeapon = UGCPlayerPawnSystem.HasPawnState(Character, EPawnState.SwitchWeapon)
    -- Effect:SetActive(not bSwitchWeapon, false)
    return Effect
end
```

### 双摇杆-独立模式

经典的双摇杆操作，左摇杆控制移动，右摇杆独立控制朝向。

![2026-01-20170644-ezgif.com-video-to-gif-converter.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/FQur62026-01-20170644-ezgif.com-video-to-gif-converter.gif)

右摇杆集成开火功能：右摇杆存在一个死区，在这个范围内不会有枪械开火输入，但是仍然会控制角色旋转，当右摇杆移动距离超过死区时候，根据不同类型的枪械，有不同模式的开火逻辑。
- 会一直开火，当手指抬起或者重新进入死区则停止开火，一般适用于一些全自动类型的枪械，比如冲锋枪
- 不会开火，但是会记录当前状态为准备开火，若此时手指抬起则执行一次开火（不抬起重新回到死区时候不开火），一般适用于一些单发类型的枪械，比如霰弹枪和狙击枪

### 自定义控制
通过API直接注入输入，实现完全自定义控制逻辑。

若开发者不想使用摇杆来控制角色的移动和朝向，也可以直接通过调用底层的API接口来进行控制
本质上两个摇杆产生的输入最终都会映射到四个输入tag上
  - Input.Move.MoveRight：为正值则表示角色朝向世界坐标中的正Y方向移动
  - Input.Move.MoveForward：为正值则表示角色朝向世界坐标中的正X方向移动
  - Input.Look.Turn：为正值表示角色会朝向世界坐标的正X方向
  - Input.Look.LookUp：为正值表示角色会朝向世界坐标中的正Y方向

所以可以直接通过调用接口，给这些tag手动传入数值来直接控制角色的移动和旋转
当前提供的接口如下
```lua
function UGCInputSystem.InjectInputMapping(WorldContext, InputTag, Value)
end
```
  - WorldContext传入本地的玩家控制器
  - InputTag传入上面四个Tag字符串
  - Value传入输入的值，可正可负
>若需要手动控制角色的移动，建议在gamepart中将“使用UI左摇杆”勾选

### 怪物命中

因为俯视角中，枪械一般不需要配置俯仰角的变化，所有的枪械默认都是朝向XY平面射击的，一些比较矮小或者飞行在空中的怪物，若不进行特殊处理则可能会出现开枪无法命中的情况，若出现了此种情况，则需要更改玩法通用设置中的“粗略命中模式”。然后将怪物中的HitBox调解大一些以保证怪物在场景中时候玩家开启的水平射线可以命中这个HitBox。

射击极限高度为150，最少需要 ``胶囊体半高+位置高度/2`` 大于这个值才可以命中。

### 自动瞄准

默认的空模板中并没有提供俯视角自瞄机制，并且第三人称下的自瞄在俯视角中也是失效的，若开发者有自瞄机制的需求，可以按照下述步骤进行操作。
1.获取场景中需要瞄准的对象，可以使用技能中的扇形选取器或者完全自定义
2.计算对象坐标到角色坐标的二维方向向量
3.根据方向向量计算角色到目标的朝向旋转体
4.调用接口直接设置角色的朝向

整体的代码如下所示
```lua
-- 转向，执行时候会朝向目标队列中的第一个目标
function TopDownAutoAim:Turn_Lua()
    local Targets = self:GetSelectTargetActor(EPESkillSelectTarget.E_PESKILL_PickerType_AllTarget)
    local NetOwnerActor = self:GetNetOwnerActor()
    if Targets ~= nil and #Targets > 0 then
        local Target = Targets[1]
        if Target ~= nil then
            local TargetLocation = Target:K2_GetActorLocation()
            local NetOwnerLocation = NetOwnerActor:K2_GetActorLocation()
            local Direction = KismetMathLibrary.Subtract_VectorVector(TargetLocation, NetOwnerLocation)
            local _Rotator = STExtraBlueprintFunctionLibrary.Vector2Rotator(Direction)
            local NewRotator = Rotator.New(0, 0, _Rotator.Yaw)
            -- ugcprint_format("NewRotator: %s", _Rotator.Yaw)
            STExtraBlueprintFunctionLibrary.SetControlRotationByReason(NetOwnerActor,NewRotator,EUGCCustomModeInputReason.UCMIR_AutoAim)
            return
        end
    end
    STExtraBlueprintFunctionLibrary.ResetControlRotationReason(NetOwnerActor, EUGCCustomModeInputReason.UCMIR_AutoAim)
end
```
在这里面有一个最关键的接口SetControlRotationByReason，此接口会直接修改玩家角色的朝向，因为UGC中可能存在多种修改角色朝向的机制，比如技能中的吸附，左摇杆的移动，右摇杆的移动等等，所以需要一个优先级来管理这些输入，当前优先级如下

1. 右摇杆的操控
2. 技能中的修改
3. 自瞄类型的修改
4. 左摇杆移动引起的旋转

越靠上优先级越高，上面的输入会直接覆盖下面的。
接口中通过传入的第三个参数来控制，此参数为枚举值，分别对应上述的四个优先级
- UCMIR_Look：右摇杆引起的
- UCMIR_Skill：技能中引起的
- UCMIR_AutoAim：自瞄引起的
- UCMIR_Movement：左摇杆移动引起的
